1. مراجعة العمليات والأنشطة المشبوهة (Host Auditing)

أوامر تتبع العمليات المشبوهة التي تعمل بصلاحيات عالية وفحص سجلات المستخدمين النشطين:

ps -u root -f

2. مراقبة حركة الشبكة والاتصالات المفتوحة (Traffic Monitor)

رصد الاتصالات المفتوحة ومحاولات الاتصال الخارجية المشبوهة لمعرفة ما إذا كان السيرفر متصلاً بشل عكسي:

netstat -antp | grep :80

3. تأمين وسد ثغرات الأنظمة (Hardening)

أوامر تفقد الصلاحيات الضعيفة وتأمين النواة والملفات الهامة المانعة لهجمات رفع الصلاحيات:

# سد صلاحيات القراءة لملف shadow
chmod 600 /etc/shadow
# تفعيل حماية النواة ضد هجمات IP Spoofing
echo 1 > /proc/sys/net/ipv4/conf/all/rp_filter

4. تحليل السجلات وكشف هجمات التخمين (Logs Analysis)

فحص سجلات نظام التشغيل لتتبع محاولات التخمين والدخول الفاشلة للـ SSH:

grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort | uniq -c | sort -nr

5. جدول الأوامر الموحد للمدافعين (BTFM Cheat Sheet)

المهمة الأمر المستعمل فائدة الأمر
رصد الاتصالات المفتوحة netstat -antp | grep :80 كشف اتصالات الـ Reverse Shells
فحص العمليات النشطة ps -u root -f كشف العمليات غير المصرح بها
مراقبة حركة الشبكة الحية tcpdump -i eth0 -n تحليل الباكتات الحية على الكارت
فحص تعديلات الملفات find / -mmin -60 -type f 2>/dev/null البحث عن الملفات المعدلة بآخر ساعة

6. بنك الأسئلة والاختبارات التفاعلية الـ 10

🧪 بنك أسئلة مراجعة كتاب BTFM (انقر لعرض الإجابة النموذجية)
الإجابة النموذجية: يساعد `tcpdump` في الإنصات المباشر للحزم عبر واجهة الشبكة وتصفيتها (مثل `tcpdump -i eth0 port 80`) لتسجيل حركة البيانات المشبوهة وحفظها بصيغة `.pcap` لتحليلها لاحقاً ببرنامج Wireshark لرصد هجمات الاختراق والـ Shells.
الإجابة النموذجية: يمكن استخدام الأوامر السريعة مثل `who` أو `w` أو `last` لعرض قائمة بالمستخدمين المتصلين حالياً وعناوين الـ IP الخاصة بهم وتاريخ تسجيل الدخول ومنافذ الـ TTY المستخدمة لرصد أي دخول غير مصرح به.
الإجابة النموذجية: مراقبته تعتبر حيوية للتأكد من عدم قيام المهاجمين بإنشاء مستخدمين جدد يملكون صلاحيات الـ Root (UID=0) لتأمين بقائهم على الخادم (Persistence)، ويمكن استخدام أدوات المراقبة الحية مثل `incron` أو `auditd` لرصد أي تعديل بالملف.
الإجابة النموذجية: تقوم Fail2ban بمراقبة سجلات النظام (مثل `/var/log/auth.log`). بمجرد رصد عدد محدد من محاولات الدخول الفاشلة من نفس الـ IP (مثلاً 5 محاولات فاشلة خلال دقيقة)، تقوم الأداة بإضافة قاعدة في جدار الحماية (iptables) لحظر الـ IP تلقائياً لفترة محددة.
الإجابة النموذجية: هي طريقة لتأمين منفذ (مثل منفذ SSH 22) بإبقائه مغلقاً تماماً بجدار الحماية، ولا يفتح إلا للمستخدم الذي يرسل سلسلة محددة من حزم الاتصال (Knocks) لمنافذ مغلقة أخرى بترتيب معين، وبمجرد إرسال السلسلة يفتح جدار الحماية المنفذ المخصص للـ IP الخاص به تلقائياً.
الإجابة النموذجية: نظام كشف التسلل (IDS) يقوم بمراقبة الشبكة وتحليل الحزم وتنبيه مسؤولي النظام عند رصد حركة مرور مشبوهة دون التدخل لحظرها. بينما نظام منع التسلل (IPS) يتدخل بشكل مباشر ويقوم بحظر حزم البيانات المشبوهة أو قطع الاتصال تلقائياً لمنع نجاح الهجمة.
الإجابة النموذجية: يتم منعها عن طريق تفعيل حماية النواة ضد انتحال العناوين (Reverse Path Filtering) عبر ضبط المعيار `net.ipv4.conf.all.rp_filter = 1` في إعدادات النواة، مما يجبر السيرفر على التحقق من أن حزم البيانات الواردة تعود عبر نفس الواجهة التي خرجت منها.
الإجابة النموذجية: لأن أول ما يفعله المهاجم عند اختراق السيرفر هو محاولة مسح السجلات (auth.log, access.log) لإخفاء آثار جريمته وتجنب التحقيق. وجود سيرفر سجلات مركزي يضمن بقاء السجلات آمنة وبعيدة عن متناول يد المخترق لإجراء التحقيق الجنائي الرقمي بنجاح.
الإجابة النموذجية: يمكن الاستعانة بأمر البحث عن الملفات التي تم إنشاؤها أو تعديلها حديثاً (مثلاً خلال آخر 60 دقيقة) عبر الأمر `find / -mmin -60 -type f 2>/dev/null` لاستعراض الملفات الجديدة ورصد ملفات الشيل أو الـ backdoors التي تم رفعها.
الإجابة النموذجية: الـ HSTS (HTTP Strict Transport Security) يجبر متصفحات الزوار على الاتصال بالموقع بشكل مشفر بالكامل عبر HTTPS فقط، مما يمنع هجمات الـ SSL Stripping أو محاولات اعتراض حركة البيانات الحساسة وفك تشفيرها بالشبكة المحلية.
