1. نظرة شاملة على أداة Burp Repeater ولوحة المفتش (Inspector)

تُعد أداة Burp Repeater السلاح الأول والأساسي لباحثي الـ Bug Bounty. تتيح لك الأداة تعديل طلبات الـ HTTP وإعادة إرسالها مئات المرات مع مراقبة استجابة السيرفر لحظة بلحظة دون الحاجة لإعادة التفاعل مع المتصفح.

🔍 لوحة المفتش (Inspector Panel) وطرق العرض:

  • Raw View: عرض كود الـ HTTP الخام للطلب والاستجابة كما ينقل عبر الشبكة.
  • Pretty View: تنسيق الأكواد المرتجعة (HTML/JSON/XML) لتسهيل القراءة.
  • Hex View: عرض القيم السداسية عشرية، وهو ممتاز لاكتشاف الحروف المخفية وقيم الـ Null Bytes.
  • Inspector: لوحة جانبية ذكية تقسم الطلب لـ Request Headers و Query Parameters و Cookies، وتتيح لك تعديل أي عنصر بضغطتين.
💡 اختصارات لوحة المفاتيح الذهبية:
• `Ctrl + R`: إرسال الطلب المعترض فوراً من الـ Proxy إلى الـ Repeater.
• `Ctrl + Space`: إرسال الطلب الحقيقي داخل الـ Repeater لرؤية الاستجابة.

💡 How Bug Bounty Hunters Actually Use Repeater

كيف يفكر الباحث الأمني عند استخدام Repeater؟ إليك سير العمل العملي المتبع في الصيد الحقيقي:

1️⃣ Intercept Request: اعتراض طلب حسّاس (مثل الشراء، تعديل الملف، أو البحث) عبر الـ Proxy.
2️⃣ Send to Repeater (Ctrl+R): تحويل الطلب للـ Repeater لعزله واختباره بحرية.
3️⃣ Modify Parameter: تغيير معامل واحد فقط في كل مرة (مثل id أو role أو price) لعزل السبب والنتيجة.
4️⃣ Observe Response (Status & Length): مراقبة رمز الاستجابة (200 vs 403 vs 500) وحجم الـ Response Length.
5️⃣ Identify Vulnerability & Exploit: تأكيد وجود الثغرة وتطوير كود استغلال موثق (PoC) لتقديمه للمنصة!
