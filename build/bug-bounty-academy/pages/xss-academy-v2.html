📤 HTTP Request

عندما تضغط على رابط أو تُرسل Form، المتصفح يُرسل HTTP Request للسيرفر. يحتوي على: Method (GET/POST)، Headers (Cookie, User-Agent)، وBody في حالة POST. اهتمامنا: أي بيانات نُرسلها قد تعود إلينا في الـ Response.

⑤ 📚 XSS Encyclopedia — الأنواع السبعة
اضغط على أي نوع لقراءة الشرح التفصيلي والـ Payload والدفاع
🔄
Reflected XSS
ينعكس في الـ Response مباشرة
كيف يحدث: المدخل يعود في الـ response بدون تخزين. يحتاج ضحية تضغط على رابط مُعدّ.

Payload: <img src=x onerror=alert(1)>

الدفاع: HTML encode كل output، CSP headers.
💾
Stored XSS
مخزّن في DB — يؤثر على كل زائر
كيف يحدث: الـ payload يُخزَّن في قاعدة البيانات ويُعرض لكل مستخدم. أخطر أنواع XSS.

مثال: Bio أو Comment يحتوي <script>...</script>

الدفاع: Sanitize على الإدخال والإخراج معاً.
🌳
DOM-Based XSS
في JavaScript — لا يمر بالسيرفر
كيف يحدث: JavaScript يقرأ بيانات من Source (location.hash) ويضعها في Sink (innerHTML).

Payload: #<img src=x onerror=alert(1)>

الدفاع: textContent بدلاً من innerHTML.
👁️‍🗨️
Blind XSS
تُنفَّذ في مكان لا تراه
كيف يحدث: Payload يُخزَّن في نظام داخلي (logging, admin panel) وينفّذ حين يراه أحد الموظفين.

Tool: XSS Hunter / Burp Collaborator

الدفاع: Sanitize في كل مكان — حتى الـ Admin panels.
🧬
Mutation XSS (mXSS)
المتصفح نفسه يُعدّل الـ HTML
كيف يحدث: Sanitizer يُنظّف الـ HTML، لكن المتصفح يُعيد تحليله بشكل مختلف يؤدي لـ XSS.

تأثر: DOMPurify قبل patch معين

الدفاع: استخدام Trusted Types + أحدث إصدار من DOMPurify.
🔃
Self XSS
يؤثر على المستخدم نفسه فقط
كيف يحدث: الـ payload لا يُرسَل للسيرفر — المستخدم ينفذه بنفسه في Browser Console.

مثال: احتيال "الصق هذا الكود في Console"

الدفاع: تحذير في Developer Console (مثل Facebook).
🌍
Universal XSS (UXSS)
يتجاوز Same-Origin Policy
كيف يحدث: ثغرة في المتصفح نفسه تسمح بتنفيذ JS عبر origins مختلفة — تؤثر على كل المواقع.

مثال: CVE-2021-21193 في Chrome

الدفاع: تحديث المتصفح دائماً. Bug Bounty يدفع $20K+.
