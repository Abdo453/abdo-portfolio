Phase 1 🌐 Web Foundation — HTML/JS Context

فهم كيف يعمل المتصفح قبل أي اختبار

📄

HTML Parsing

المتصفح يحول HTML إلى DOM tree. أي نص يُدرج مباشرة بدون Encoding يمكن أن يُنفَّذ كـ Tag حقيقية.

<div>Hello USERDATA</div>
<div>Hello &lt;script&gt;</div>
🏷️

Execution Contexts

لكل مكان في الصفحة سياق مختلف — نص HTML، داخل Attribute، داخل JavaScript، داخل CSS.

HTML Body Attribute JavaScript CSS
🔒

لماذا يحدث XSS؟

عندما يُعيد التطبيق إدخال المستخدم في الصفحة بدون تشفيره (Encoding) أو التحقق منه (Validation).

💡
الثغرة ليست في المتصفح — هي في الكود الخاطئ للتطبيق.
Phase 2 🔍 XSS Discovery Engine

أين تبحث عن ثغرات XSS؟ — جميع السطوح الهجومية

🔎 Search Bar

حقول البحث في الموقع تُعيد الـ query في الصفحة.

كيف تختبر: أدخل <script>console.log(1)</script> وراقب الانعكاس في source.
👤 User Profile

الاسم، البايو، الرابط — يُعرض للمشرفين والمستخدمين.

كيف تختبر: خصائص الـ name و bio أكثر سطوح Stored XSS شيوعاً.
💬 Comments

قسم التعليقات قد يُخزّن payload وتُنفّذ لكل زائر.

Stored XSS مثالي: كود ينفّذ على كل من يرى التعليق.
✉️ Messages

رسائل داخلية — غالباً بدون HTML encoding.

هدف رئيسي: رسالة تستهدف مشرف النظام بـ Stored XSS.
📎 File Upload

اسم الملف أو بيانات SVG تُعرض مباشرة.

SVG Trick: ملفات SVG يمكن أن تحتوي على JavaScript نشط.
🔌 API Responses

واجهات API التي تُعيد HTML بدلاً من JSON.

راقب: Content-Type header ومدى انعكاس الـ input في response.

Phase 6 🌊 DOM XSS Deep Dive

Sources → Sinks: كيف يتدفق الكود الخطير؟

📥 Sources (المصادر)

مصادر البيانات التي يتحكم فيها المهاجم:

location.href — URL الصفحة الكاملة
location.hash — قسم الـ # في الـ URL
location.search — Query string
document.referrer — الصفحة السابقة
postMessage() — رسائل بين الـ origins

📤 Sinks (المصبّات)

أماكن تنفيذ الكود الخطير:

innerHTML — إدراج HTML مباشرة
eval() — تنفيذ string كـ code
document.write() — كتابة في الصفحة
setTimeout(string) — تأخير تنفيذ كود
location.href = ... — إعادة توجيه

🛡️ Secure Coding — قبل وبعد

الفرق بين الكود المصاب والكود الآمن

❌ Vulnerable Code
// مشكلة: innerHTML
const q = location.hash.slice(1);
results.innerHTML = "نتائج: " + q;
✅ Secure Code
// آمن: textContent فقط
const q = location.hash.slice(1);
results.textContent = "نتائج: " + q;
