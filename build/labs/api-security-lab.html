Theory / النظرية

AR

المستوى 1: تجاوز طرق طلب HTTP (Method Overriding). في الشبكات المقيدة، يتم حظر طرق الطلب مثل DELETE أو PUT على الجدران النارية. لتجاوز ذلك، تسمح بعض الأطر البرمجية بإرسال طلب POST مع ترويسة X-HTTP-Method-Override: DELETE ليعامل الخادم الطلب كحذف.

EN

Level 1: HTTP Method Overriding. Firewalls often block DELETE or PUT requests. To support clients, APIs support headers like X-HTTP-Method-Override: DELETE to wrap restricted actions inside standard POST requests.

AR

المستوى 2: تخطي مصادقة مسار API. يمر فحص الأمان عبر بوابة ذكية (API Gateway) تفحص عنوان المسار. إذا تم حظر /api/v1/admin، قد ننجح في إرسال طلب يحتوي على تخطي مسار نسبي مثل /api/v1/auth/../admin/users ليتخطى فحص البوابة ولكن يعالجه الخادم الداخلي للوصول للبيانات.

EN

Level 2: API Path Authentication Bypass. Routers check access controls against simple URL matches. Attackers utilize path traversal characters (like /api/v1/auth/../admin/users or /api/v1/*/admin) to deceive filters while reaching admin targets.

AR

المستوى 3: التخصيص الجماعي التراكمي (API Mass Assignment). عند إنشاء حساب جديد عبر POST /api/v1/register، يتم تحويل الحقول تلقائياً للمخطط البرمجي (Model). استغل ذلك بإضافة خصائص مثل "user": { "role": "admin" } لتسجيل الحساب كمسؤول.

EN

Level 3: API Mass Assignment. API developers map raw request data directly to entity fields. Inject nested parameters (like "user": {"is_admin": true} or "role": "admin") inside register payloads to escalate profile rights.
