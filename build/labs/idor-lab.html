EN

Simulated Insecure Direct Object Reference (IDOR) lab. Manipulate user identifiers and request parameters to access other users' data and elevate your account permissions.

Level: 1/3 Flags: 0/3 Progress: 0%
https://api.hunteros.com
[*] IDOR API Simulator Ready. Level 1: View administrative profile details.
Your account ID is 2048. Try inspecting the parameters.

Theory / النظرية

AR

المستوى 1: التلاعب بمعرفات عناوين URL. الواجهة الخلفية لا تتحقق مما إذا كان المستخدم المصادق عليه يمتلك حق الوصول إلى المعرف المستهدف. غيّر id=2048 إلى 1001 لقراءة بيانات المسؤول.

EN

Level 1: URL Parameter Tampering. The backend does not verify if the authenticated user owns the requested resource ID. Change id=2048 to 1001 to view the admin profile.

AR

المستوى 2: التلاعب بمعرفات API JSON. عند تحديث البريد الإلكتروني للمستخدم، يرسل المتصفح طلب PUT بجسم JSON يحتوي على معرف المستخدم. غير "user_id": 2048 إلى 1001 لتعديل البريد الإلكتروني للمسؤول واختراق حسابه.

EN

Level 2: API JSON ID Manipulation. When updating user profile, a PUT request is sent with a JSON body. Change "user_id": 2048 to 1001 inside the request body to alter the admin email address.

AR

المستوى 3: ثغرة التخصيص الجماعي (Mass Assignment). عند إرسال طلب تحديث البريد، يمكنك حقن متغيرات إضافية لا تعرضها الواجهة الرسومية ولكن يقبلها النموذج الخلفي. جرب حقن "role": "admin" أو "is_admin": true في جسم طلب PUT الخاص بك لترقية حسابك.

EN

Level 3: Mass Assignment. Backends sometimes automatically map JSON parameters directly to DB objects. Try injecting hidden parameters like "role": "admin" or "is_admin": true to elevate your account privileges.
