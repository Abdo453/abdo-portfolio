مختبر تزوير وتعديل JWT

Interactive JWT Forgery & Exploitation Lab — 3 Levels

HIGH OWASP A01
AR

محاكي ثغرات JWT. تلاعب برأس التوكن (Header) وحمولته (Payload) وتوقيعه لتسجيل الدخول كمسؤول (admin) والحصول على الـ Flags.

EN

Simulated JWT exploitation lab. Manipulate the token header, payload, and signature to authenticate as "admin" and retrieve the flags.

Level: 1/3 Flags: 0/3 Progress: 0%
[*] JWT Lab initialized.
Edit the header/payload above, it will automatically encode. Target: Elevate role to 'admin' and bypass validations.
[*] Hint L1: Change "alg" to "none" (or "NONE") in Header, and set "user" to "admin" or "role" to "admin" in Payload. Remove signature.

Theory / النظرية

AR

المستوى 1: خوارزمية (none). تسمح بعض المكتبات الضعيفة بقبول التوكنات الخالية من التوقيع إذا تم تعيين "alg": "none". قم بتعديل الرأس والحمولة ثم احذف قيمة التوقيع تماماً.

EN

Level 1: (none) Algorithm. Some vulnerable libraries accept tokens with no signature if "alg": "none" is set. Alter the header and payload, and empty the signature input.

AR

المستوى 2: كسر مفتاح HMAC الضعيف. المفتاح السري المستخدم هو كلمة شائعة. خمن المفتاح (مثلاً secret أو admin أو supersecret) ووقّع به توكن الـ admin.

EN

Level 2: Cracking Weak HMAC key. The signing key is a common dictionary word. Guess/Bruteforce the key (e.g. secret or 123456) to forge your admin token signature.

AR

المستوى 3: حقن JKU (JWK Set URL). تلاعب بمتغير jku في الرأس ليشير إلى رابط خارجي موثوق أو متخطي الفحص (مثل http://localhost/keys.json) لتجاوز التحقق من التوقيع.

EN

Level 3: JKU (JWK Set URL) Injection. Manipulate the jku header parameter to point to an attacker-controlled or local URL (e.g., http://localhost/keys.json) to control the verification key.
