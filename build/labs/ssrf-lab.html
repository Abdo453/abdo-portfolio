مختبر ثغرات SSRF

Interactive Server-Side Request Forgery Lab — 3 Levels

HIGH OWASP A10:2021
AR

محاكي ثغرات SSRF. تلاعب بمعاملات جلب الملفات والصور لإجبار السيرفر على الاتصال بالخوادم الداخلية وشبكات السحابية لقراءة البيانات الحساسة.

EN

Simulated Server-Side Request Forgery (SSRF) lab. Manipulate URL fetchers and proxy utilities to coerce the server to interact with internal networks, private portals, and cloud metadata endpoints.

Level: 1/3 Flags: 0/3 Progress: 0%
GET /proxy.php?url=
[*] SSRF Proxy simulator ready. Level 1: fetch local server assets.
Try changing the URL parameters to point to internal networks or localhost.
[*] Hint L1: Try fetching the internal admin portal at http://localhost:8080/admin or http://127.0.0.1:8080/admin.

Theory / النظرية

AR

المستوى 1: الوصول إلى الخدمات الداخلية. الخوادم تمتلك أحياناً لوحات تحكم داخلية غير مصادق عليها متاحة فقط للمضيف المحلي (localhost). تلاعب بمعامل url ليطلب السيرفر صفحة http://localhost:8080/admin بالنيابة عنك.

EN

Level 1: Fetching internal hosts. Apps often trust internal network queries coming from localhost. Force the server to query http://localhost:8080/admin using the url parameter.

AR

المستوى 2: واجهة بيانات الميتا السحابية (Cloud Metadata). الخوادم السحابية (AWS, GCP, DigitalOcean) توفر خوادم بيانات وصفية عند عناوين IP ثابتة (مثل AWS http://169.254.169.254/latest/meta-data/). استغل الـ SSRF لتسريب مفاتيح وسرية الوصول السحابية (IAM Credentials).

EN

Level 2: Cloud Instance Metadata API. Cloud infrastructure (AWS, Azure, DigitalOcean) exposes internal metadata servers on fixed link-local IP 169.254.169.254. Query this endpoint to read AWS credentials (AWS IAM Keys).

AR

المستوى 3: ثغرة SSRF العمياء (Blind SSRF). لا تظهر مخرجات الطلب في الاستجابة، ولكن يقوم الخادم بإرسال الطلب بالفعل. لمعرفة نجاح الأمر، يرسل المخترق الطلب إلى سيرفر خارجي يراقبه (Collaborator). استخدم نطاق متعاون خارجي (مثل http://collaborator.hunteros.com) لمراقبة التفاعلات.

EN

Level 3: Blind Out-of-band (OAST) SSRF. The server fetches the URL but suppresses the output. Attackers must trigger a connection to an external site they monitor (e.g. http://collaborator.hunteros.com) to register incoming DNS/HTTP lookup events.
