Linux Fundamentals for Security Professionals

TRACK 1 ⏱ ~20 min

1. Introduction / مقدمة

AR

لينكس هو نظام التشغيل الأساسي الذي تعمل عليه أغلب خوادم الويب والبنى التحتية السحابية وأدوات اختبار الاختراق (مثل Kali Linux). لفهم كيفية اختراق أو حماية هذه الأنظمة، يجب أولاً إتقان كيفية التعامل مع سطر الأوامر وفهم بيئة العمل الداخلية للنظام.

EN

Linux is the backbone of most web servers, cloud infrastructures, and penetration testing platforms (e.g. Kali Linux). To understand how to compromise or secure these systems, you must first master the command-line interface and the inner workings of the operating system.

2. Linux Directory Structure / هيكل المجلدات

AR

على عكس ويندوز، لا توجد أقراص مثل C: أو D:. كل شيء في لينكس يبدأ من المجلد الجذري (Root Directory) /:

  • /bin & /sbin: ملفات البرامج والأوامر الأساسية.
  • /etc: ملفات تكوين وإعدادات النظام والخدمات.
  • /home: المجلدات الخاصة بالمستخدمين العاديين.
  • /root: المجلد الخاص بمسؤول النظام (Root User).
  • /var/log: ملفات سجلات النظام والأحداث (Logs).
  • /tmp & /var/tmp: ملفات مؤقتة (تُحذف تلقائياً في الغالب وتكون قابلة للكتابة لأي مستخدم).
EN

Unlike Windows, Linux has no drive letters (like C: or D:). Everything starts from the root directory /:

  • /bin & /sbin: Essential user binaries and system binaries.
  • /etc: System configuration files.
  • /home: User home directories.
  • /root: Superuser (Root) home directory.
  • /var/log: System log files.
  • /tmp & /var/tmp: Temporary directories (often world-writable).

4. Permissions Model / نظام الصلاحيات

AR

كل ملف أو مجلد في لينكس ينتمي إلى مستخدم (Owner) ومجموعة (Group)، ويحتوي على ثلاثة أنواع من الصلاحيات:

  • r (Read): القراءة (تمثل بالقيمة 4).
  • w (Write): الكتابة والتعديل (تمثل بالقيمة 2).
  • x (Execute): التشغيل (تمثل بالقيمة 1).

يتم تمثيل الصلاحيات كأرقام ثلاثية، مثلاً chmod 755 file.sh تعني:

  • 7 (4+2+1): صلاحيات كاملة للمالك (rwx).
  • 5 (4+1): قراءة وتشغيل للمجموعة (r-x).
  • 5 (4+1): قراءة وتشغيل للآخرين (r-x).
EN

Every file and folder in Linux is assigned to an Owner and a Group, with three permission classes:

  • r (Read): represented numerically by 4.
  • w (Write): represented numerically by 2.
  • x (Execute): represented numerically by 1.

Permissions are often set as three octal digits, for example chmod 755 file.sh sets:

  • 7 (4+2+1): Owner has full rights (rwx).
  • 5 (4+1): Group has read & execute (r-x).
  • 5 (4+1): Others have read & execute (r-x).
