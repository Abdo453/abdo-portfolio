🧠 ما هو الـ Footprinting؟

الـ Footprinting هو المرحلة الأولى في أي عملية اختراق، والهدف منها جمع أكبر قدر ممكن من المعلومات عن الهدف قبل أي هجوم فعلي.
القاعدة الذهبية في عالم الاختراق: "كلما عرفت أكثر عن الهدف، كان اختراقك أسهل وأسرع."

⚡ أنواع الـ Footprinting

🕵️ Passive Footprinting

جمع المعلومات بدون الاتصال المباشر بالهدف — الهدف لن يحس بك أبداً!

  • ✅ لا تترك أثراً في Logs الهدف
  • ✅ آمن قانونياً نسبياً
  • ✅ يصلح لفترة طويلة
أدوات: Google Dorks, Shodan, WHOIS, hunter.io, crt.sh

🚨 Active Footprinting

الاتصال المباشر بالهدف — أسرع في النتائج لكن الهدف ممكن يشوفك!

  • ⚠️ يترك أثراً في Logs الهدف
  • ⚠️ أسرع في جمع بيانات دقيقة
  • ⚠️ لازم تكون حذر
أدوات: Ping, Traceroute, dnsrecon, dirb, wfuzz

🕵️ أدوات الـ Passive Footprinting بالتفصيل

🔎 1. Google Dorks (Google Hacking) مجاني 100%

استخدام مشغلات بحث متقدمة في Google لإيجاد معلومات حساسة مش مقصود نشرها.

site:example.com # كل صفحات الموقع
intitle:"index of" # فولدرات مكشوفة 🔥
filetype:pdf site:example.com # ملفات PDF
inurl:admin site:example.com # لوحات تحكم
ext:sql | ext:log | ext:env # ملفات حساسة
cache:example.com # نسخة مخزنة
💡 موقع يجمع كل الـ Dorks: GHDB (Google Hacking Database) على exploit-db.com
🌐 2. Shodan — "محرك بحث الأجهزة" الأخطر 😈

Shodan مش بيبحث في المواقع، بيبحث في الأجهزة المتصلة بالإنترنت — كاميرات، سيرفرات، راوترات، أجهزة IoT.

apache country:EG # سيرفرات Apache في مصر
port:3389 country:SA # RDP مفتوح في السعودية
shodan search "default password" # أجهزة بكلمة مرور افتراضية!
⚠️ تحذير: استخدام Shodan على هدف بدون إذن مخالف للقانون.
🛰️ WHOIS

بيجيب معلومات تسجيل الدومين (المالك، DNS، تاريخ الانتهاء).

whois example.com
🔐 crt.sh

بيجيب Subdomains مخفية من SSL Certificates — مجاني ومخيف جداً!

https://crt.sh/?q=%.example.com
📧 hunter.io

بيجيب إيميلات الشركة — مهم جداً في الـ Social Engineering والـ Phishing.

https://hunter.io/domain/example.com
⚙️ theHarvester + Sublist3r — أدوات OSINT الكلاسيكية

theHarvester: يستخرج إيميلات، Subdomains، وعناوين IP من مصادر متعددة.

theHarvester -d example.com -b google
theHarvester -d example.com -b google,linkedin,bing -l 500

-d الدومين | -b مصدر البيانات | -l حد النتائج

Sublist3r: يجمع Subdomains من Brute Force + مصادر OSINT.

sublist3r -d example.com
sublist3r -d example.com -v

-v verbose — يعرض النتائج لحظة بلحظة

🚨 أدوات الـ Active Footprinting بالتفصيل

📡 Ping — هل الجهاز شغال؟
ping example.com

بيبعت ICMP Packets ويشوف لو السيرفر رد — يجيب الـ IP و Response Time.

🧭 Traceroute — المسار كامل
traceroute example.com

بيكشف كل الـ Hops (Routers) بين جهازك والهدف — مفيد لفهم الـ Network topology.

🌐 dnsrecon — تحليل DNS
dnsrecon -d example.com

يجيب كل DNS Records: A, MX, NS, TXT، ويحاول Zone Transfer.

🕷️ Photon — Crawler متطور
photon -u http://example.com

يعمل Crawling عميق ويجيب URLs مخفية، صور، JS files، إيميلات.

🚪 Knockpy — Subdomain Bruteforce
knockpy example.com

يجرب آلاف الأسماء ليجد Subdomains مخفية (dev, staging, api, mail...).

⚡ Wfuzz — Directory Fuzzing
wfuzz -c -w wordlist.txt http://example.com/FUZZ

يجرب كلمات من قاموس مكان FUZZ — يجد صفحات مخفية وAPIs.

🔥 فرق مهم: Subdomain vs Subdirectory

🌐
Subdomain
test.example.com
api.example.com

خادم مختلف تماماً — قد يكون بيئة تطوير أو API منفصل.

📁
Subdirectory
example.com/admin
example.com/backup

مسار داخل نفس الخادم — قد يحتوي على صفحات حساسة مخفية.
