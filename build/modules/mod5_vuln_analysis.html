Module 05 CVE Research Pre-Exploitation

🧠 ما هي الثغرة (Vulnerability)؟

الثغرة هي نقطة ضعف في النظام يمكن للمهاجم استغلالها للوصول أو التحكم أو سرقة البيانات.

⚙️
Misconfiguration

إعدادات خاطئة (مثل: FTP مسموح anonymous)

🐛
Software Bug

خطأ برمجي في التطبيق أو نظام التشغيل

📦
Outdated Software

إصدار قديم به ثغرات معروفة ومنشورة

👤
Human Error

خطأ بشري كنسيان كلمة مرور افتراضية

⚠️ فرق جوهري مهم جداً:

🔍
Vulnerability Analysis

اكتشاف الثغرة ومعرفة وجودها
"الثغرة موجودة!"

💥
Exploitation

استغلال الثغرة فعلياً للاختراق
"دخلنا!"

📚 CVE — قاعدة بيانات الثغرات العالمية

CVE اختصار لـ Common Vulnerabilities and Exposures — هو نظام ترقيم عالمي موحد لكل ثغرة أمنية مكتشفة. كل ثغرة بتاخد رقم فريد: CVE-[السنة]-[الرقم التسلسلي]

🏛️ NIST — nvd.nist.gov

قاعدة البيانات الوطنية الأمريكية. توضح التفاصيل الكاملة لكل ثغرة مع نقاط خطورتها (CVSS Score).

مثال: CVE-2021-44228 → Log4Shell (CVSS: 10.0)
🔬 MITRE — cve.mitre.org

المصدر الأصلي والأساسي لأرقام الـ CVE. كل ثغرة جديدة بتبدأ تسجيلها هنا.

مثال: CVE-2021-41773 → Apache Path Traversal + RCE
💣 Exploit-DB — exploit-db.com

المستودع الأكبر للـ Exploits الجاهزة. كل ثغرة فيها كود جاهز للتنفيذ!

searchsploit apache 2.4 # محلياً في Kali

🤖 أدوات الفحص الآلي (Automated Scanners)

Nessus 🏆

أشهر أداة فحص ثغرات في العالم. تُصدر تقارير تفصيلية مع نقاط الـ CVSS وطرق الإصلاح.

  • ✅ يفحص الشبكات والتطبيقات
  • ✅ يربط النتائج بـ CVEs
  • ✅ يعطيك Severity Ranking
  • ⚠️ مدفوع (نسخة مجانية محدودة)
OpenVAS

البديل المجاني ومفتوح المصدر لـ Nessus. جزء من مشروع Greenbone Vulnerability Manager.

  • ✅ مجاني 100%
  • ✅ يُحدَّث بانتظام
  • ✅ واجهة ويب سهلة
  • ⚠️ أبطأ قليلاً من Nessus
⚠️ انتبه من False Positives! الأدوات الآلية مش دايماً صح — لازم تتحقق يدوياً من النتائج المهمة.

🧠 أهم أنواع الثغرات المعروفة

💣 RCE
Remote Code Execution — تشغيل كود على الخادم مباشرة. الأخطر!
📁 LFI / RFI
Local/Remote File Inclusion — قراءة ملفات الخادم الداخلية.
💉 SQL Injection
حقن أوامر في قاعدة البيانات — سرقة أو حذف البيانات.
🌐 XSS
Cross-Site Scripting — حقن سكريبتات في المتصفح.
📊 Buffer Overflow
إرسال بيانات أكثر من المتوقع لتحطيم الذاكرة والسيطرة على الـ EIP.
⚙️ Misconfiguration
إعدادات خاطئة مثل Anonymous FTP، Default Passwords.

🏆 Workflow عملي كامل — من الصفر للثغرة

1
Scan للخدمات والإصدارات:
nmap -sS -sV -O 192.168.1.1
2
احفظ الإصدارات:

مثال: FTP → vsftpd 2.3.4 | HTTP → Apache 2.4.49

3
ابحث عن الـ Exploits:
searchsploit vsftpd 2.3.4
searchsploit apache 2.4.49
4
تحقق من الثغرة:

ابحث في Google، exploit-db.com، وGitHub للتأكد وفهم الثغرة.

5
جهز للـ Exploitation:

استخدم Metasploit أو كود الـ Exploit يدوياً في المرحلة القادمة! 🔥

💥 مثال عملي حقيقي (Metasploitable):
nmap -sV 192.168.1.5
PORT STATE SERVICE VERSION
21/tcp open ftp vsftpd 2.3.4
searchsploit vsftpd 2.3.4
