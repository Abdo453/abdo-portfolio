14. Active Directory Security

فهم آليات عمل الدليل النشط (Active Directory) وبروتوكولات المصادقة مثل Kerberos و NTLM، وطرق تقييمها أمنياً والدفاع عنها.

[Theory] نظرية العمل والمفاهيم

يعتبر Active Directory (AD) العصب الرئيسي لإدارة الهويات والصلاحيات في بيئات Windows المؤسسية. يعتمد بشكل أساسي على بروتوكولات مثل Kerberos لإصدار تذاكر المصادقة (TGT و TGS) و NTLM للتوافقية مع الأنظمة القديمة. تشمل المفاهيم الأمنية تحليل مسارات الهجوم (Attack Paths) مثل تمرير التجزئة (Pass The Hash) ومخاطر بروتوكولات البث مثل LLMNR، وكيفية تأمين الشبكة ضدها.

[Attack Flow] مسار المصادقة وبروتوكول Kerberos


sequenceDiagram
    participant Client
    participant KDC as Domain Controller

    Client->>Service: 5. AP-REQ (Application Request with Ticket)
    Service->>Client: 6. AP-REP (Access Granted)

[Manual Testing] الفحص اليدوي والتدقيق

لتحليل أمان بيئة AD يدوياً، يتم التركيز على استخراج البيانات المتاحة للمستخدمين العاديين، مثل فحص إعدادات SMB Signing لتحديد الأنظمة المعرضة لهجمات NTLM Relay. بالإضافة إلى تحليل حسابات الخدمات (Service Principal Names) باستخدام استعلامات LDAP القياسية لفهم سطح الهجوم المتاح وتحديد الحسابات التي قد تكون عرضة لهجمات Kerberoasting.

[Automation] الأتمتة وتقييم المخاطر

تستخدم أدوات مثل BloodHound لجمع بيانات الدليل النشط أوتوماتيكياً وتخزينها في قاعدة بيانات Neo4j. يتيح ذلك للمدققين والفرق الدفاعية رسم خرائط بصرية للعلاقات والصلاحيات المفرطة وتحديد مسارات تصعيد الصلاحيات المعقدة لتأمينها قبل استغلالها.

[Tools] أدوات التدقيق الأساسية

# استعلام عن مستخدمي النطاق لتدقيق الصلاحيات (PowerShell)
Get-ADUser -Filter * -Properties * | Select-Object Name, SID, PasswordLastSet

# فحص توافر توقيع SMB (SMB Signing) لمنع هجمات Relay
Get-SmbClientConfiguration | Select-Object RequireSecuritySignature

# الاستعلام عن إعدادات نهج المجموعة (Group Policy) الخاصة بـ LLMNR
Get-ItemProperty -Path "HKLM:\Software\Policies\Microsoft\Windows NT\DNSClient" -Name "EnableMulticast" -ErrorAction SilentlyContinue

[Cheat Sheet] المرجع السريع

المصطلح (Term)الوصف والتوضيح
KDC (Key Distribution Center)مركز توزيع المفاتيح، وعادة ما يكون الـ Domain Controller نفسه.
TGT (Ticket Granting Ticket)التذكرة الأساسية التي تثبت هوية المستخدم للنظام.
SPN (Service Principal Name)المعرف الفريد لخدمة معينة تعمل على حساب داخل الدليل النشط.
LLMNRبروتوكول بث قديم لحل الأسماء، يشكل خطراً أمنياً ويُنصح بتعطيله لمنع التسمم (Poisoning).
