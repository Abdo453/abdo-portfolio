🔥 Active Reconnaissance & Subdomain Enumeration

مرحلة الاستطلاع النشط واكتشاف النطاقات الفرعية المتقدمة لتوسيع سطح الهجوم

1.1 Overview

الاستطلاع النشط (Active Reconnaissance) واكتشاف النطاقات الفرعية (Subdomain Enumeration) يتضمن التفاعل المباشر مع البنية التحتية لنظام أسماء النطاقات (DNS) الخاص بالهدف. يهدف هذا التفاعل المباشر إلى اكتشاف النطاقات الفرعية غير المفهرسة علنًا (Publicly Indexed). هذه الخطوة حاسمة جداً لأنها تقوم بتوسيع سطح الهجوم (Attack Surface) ليتجاوز ما يتم اكتشافه خلال عملية الاستطلاع السلبي (Passive Reconnaissance)، مما يفتح أبواباً ونقاط دخول جديدة غير معروفة للعلن قد تحتوي على ثغرات أمنية خطيرة.

1.2 DNS Brute Forcing

تقنية الـ DNS Brute Forcing تعتمد على تخمين النطاقات الفرعية باستخدام قوائم كلمات (Wordlists) ضخمة تحتوي على أسماء شائعة، وتمريرها لأدوات تقوم بإرسال استعلامات للمخدمات لتأكيد وجود هذه النطاقات.

Gobuster DNS Mode 
gobuster dns -d target.com -w wordlist.txt -t 50 --delay 100ms -o gobuster_results.txt

شرح: أداة Gobuster قوية وسريعة جداً. في هذا الأمر نستخدم وضع `dns` لتخمين النطاقات للهدف `target.com`، باستخدام قائمة كلمات مع تحديد `50` مسار متوازي (Threads)، مع وضع تأخير `100ms` لتفادي الحظر (Rate Limiting)، وتصدير النتائج إلى ملف.

FFuF for Subdomain Fuzzing 
ffuf -w subdomains.txt -u https://FUZZ.target.com -mc 200,301,302 -t 100 -rate 50 -o fuf_results.json

شرح: أداة FFuF تُستخدم هنا لعمل Fuzzing لمكان النطاق الفرعي عن طريق استبدال الكلمة `FUZZ` بكلمات من القائمة. تم تحديد الحالات المقبولة للرد (Status Codes) بـ 200، 301، 302 لفلترة النتائج، مع ضبط معدل الطلبات (Rate limit) بـ 50 طلب بالثانية لتجنب الجدار الناري (WAF).

Puredns (MassDNS Wrapper) 
puredns -r resolvers.txt bruteforce wordlist.txt target.com

شرح: أداة Puredns تعتبر واجهة متقدمة لأداة MassDNS. وظيفتها تسريع عملية التخمين مع فلترة النطاقات التي تحتوي على Wildcard (أي التي تعيد إجابة إيجابية دائماً لتضليل الفاحص)، وتستخدم ملف `resolvers.txt` للبحث عبر عدة خوادم DNS في نفس الوقت.

MassDNS (High-speed resolver) 
massdns -r lists/resolvers.txt -t A domains.txt -o results.txt

شرح: MassDNS هي أداة سريعة للغاية تعمل كمحلل DNS عالي الكفاءة، تستطيع التعامل مع آلاف الاستعلامات بالثانية. الأمر أعلاه يطلب سجلات من نوع A (IPv4) للقائمة المقدمة.

ShuffleDNS (ProjectDiscovery) 
# Subdomain resolving
shuffledns -d example.com -list subdomains.txt -r resolvers.txt -mode resolve
# Subdomain bruteforcing
shuffledns -d hackerone.com -w wordlist.txt -r resolvers.txt -mode bruteforce

شرح: ShuffleDNS من تطوير ProjectDiscovery. توفر وضعين: إما للتحقق من قائمة نطاقات فرعية (Mode Resolve) لمعرفة أيهم يعمل فعلياً، أو وضع التخمين المباشر (Mode Bruteforce) للبحث عن أسماء جديدة.

1.3 Permutation & Alteration Techniques

تقنيات التبديل والتغيير (Permutations & Alterations) هي طرق ذكية لاكتشاف نطاقات فرعية مخفية من خلال أخذ أسماء النطاقات المكتشفة سابقاً ودمجها مع كلمات أو أرقام إضافية (مثل dev, prod, staging).

Altdns 
python altdns.py -i input_domains.txt -o ./output/path -w altdns/words.txt

شرح: أداة Altdns تقوم بدمج قائمة من الكلمات مع النطاقات المعطاة لتوليد تراكيب جديدة واختبارها (مثل إضافة كلمة test قبل أو بعد النطاق الفرعي).

Gotator (Advanced permutation) 
gotator -sub domains.txt -perm permutations.txt -depth 1 -numbers 10 -mindup -adv -md

شرح: Gotator أداة متقدمة تقوم بتوليد تبديلات عميقة ومعقدة باستخدام الكلمات والأرقام (حتى 10 أرقام) لضمان تغطية كل الاحتمالات الممكنة لتسمية الخوادم ضمن البنية التحتية.

ShuffleDNS for permutations 
shuffledns -l all.txt -r resolvers.txt -o permuted.txt
Permutation Examples:
إذا كان لدينا النطاق api.target.com، ستقوم هذه الأدوات بإنشاء وتحليل احتمالات مثل:
- api1.target.com, api2.target.com
- api-dev.target.com, dev-api.target.com
- api-staging.target.com, api-prod.target.com

1.4 DNS Zone Transfer (AXFR)

نقل نطاق الـ DNS (AXFR) هو نوع من الاستعلامات يُستخدم لنسخ سجلات النطاق (Zone File) من خادم إلى آخر. إذا كان الخادم مسيئ الإعداد (Misconfigured) ويسمح بالنقل لأي شخص، يمكن للمهاجم سحب جميع أسماء النطاقات والسجلات المتعلقة بالهدف دفعة واحدة!

AXFR Testing Tools 
# Attempt zone transfer
dig axfr @ns1.target.com target.com

# Using DNSRecon
dnsrecon -d target.com -t axfr

# Using fierce
fierce --domain target.com

شرح:
- dig: الأداة الكلاسيكية للطلب المباشر (AXFR) من خادم الأسماء (ns1).
- DNSRecon / fierce: أدوات متخصصة في جمع معلومات الـ DNS تقوم تلقائياً باكتشاف خوادم الأسماء (Nameservers) ومحاولة استغلال ثغرة الـ Zone Transfer عليها.

1.5 Advanced DNS Enumeration

تقنيات الاستطلاع المتقدمة لا تقتصر على تخمين الأسماء فقط، بل تحليل ردود الخوادم بدقة للبحث عن أدلة لبيئات مخفية أو ثغرات الاستيلاء (Subdomain Takeover).

Empty Non-Terminal (ENT) Detection:
ركز على استجابات الـ DNS Codes: التفرقة بين NOERROR (لا يوجد خطأ) و NXDOMAIN (النطاق غير موجود).
إذا تلقيت NODATA (رد بدون بيانات لكن NOERROR)، فهذا يدل بشكل قاطع على وجود نطاق فرعي مخفي أو عقدة (Node) تفرع منها نطاقات أخرى!
- الأدوات التي تدعم هذا التحليل: dnsx و massdns.
CNAME Analysis 
# Check CNAME records for takeover indicators
dig CNAME suspicious-subdomain.target.com

# Check for dangling CNAMEs
host suspicious-subdomain.target.com

شرح: تحليل سجلات الـ CNAME يهدف إلى إيجاد النطاقات الفرعية التي تشير إلى خدمات سحابية (مثل AWS, GitHub Pages, Heroku) تم حذفها أو لم يعد العميل يمتلكها (Dangling CNAMEs)، مما يتيح للمهاجم حجز تلك الخدمة والسيطرة على النطاق الفرعي للهدف، وهو ما يعرف بـ Subdomain Takeover.

SPF/TXT Record Analysis 
dig TXT target.com
dig MX target.com
dig NS target.com

شرح: استخراج سجلات TXT و MX و NS مفيد للغاية، لأنها غالباً ما تحتوي على معلومات عن خدمات الطرف الثالث (Third-party services) التي تستخدمها الشركة، أو بيانات حساسة داخل سجلات نظام سياسة مرسل البريد (SPF).

1.6 Certificate Transparency Logs

سجلات شفافية الشهادات (Certificate Transparency Logs) هي قواعد بيانات عامة يتم فيها تسجيل كل شهادات الأمان (SSL/TLS) المُصدرة لأي موقع. يمكننا البحث في هذه السجلات للعثور على النطاقات الفرعية الداخلية والخاصة التي أُصدرت لها شهادات!

crt.sh via command line 
curl -s "https://crt.sh/?q=%.target.com&output=json" | jq -r '.[].name_value' | sed 's/\*\.//g' | sort -u > certificate_subdomains.txt

شرح: هذا الأمر يقوم بالاتصال بموقع crt.sh عن طريق curl للحصول على كل النطاقات التابعة لـ target.com بتنسيق JSON. ثم يمرر النتائج لأداة jq لاستخراج الأسماء، وأداة sed لحذف علامة النجمة (Wildcard *.)، وأخيراً استخدام sort -u لترتيب النتائج وحذف التكرار، وحفظها في ملف نصي.

1.7 Complete Workflow

مسار العمل المتكامل (Complete Workflow) يجمع بين تقنيات الاستطلاع السلبي، والتبديل (Permutation)، والتحليل النشط، ثم التحقق من الأنظمة الحية للحصول على سطح هجوم دقيق وشامل.

Subdomain Enumeration Pipeline 
# Step 1: Passive collection
subfinder -d target.com -all -recursive -silent -o subs.txt
amass enum -passive -d target.com -o amass.txt
assetfinder --subs-only target.com > assetfinder.txt

# Step 2: Combine and deduplicate
cat subs.txt amass.txt assetfinder.txt | sort -u > all_passive.txt

# Step 3: Generate permutations
altdns -i all_passive.txt -w permutationWordlist.txt -o permuted.txt

# Step 4: Resolve everything
shuffledns -d target.com -list permuted.txt -r resolvers.txt -o resolved.txt

# Step 5: Check live hosts
httpx -l resolved.txt -status-code -title -tech-detect -o live_hosts.txt

شرح خطوات الـ Workflow:
1. الجمع السلبي (Passive collection): استخدام أدوات subfinder, amass, assetfinder للبحث بهدوء دون تنبيه الهدف وتجميع البيانات من مصادر مفتوحة.
2. الدمج وإزالة التكرار (Deduplicate): وضع النتائج من جميع الأدوات في ملف واحد وتنظيفه عبر sort -u.
3. التوليد والتبديل (Permutations): استخدام altdns لخلق احتمالات وأسماء نطاقات جديدة بناءً على الكلمات المكتشفة.
4. التحقق والفحص (Resolve): استخدام shuffledns للتأكد من أن هذه النطاقات تشير إلى خوادم حقيقية ولديها IP فعّال.
5. اكتشاف المضيفين الأحياء (Live hosts): تمرير القائمة النهائية لأداة httpx للاتصال بها واستخراج حالة الاستجابة (Status Code)، عنوان الصفحة (Title)، والتقنيات المستخدمة (Tech Detect).

1.8 Best Practices & Anti-Detection

أفضل الممارسات وتقنيات مكافحة الكشف (Anti-Detection) تضمن استمرارية الفحص دون التعرض للحظر من الجدران النارية (Firewalls/WAFs) وللبقاء غير مرئي قدر الإمكان.

  • Rate limiting: أضف تأخيراً بين الطلبات (مثل --delay 100ms) لمنع أنظمة كشف التسلل (IDS) من حظر الـ IP الخاص بك.
  • Rotate DNS resolvers: استخدم قوائم متعددة من خوادم الـ DNS (Resolvers) لكي لا تركز ضغط الاستعلامات على خادم واحد فيتم حظرك.
  • Use VPS: قم بتشغيل عمليات الاستطلاع النشط من خلال خوادم سحابية افتراضية (VPS) لضمان سرعة الإنترنت وإخفاء هويتك الفعلية.
  • Respect scope: راجع دائماً نطاق العمل (Scope) الخاص ببرامج صيد الثغرات (Bug Bounty) لتجنب فحص أو مهاجمة أنظمة خارج النطاق المصرح به قانونياً.
  • Wildcard detection: استخدم أدوات مثل puredns أو shuffledns لتصفية (Filter) الردود الكاذبة من النطاقات التي تحتوي على Wildcard (إعداد يقوم بتوجيه أي اسم فرعي وهمي إلى صفحة معينة).

Proxies & Scanning Toolchains

استكمالاً لمسار الاستطلاع والفحص، يُعتبر إعداد وكيل اعتراض الطلبات (Intercepting Proxy) واستخدام ماسحات الويب الآلية خطوة حاسمة لفهم التفاعل مع التطبيق واكتشاف الثغرات العميقة.

Burp Suite Proxy Configuration:
1. قم بضبط الوكيل (Proxy) في المتصفح على 127.0.0.1:8080.
2. قم بتثبيت شهادة الـ CA (Certificate Authority) في المتصفح لتتمكن من اعتراض زيارات الـ HTTPS.
3. فعل خيار الاعتراض (Enable Interception) للتحكم بالطلبات وتعديلها قبل وصولها للخادم.
4. تصفح تطبيق الهدف واستكشف أقسامه لتسجيل الحركة والمنافذ.

Burp Suite Key Features Usage

Feature الوظيفة الأساسية والاستخدام
Proxy لاعتراض وتعديل الطلبات والردود بين المتصفح والخادم بشكل حي.
Repeater لإعادة إرسال طلبات محددة وتعديلها يدوياً لتجربة استغلال الثغرات بشكل دقيق.
Intruder لأتمتة الهجمات، مثل الـ Fuzzing وتخمين كلمات المرور عبر إدخال قوائم كلمات في أماكن محددة بالطلب.
Scanner (Pro) في النسخة المدفوعة، يقوم بالمسح الآلي لاكتشاف الثغرات الشائعة بشكل شامل.
Sequencer لتحليل مدى عشوائية الـ Session Tokens (الرموز المميزة للجلسات) لاكتشاف الضعف فيها.
Decoder أداة مدمجة لتشفير وفك تشفير البيانات بالصيغ المختلفة (مثل Base64, URL Encoding).
Comparer لمقارنة الاختلافات الدقيقة بين طلبين أو استجابتين (مفيد جداً في ثغرات الـ Blind SQLi).

Burp Suite Extensions to Install

  • Autorize: لاختبار ثغرات الصلاحيات (IDOR/Authorization Testing) بشكل آلي.
  • Logger++: لتوفير واجهة متقدمة ومفصلة لتسجيل حركة المرور ومتابعتها.
  • Turbo Intruder: أداة سريعة جداً مبنية على لغة Python لشن هجمات الـ HTTP المكثفة و Race Conditions.
  • Param Miner: لاكتشاف المعاملات (Parameters) المخفية في الطلبات والتي قد تقود لاكتشاف ثغرات غير واضحة.
  • Upload Scanner: للتدقيق واختبار أمان ملفات الرفع (File Uploads) وأنواعها ومحتواها.

4.7 OWASP ZAP Workflow

OWASP ZAP هو بديل قوي ومجاني ومفتوح المصدر للـ Burp Suite، ويأتي بأدوات ماسحة مدمجة قوية لتنفيذ عمليات البحث الآلي عن الثغرات والتفاعل مع الـ APIs.

ZAP Command Line 
# Automated scan
zap-cli quick-scan --self-contained --start-options "-config api.disablekey=true" --spider -r -s xss,sqli https://target.com

# API scan
zap-api-scan.py -t openapi.json -f openapi

شرح:
- Automated scan: أداة zap-cli تُشغل المسح السريع مع عنكبوت (Spider) لاكتشاف الروابط وتوجيه البحث عن ثغرات XSS و SQLi، مع تعطيل مفتاح الـ API للتسهيل.
- API scan: أداة متخصصة لفحص واجهات برمجة التطبيقات (API) من خلال قراءة ملف الـ openapi.json الموصف للخدمة واكتشاف ثغراته آلياً.

4.8 Nikto Web Scanner

Nikto هو ماسح خوادم ويب أسطوري ومشهور جداً، يعمل من خلال سطر الأوامر للكشف عن آلاف الملفات الخطيرة أو القديمة وإعدادات الخادم السيئة.

Nikto Commands 
nikto -host target.com
nikto -host target.com -ssl -port 443
nikto -host target.com -Tuning 1,2,3,4,5,6,7,8,9,0

شرح:
- الفحص الأساسي باستخدام -host يمرر على المنفذ الافتراضي.
- استخدام -ssl -port 443 لفرض الفحص من خلال بروتوكول HTTPS الآمن.
- الخيار -Tuning يُستخدم لتخصيص نوع الهجمات والفحوصات بدقة لتوفير الوقت أو التركيز على تصنيف معين (مثلاً حقن SQL، رفع الملفات، قراءة الملفات المحلية).
