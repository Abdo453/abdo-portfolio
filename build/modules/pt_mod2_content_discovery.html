🔍 2. Content Discovery

استكشاف البنية التحتية للموقع، استخراج المسارات المخفية، وتحليل تاريخ الويب والملفات بشكل معمق (Content Discovery).

2.1 Directory & File Brute Forcing

هذا القسم يركز على استخدام هجمات القوة الغاشمة (Brute Forcing) لتخمين واستكشاف المسارات (Directories) والملفات (Files) المخفية داخل خوادم الويب. هذه العملية ضرورية جداً لكشف صفحات لوحة التحكم، النسخ الاحتياطية (Backups)، والملفات الحساسة التي لم يتم الإعلان عنها للمستخدمين العاديين، مما قد يؤدي لثغرات خطيرة مثل تسريب البيانات.

أداة Feroxbuster (Rust-based, fast):

أداة سريعة جداً ومبنية بلغة Rust، تُستخدم للبحث المتكرر عن المسارات. نحدد الهدف باستخدام -u وقائمة الكلمات (Wordlist) باستخدام -w. الراية -C 404 تخبر الأداة بتجاهل أي استجابة من الخادم تحمل كود 404 (Not Found) لتقليل التشويش والتركيز على الصفحات الموجودة فعلياً.

Feroxbuster 
feroxbuster -u https://target.com/ -w /usr/share/wordlists/seclists/Discovery/Web-Content/raft-small-words.txt -C 404
أداة FFuF (Go-based, highly flexible):

أداة Fuzz Faster U Fool من أقوى الأدوات المبرمجة بلغة Go في عمليات الـ Fuzzing. تعتمد على وضع كلمة FUZZ كمكان لتبديل الكلمات من القائمة. الأوامر التالية توضح كيفية فحص المسارات المباشرة (Directory Fuzzing) مع السماح بأكواد حالة معينة -mc، واكتشاف الملفات بصيغ معينة مثل .php (File extension fuzzing)، وحتى التلاعب بترويسة المضيف Host Header لاكتشاف النطاقات الفرعية الافتراضية المخفية (Virtual host fuzzing).

FFuF - Directory, Extension & VHost 
# Directory fuzzing
ffuf -u https://target.com/FUZZ -w /usr/share/wordlists/seclists/Discovery/Web-Content/common.txt -mc 200,301,302,403

# File extension fuzzing
ffuf -u https://target.com/FUZZ.php -w /usr/share/wordlists/seclists/Discovery/Web-Content/raft-small-words.txt -mc 200

# Virtual host fuzzing
ffuf -u http://target.com -H "Host: FUZZ.target.com" -w subdomains.txt -mc 200
أداة Gobuster Directory Mode:

من الكلاسيكيات المستقرة في عالم اكتشاف المحتوى. استخدام الوضع dir يبحث عن المسارات. الراية -x php,html,js,txt هامة جداً لإجبار الأداة على البحث عن الملفات بتلك الامتدادات المحددة جنباً إلى جنب مع المجلدات، بينما -t 50 تحدد عدد العمليات المتزامنة (Threads) لرفع سرعة الفحص.

Gobuster 
gobuster dir -u https://target.com -w /path/to/wordlist.txt -x php,html,js,txt -t 50
أداة Dirsearch:

سكربت بايثون متخصص وسهل الاستخدام يوفر واجهة نتائج نظيفة وتلقائية. مثل Gobuster، يتم استخدام -e لإرفاق امتدادات الملفات المستهدفة أثناء عملية التخمين وتوسيع نطاق البحث.

Dirsearch 
dirsearch -u https://target.com -e php,html,js -w wordlist.txt

2.2 Archive & Historical Analysis

تحليل السجلات المؤرشفة وتاريخ الموقع (Historical Analysis) هي تقنية استخباراتية ذكية جداً. بدلاً من الاتصال بالسيرفر مباشرة للتخمين، نقوم بسحب جميع الروابط التي تم أرشفتها قديماً عبر محركات البحث أو الأرشيف. قد تكشف هذه الطريقة عن واجهات (APIs) قديمة منسية أو ملفات سرية غير محمية (Zombie APIs).

أداة Waybackurls:

هذا الأمر يمرر اسم النطاق ليتصل بخدمة أرشيف الإنترنت (Wayback Machine) ويستخرج كل الروابط المحفوظة مسبقاً لهذا الموقع عبر الزمن، ويخزنها في ملف نصي للتحليل.

Waybackurls 
echo target.com | waybackurls > old_endpoints.txt
أداة Gau (GetAllUrls):

تُعد أداة Gau أشمل من سابقتها، حيث تجمع الروابط المؤرشفة من مصادر متعددة في نفس الوقت (مثل AlienVault OTX و Common Crawl بالإضافة إلى Wayback Machine) مما يمنحنا رؤية أعمق لتاريخ الموقع.

Gau 
echo target.com | gau > urls.txt
أداة Katana (Modern crawler):

زاحف ويب حديث ومتطور (Crawler) من تطوير ProjectDiscovery. لا يكتفي باستخراج الروابط السطحية بل يتفاعل مع بيئة الموقع. الرايات -jc و -jsl تجعله قادراً على سحب وتحليل الروابط المدفونة داخل ملفات الجافاسكريبت. بينما يقتصر البحث في وضع -f qurl على تنسيق المخرجات لعرض الروابط التي تحتوي على معطيات ومتغيرات (Queries/Parameters) مفيدة فقط.

Katana 
katana -u https://target.com -jc -jsl -f qurl -o endpoints.txt

2.3 JavaScript Analysis

تُعد ملفات الجافاسكريبت (JavaScript) كنزاً دفيناً لصائدي الثغرات. التحليل المعمق لهذه الملفات (JS Analysis) يكشف عن مسارات الـ API المخفية، وأسرار أو مفاتيح تشفير تُركت بالخطأ (Hardcoded Secrets).

التقنيات الأساسية (Key Techniques):
  • Extract endpoints from JS files: استخراج الروابط والمسارات المدفونة والمنسية داخل كود الـ JS.
  • Find API keys, secrets, and credentials: اصطياد مفاتيح الـ API، أسرار التشفير، وبيانات الدخول المتروكة سهواً في الأكواد.
  • Discover hidden parameters and routes: استكشاف المعاملات والمسارات الإدارية لتوسيع سطح الهجوم واكتشاف نقاط ضعف أعمق.

الأدوات المستخدمة: نستخدم Katana للزحف بصمت -silent، وسكربت xnLinkFinder للبحث الفعّال بالتعابير النمطية، وأداة jsluice للتحليل المعمق المبني على شجرة الأكواد (AST)، مع الاستعانة بأوامر grep اليدوية لاصطياد الأنماط السريعة مثل استدعاءات fetch و axios التي تدل على تخاطب الموقع مع قواعد البيانات الخلفية.

JavaScript Analysis Tools 
# Using Katana for JS endpoint extraction
katana -u https://target.com -jc -jsl -f url -silent -o js_endpoints.txt

# Using xnLinkFinder
python3 xnLinkFinder.py -i target.com -o output.txt

# Using jsluice
jsluice https://target.com/app.js

# Manual grep patterns
grep -r "api\." *.js | grep -oP 'https?://[^"'\`]+'
grep -r "fetch\|axios\|XMLHttpRequest" *.js

2.4 Parameter Discovery

اكتشاف المعاملات (Parameters) المخفية في الروابط (مثل ?id=1) يعتبر مرحلة حرجة. اكتشاف متغيرات غير موثقة (Hidden Parameters) يفتح الباب لثغرات حقن خطيرة مثل حقن قواعد البيانات (SQLi) وتنفيذ السكربتات (XSS) أو ثغرات التحكم المباشر (IDOR).

أداة Arjun:

أداة ذكية للغاية في اكتشاف المعاملات. بدلاً من التجربة واحدة تلو الأخرى، تستخدم تقنيات متطورة لتجربة آلاف المعاملات في طلبات مجتمعة وتحليل الاستجابات لمعرفة بدقة أي متغير (Parameter) أحدث تأثيراً في نتيجة الصفحة.

Arjun 
python3 arjun.py -u https://target.com/page.php
أداة ParamSpider:

لا تقوم بالتخمين النشط، بل تعتمد على سجلات أرشيف الإنترنت للبحث السريع عن الروابط القديمة للموقع التي تحتوي فعلياً على Parameters حقيقية بدلاً من التخمين العشوائي، مما يوفر وقت وجهد كبير.

ParamSpider 
paramspider --domain target.com
أداة FFuF Parameter Fuzzing:

تستخدم أداة FFuF بالأسلوب التقليدي (Parameter Fuzzing) عبر وضع كلمة FUZZ مكان المعامل المجهول وتمرير قائمة تحتوي على أسماء المتغيرات الشائعة لمعرفة ما يقبله السيرفر، مع إعطاء قيمة تجريبية test للمتغير.

FFuF Parameter Fuzzing 
ffuf -u https://target.com/page.php?FUZZ=test -w /usr/share/wordlists/seclists/Discovery/Web-Content/burp-parameter-names.txt -mc 200
أداة X8 (Hidden parameter discovery):

أداة سريعة جداً مبنية بلغة Rust، تُعد من أقوى البدائل العصرية. تعتمد على خوارزميات متقدمة لإرسال كميات كبيرة من المتغيرات وتحليل ردود السيرفر لاكتشاف الـ Parameters المخفية التي تؤدي لتغيير في سلوك الصفحة بدقة فائقة.

X8 
x8 -u https://target.com/page.php -w parameters.txt

2.5 Content Discovery Pipeline

هذا القسم يوضح "مسار العمل الكامل" أو الـ (Pipeline) لمرحلة اكتشاف المحتوى. في البيئة الاحترافية، لا نعتمد على أداة واحدة فحسب، بل نقوم بدمج المخرجات من أدوات الزحف والأرشفة لتكون مدخلات لأدوات البحث المتعمق والـ Fuzzing للحصول على صورة بانورامية شاملة للبنية التحتية للهدف.

التسلسل المنطقي للبحث (Pipeline Breakdown):
  • الخطوة 1: الزحف المباشر واستخراج الروابط المرئية والحية الحالية باستخدام Katana وحفظها في ملف.
  • الخطوة 2: توسيع دائرة البحث عبر إضافة الروابط المؤرشفة تاريخياً عبر Gau وإلحاقها >> بنفس الملف.
  • الخطوة 3: فلترة قائمة الروابط الكلية لاقتناص ملفات .js وتمريرها تباعاً باستخدام xargs لأداة Katana مرة أخرى لاستخراج أي مسارات خفية بداخلها.
  • الخطوة 4: إجراء فحص التخمين Brute Force الشامل على مسارات الخادم باستخدام FFuF.
  • الخطوة 5: أخذ جميع الروابط، وفلترة تلك التي تحتوي على معاملات (علامة ?)، وتمريرها لأداة Arjun لاكتشاف المتغيرات الإضافية والمخفية (Parameter Fuzzing) وحفظها كملف JSON.
Full Discovery Pipeline 
# Step 1: Crawl and collect URLs
katana -u https://target.com -jc -jsl -f url -silent -o all_urls.txt

# Step 2: Add historical URLs
echo target.com | gau >> all_urls.txt

# Step 3: Extract JS endpoints
cat all_urls.txt | grep "\.js$" | xargs -I {} katana -u {} -jc -jsl -f url >> js_endpoints.txt

# Step 4: Directory brute force
ffuf -u https://target.com/FUZZ -w /usr/share/wordlists/seclists/Discovery/Web-Content/raft-medium-directories.txt -mc 200,301,302,403 -o directories.txt

# Step 5: Parameter discovery
cat all_urls.txt | grep "?" | xargs -I {} arjun -u {} -oJ params.json
