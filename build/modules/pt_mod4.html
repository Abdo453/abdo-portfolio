04. Content Discovery

اكتشاف المحتوى المخفي وتخمين المسارات (Content Discovery & Directory Fuzzing)

الجانب النظري (Theory)

تعتبر عملية اكتشاف المحتوى (Content Discovery) من أهم مراحل اختبار الاختراق (Reconnaissance) وأكثرها دقة. تعتمد هذه المرحلة على العثور على الملفات، المجلدات، الواجهات البرمجية (APIs)، والمسارات غير المعلن عنها والتي قد تحتوي على بيانات حساسة، أو لوحات تحكم (Admin Panels)، أو شفرات مصدرية (Source Code)، أو ثغرات أمنية غير مرقعة.

تشمل هذه العملية تقنيات متعددة مثل التخمين العنيف (Fuzzing / Brute-forcing) باستخدام قوائم كلمات (Wordlists) متخصصة، وتحليل ملفات الجافاسكريبت (JS Analysis) لاستخراج مسارات الـ Endpoints المخفية، ومراجعة ملفات الفهرسة مثل robots.txt و sitemap.xml، بالإضافة إلى استخدام محركات البحث (Google Dorks) أو أرشيف الإنترنت (Wayback Machine). الهدف الأساسي هو توسيع السطح الهجومي (Attack Surface) قدر الإمكان.

الاختبار اليدوي (Manual Testing)

  • فحص الملفات القياسية: قم بزيارة مسارات مثل /robots.txt، /sitemap.xml، /.well-known/، /crossdomain.xml بشكل يدوي لاستكشاف الروابط والمسارات التي تم منع العناكب (Crawlers) من الوصول إليها.
  • تحليل الكود المصدري: اضغط على Ctrl+U لقراءة كود HTML والبحث عن تعليقات المطورين (Developer Comments) التي قد تحتوي على روابط، بيانات اعتماد، أو ملاحظات هامة.
  • مراجعة ملفات JavaScript: افتح DevTools (F12) وانتقل إلى قسم Network أو Sources، وابحث في ملفات .js عن مسارات (Endpoints)، مفاتيح API، أو بيانات ثابتة (Hardcoded Credentials).
  • استخدام محركات البحث المتقدمة (Dorks): استعن بـ Google للبحث عن مسارات مكشوفة مثل: site:target.com intitle:"index of" أو site:target.com filetype:pdf.

الأتمتة والسكربتات (Automation)

أتمتة اكتشاف المحتوى تتطلب استخدام أدوات عالية السرعة مكتوبة بلغات مثل Go، مدعومة بقوائم كلمات مخصصة (مثل قوائم SecLists). من المهم جداً ضبط أعداد الخيوط (Threads) وتقنيات تجاوز الجدار الناري (WAF) وتصفية الردود (Filtering) لتقليل النتائج الزائفة (False Positives).

  • Ffuf / Gobuster / Feroxbuster: أدوات سريعة لعمل Fuzzing واكتشاف المجلدات والملفات.
  • Katana / Hakrawler: أدوات زحف (Crawling) متطورة لاستخراج الروابط من الصفحات ومحركات البحث.
  • LinkFinder / SecretFinder: سكربتات متخصصة في تحليل ملفات الجافاسكريبت (JS) لاستخراج المسارات والأسرار.
  • Waybackurls / Gau: لجلب الروابط المؤرشفة للنطاق المستهدف من مصادر متعددة.

الأدوات والأوامر (Tools)

1. اكتشاف المجلدات باستخدام Ffuf

$ ffuf -w /usr/share/seclists/Discovery/Web-Content/raft-large-directories.txt -u https://target.com/FUZZ -mc 200,301,302,403 -c

2. اكتشاف الملفات بامتدادات محددة

$ ffuf -w /usr/share/seclists/Discovery/Web-Content/raft-medium-words.txt -u https://target.com/FUZZ -e .php,.txt,.bak,.zip -mc 200 -c

3. الزحف الذكي والتكراري باستخدام Feroxbuster

$ feroxbuster -u https://target.com -w /usr/share/seclists/Discovery/Web-Content/directory-list-2.3-medium.txt -t 50 -x php,bak,txt

4. جلب الروابط المؤرشفة (Waybackurls)

$ echo "target.com" | waybackurls | sort -u > archive_links.txt

5. استخراج المسارات من ملفات JS (LinkFinder)

$ python3 linkfinder.py -i https://target.com/app.js -o cli

ورقة مرجعية (Cheat Sheet)

التقنية / الأداة (Tool/Technique) الاستخدام الشائع (Usage) ملاحظات (Notes)
ffuf Fuzzing للمسارات والبارامترات والملفات. سريعة، تدعم الـ Filtering بشكل متقدم (-fs, -fw, -fc).
Feroxbuster Fuzzing تكراري (Recursive). تقوم تلقائياً بفحص المجلدات الجديدة التي تجدها.
Waybackurls / gau جلب روابط مؤرشفة من مصادر خارجية. ممتازة لإيجاد مسارات API قديمة غير محمية.
JS Analysis تحليل ملفات جافاسكريبت بحثاً عن endpoints و secrets. استخدم أدوات مثل LinkFinder أو إضافة TruffleHog.
SecLists مستودع شامل لقوائم الكلمات (Wordlists). استخدم قوائم Discovery/Web-Content للبحث.
