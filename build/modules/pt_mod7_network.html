🌐 7. Network Layer Testing

فحص وتقييم أمان طبقة الشبكة: من تحليل مسار الحزم واكتشاف الأنفاق إلى تقييم التجزئة والبروتوكولات الحساسة.

7.1 Traceroute & Path Analysis

تهدف هذه المرحلة إلى رسم خريطة لمسار البيانات (Packets) من المصدر حتى الهدف. من خلال تحليل القفزات (Hops)، يمكننا تحديد جدران الحماية (Firewalls)، وأجهزة التوجيه (Routers)، وأي عقبات أمنية في الشبكة. يتم استخدام بروتوكولات مختلفة (مثل TCP و ICMP) لتخطي محاولات الحجب واكتشاف المسار الحقيقي.

Standard traceroute 
# Standard traceroute
traceroute target.com

الاستخدام الافتراضي يعتمد عادةً على بروتوكول ICMP أو UDP. قد يتم حجبه بواسطة العديد من جدران الحماية الحديثة.

TCP traceroute 
# TCP traceroute (bypasses ICMP blocks)
traceroute -T -p 443 target.com

استخدام -T يرسل حزم TCP SYN إلى منفذ محدد (مثل 443). هذه الطريقة فعالة جداً لتجاوز جدران الحماية التي تحجب رسائل ICMP، حيث يصعب على الجدار الناري حجب حركة مرور الويب (TCP 443) دون التأثير على عمل الموقع.

Using Nmap & MTR 
# Using Nmap for traceroute
nmap --traceroute target.com
# MTR (My Traceroute) - continuous monitoring
mtr target.com

7.2 VPN & Tunnel Detection

اكتشاف نقاط النهاية الخاصة بالشبكات الافتراضية (VPN Endpoints) وبروتوكولات الأنفاق (Tunneling) يُعتبر خطوة حاسمة. هذه المنافذ غالباً ما تُوفر وصولاً مباشراً لشبكة المؤسسة الداخلية، ويجب التدقيق في إعدادات التشفير وإصدارات البروتوكول المستخدمة للتحقق من عدم وجود ثغرات في عملية المصادقة (Authentication).

VPN endpoints & protocols 
# Check for VPN endpoints
nmap -p 443,1194,500,4500,1723 target.com
# Check for VPN protocols
nmap --script=ike-version -p 500 target.com
nmap --script=ssl-enum-ciphers -p 443 target.com
WireGuard detection 
# WireGuard detection (UDP 51820)
nc -u -v target.com 51820

بروتوكول WireGuard يعتمد بالكامل على بروتوكول UDP. استخدام أداة nc (Netcat) للاتصال بمنافذ UDP يساعد في الكشف عن الخدمة بصمت، ولكن يجب الانتباه إلى أن UDP قد لا يرد في حال عدم وجود خدمة نشطة أو وجود جدار حماية (Firewall Drop).

7.3 Load Balancer Detection

الأنظمة الكبيرة تعتمد على موزعات الأحمال (Load Balancers) لتوزيع حركة المرور عبر عدة خوادم خلفية (Backend Servers). اكتشافها يساعد المهاجم (أو مختبر الاختراق) في فهم البنية التحتية، وتجنب الخوادم الكاذبة، واستهداف خوادم خلفية محددة قد تحمل ثغرات مختلفة عن الخوادم الأخرى.

Check for load balancer headers 
# Check for load balancer headers
curl -I https://target.com | grep -i "x-forwarded\|via\|x-cache\|x-served-by"

تحليل ترويسات HTTP (Headers) يمكن أن يكشف صراحة عن هوية ونوع موزع الأحمال أو الـ Proxy العكسي المستخدم، حيث تتم إضافة ترويسات مثل Via أو X-Forwarded-For بواسطة هذه الأنظمة.

Check different backends & Nmap scripts 
# Multiple requests to check different backends
for i in {1..10}; do curl -s https://target.com | md5sum; done
# Nmap load balancer detection
nmap --script=http-loadbalancer -p 80,443 target.com

7.4 Network Segmentation Testing

اختبار تجزئة الشبكة (Network Segmentation) يهدف إلى التأكد من أن الشبكات الفرعية (Subnets) أو (VLANs) معزولة بشكل صحيح. إذا تم اختراق جهاز في شبكة ذات صلاحيات منخفضة، يجب ألا يتمكن المهاجم من الوصول إلى الشبكات الحساسة (مثل شبكة الخوادم المركزية أو قواعد البيانات).

Inter-VLAN routing & Firewalls 
# Check inter-VLAN routing
# From compromised host, scan adjacent subnets
nmap -sn 10.0.1.0/24

# Check firewall rules
# Attempt connections to various ports across subnets

الاعتماد على -sn يقوم بفحص الأجهزة النشطة دون فحص المنافذ (Ping Sweep). تتبع هذه الخطوة محاولات اتصال فعلية بمنافذ محددة للتأكد من قواعد التوجيه وجدران الحماية الداخلية (Internal ACLs).

7.5 DNS Security Testing

نظام أسماء النطاقات (DNS) هو العمود الفقري لأي شبكة. تقييم أمان الـ DNS يشمل التحقق من تطبيق DNSSEC لمنع هجمات التسميم (DNS Spoofing)، بالإضافة إلى التأكد من أن الخوادم غير معدة لتكون "Open Resolvers"، وهو ما يمنع استغلالها في هجمات تضخيم حجب الخدمة الموزع (DDoS Amplification).

DNS Security Checks 
# DNSSEC check
dig +dnssec target.com DNSKEY

# Check for DNS amplification (open resolvers)
dig @resolver.target.com ANY target.com

# Check for DNS cache snooping
dig @resolver.target.com target.com +norecurse

استخدام +norecurse في فحص "Cache Snooping" يفحص إذا ما كان خادم الـ DNS يحتفظ بطلبات نطاقات معينة مسبقاً، مما يكشف للمهاجم عن طبيعة الخدمات والمواقع التي تزورها الشركة من داخل شبكتها.

7.6 Network Protocol Testing

فحص بروتوكولات الشبكة الحساسة هو قلب التقييم الداخلي. بروتوكولات مثل SMB، SNMP، LDAP، و RDP غالباً ما تكون مليئة بالثغرات إذا لم يتم تحديث الأنظمة، أو إذا تم تهيئتها بإعدادات افتراضية تمنح صلاحيات مفرطة دون الحاجة إلى مصادقة قوية.

Network Protocols Enumeration 
# SMB testing
nmap --script=smb-security-mode,smb-enum-shares -p 445 target.com

# SNMP testing
nmap --script=snmp-brute,snmp-interfaces -p 161 target.com

# LDAP testing
nmap --script=ldap-search -p 389 target.com

# RDP testing
nmap --script=rdp-enum-encryption,rdp-vuln-ms12-020 -p 3389 target.com

سكربتات Nmap الخاصة بالـ (NSE) تعتبر أدوات قوية جداً. على سبيل المثال، smb-enum-shares يحدد الملفات المشتركة المتاحة ويختبر صلاحيات الوصول إليها، بينما يقوم سكربت rdp-vuln-ms12-020 بالتحقق مما إذا كان خادم RDP عرضة لواحدة من أخطر الثغرات التي تسمح بتنفيذ الأوامر عن بعد والتسبب بشاشة الموت الزرقاء في أنظمة الويندوز.
