09. Authentication & Session Management

الاختبار الشامل لآليات المصادقة، إدارة الجلسات، OAuth، SAML، و JWT.

1. النظرية (Theory)

تُعد المصادقة (Authentication - AuthN) العملية التي يتم من خلالها التحقق من هوية المستخدم، بينما التفويض (Authorization - AuthZ) يحدد الصلاحيات الممنوحة لهذه الهوية. تمثل هذه الأنظمة خط الدفاع الأول لأي تطبيق.

في التطبيقات الحديثة، تطورت آليات المصادقة لتشمل:

  • Session Management: الإدارة التقليدية عبر ملفات تعريف الارتباط (Cookies) ومعرفات الجلسة (Session IDs).
  • JWT (JSON Web Tokens): أسلوب لامركزي لتوثيق الجلسات يحمل البيانات في الـ Payload (Base64) ويتم توقيعه باستخدام خوارزميات مثل HS256 أو RS256.
  • OAuth 2.0 & SAML: بروتوكولات التفويض والمصادقة الموحدة (SSO). تشمل هجمات OAuth التلاعب بـ redirect_uri للسيطرة على الـ Access Token، أو استغلال غياب متغير state لتنفيذ هجمات CSRF.

أهم الثغرات: ضعف التشفير (Broken Authentication)، تجاوز المصادقة، التثبيت (Session Fixation)، وهجمات تخمين كلمات المرور (Brute Force/Credential Stuffing).

3. الاختبار اليدوي (Manual Testing)

  • تحليل JWT: قم بالتقاط الـ Token، وفك تشفير الـ Payload. جرب تعديل الـ Header لتغيير الخوارزمية إلى None وإزالة التوقيع.
  • اختبار OAuth: أثناء عملية الـ Authorization Request، جرب تغيير redirect_uri إلى خادم تتحكم به (مثلاً https://attacker.com) لاختطاف الـ Authorization Code.
  • تثبيت الجلسة (Session Fixation): قم بتسجيل الدخول ولاحظ ما إذا كان التطبيق يعيد استخدام نفس الـ Session ID بعد نجاح المصادقة. إذا كان الأمر كذلك، يمكنك تمرير الـ ID للضحية لاختطاف جلسته.
  • اختبار غياب الحماية من التخمين: حاول تسجيل الدخول بكلمات مرور خاطئة بشكل متكرر لمعرفة ما إذا كان هناك آلية Lockout أو Rate Limiting.

4. الأتمتة (Automation)

الأتمتة ضرورية في اختبار المصادقة لاكتشاف كلمات المرور الضعيفة والتلاعب السريع بالرموز.

  • Hydra & ffuf: لتنفيذ هجمات Brute Force و Credential Stuffing بسرعة عالية عبر الـ HTTP POST.
  • jwt_tool: أداة بايثون قوية تقوم بأتمتة هجمات JWT (مثل Key Confusion، None Algorithm bypass).
  • Burp Suite Extensions: استخدام "AuthMatrix" لاختبار مصفوفات الصلاحيات والمصادقة الموازية، و "JSON Web Tokens" للتعامل مع الرموز بذكاء.

8. ورقة المراجع السريعة (Cheat Sheet)

الثغرة (Vulnerability) طريقة الفحص (Test) الأثر (Impact)
JWT None Algorithm تغيير قيمة `alg` إلى `none` أو `None` وحذف التوقيع. Account Takeover (اختطاف الحساب).
OAuth Redirect URI Spoofing تعديل redirect_uri=https://evil.com في طلب المصادقة. سرقة الـ Authorization Code واختطاف الجلسة.
Session Fixation تمرير Session ID خاص بك في الرابط أو الكوكيز ومراقبة المصادقة. استيلاء على حساب الضحية بمجرد تسجيل دخوله.
Credential Stuffing استخدام قوائم بيانات مسربة (Combos) ومحاولة الدخول. الوصول لحسابات مستخدمين حقيقيين.
