💻 COMMANDS
subdomain-enum.sh
① Passive Collection
# Passive subdomain discovery
subfinder -d target.com -all -recursive -silent -o subs.txt
amass enum -passive -d target.com -o amass.txt
assetfinder --subs-only target.com > assetfinder.txt

# Merge & deduplicate all sources
cat subs.txt amass.txt assetfinder.txt | sort -u > all_subs.txt
auto-recon.sh (All-in-One)
#!/bin/bash
TARGET=$1
echo "[+] Running Subfinder..."
subfinder -d $TARGET -all -silent > subs.txt
echo "[+] Resolving & Probing HTTP..."
cat subs.txt | httpx -silent -threads 200 > live.txt
echo "[+] Done! Found $(wc -l < live.txt) live hosts."

⚠️ COMMON MISTAKES VS BEST PRACTICES
■ Common Mistakes
✗ الاكتفاء بـ subfinder وحدها فقط
✗ تجاهل 404 responses بدون فحص CNAME
✗ عدم فحص المنافذ غير المعيارية
✗ بدء Active Recon قبل انتهاء Passive
✗ نسيان wildcard: *.target.com
■ Best Practices
✓ دمج 4+ مصادر + sort -u دائماً
✓ فحص CNAME لكل 404 بـ dig
✓ naabu على كل host للمنافذ غير المعيارية
✓ الأولوية الأولى دائماً = dev / staging / admin
✓ gau + waybackurls للأرشيف التاريخي
✓ لاحقاً دائماً Active ← أولاً Passive

📝 REPORT TEMPLATE: Subdomain Takeover
Title: Subdomain Takeover on dev.target.com
Description:
I found that dev.target.com is pointing to a third-party service (e.g., GitHub Pages / S3 Bucket) that is no longer claimed. An attacker can claim this resource and host arbitrary content on a subdomain belonging to the company.
Steps To Reproduce:
1. Navigate to http://dev.target.com
2. Observe the '404 Not Found' or 'NoSuchBucket' error.
3. Register an account with the provider and claim the domain.
Impact:
An attacker can host phishing pages, steal session cookies via XSS, or damage the company's reputation.
