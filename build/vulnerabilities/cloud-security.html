أمان السحابة (Cloud Security) — الدليل الشامل

Cloud Security (AWS, Azure, GCP) — The Ultimate Deep Dive

CRITICAL / HIGH Cloud Infrastructure
AR

أمان السحابة (Cloud Security) يختلف جذرياً عن أمان الشبكات التقليدية (On-Premises). في السحابة، البنية التحتية عبارة عن كود (IaC)، والهوية (Identity/IAM) هي المحيط الأمني الجديد (New Perimeter). أخطاء التكوين (Misconfigurations) وعدم تطبيق مبدأ "أقل الصلاحيات" (Least Privilege) يؤديان إلى اختراقات كارثية، حيث يمكن لمهاجم استغلال ثغرة بسيطة (مثل SSRF في تطبيق ويب) لسرقة بيانات اعتماد (Credentials) من السحابة والوصول إلى كافة الخوادم وقواعد البيانات.

EN

Cloud Security focuses on securing infrastructure hosted on platforms like AWS, Azure, and GCP. In the cloud, the traditional network perimeter is replaced by Identity and Access Management (IAM). Misconfigurations, such as publicly accessible storage buckets or overly permissive IAM roles, are the primary vectors for devastating data breaches and complete account takeovers.

أبرز ثغرات أمان السحابة (Cloud Vulnerabilities)

1. Public Storage Buckets (S3/Blob Misconfigurations)

أكثر الأخطاء شيوعاً. ترك حاويات التخزين (مثل AWS S3 Buckets أو Azure Blob Storage) مفتوحة للعامة (Public Read/Write) مما يؤدي إلى تسريب بيانات العملاء أو السماح للمهاجم برفع ملفات خبيثة.

2. استغلال Metadata Service عبر SSRF

إذا كان هناك تطبيق ويب يعاني من ثغرة SSRF ويعمل على AWS EC2، يمكن للمهاجم استدعاء مسار الـ Metadata (169.254.169.254/latest/meta-data/iam/security-credentials/) لسرقة الـ Access Keys الخاصة بالخادم (تجاوز الـ IMDSv1).

3. Overly Permissive IAM Roles

منح سيرفر الويب صلاحيات زائدة عن حاجته (مثلاً إعطاءه s3:* أو iam:PassRole). إذا تم اختراق هذا السيرفر، يستطيع المهاجم التوسع (Privilege Escalation) للوصول إلى خدمات أخرى لم تكن مستهدفة أساساً.

4. تسريب الـ Secrets & Access Keys

رفع الـ Access Keys أو مفاتيح الـ API عن طريق الخطأ إلى GitHub أو تركها مكشوفة في ملفات .env أو داخل كود التطبيق (Hardcoded Secrets). المخترقون يستخدمون روبوتات (Bots) تفحص GitHub لاستغلال هذه المفاتيح في ثوانٍ لاستخراج العملات الرقمية (Cryptomining).

مراحل الهجوم في السحابة (Cloud Kill Chain)

AR

الهجوم السحابي يتبع مساراً مميزاً:

  • Initial Access (الوصول الأولي): عبر تسريب مفاتيح (Leaked Keys)، أو SSRF، أو تطبيق ويب مخترق، أو Phishing لمسؤول السحابة.
  • Discovery (الاستكشاف): استخدام المفاتيح المسروقة لمعرفة الصلاحيات المتاحة (مثلاً أمر aws sts get-caller-identity لمعرفة من أنت، ثم aws iam list-attached-user-policies).
  • Privilege Escalation (تصعيد الصلاحيات): استغلال أخطاء IAM، مثل قدرة المستخدم على إنشاء Policy جديدة وإرفاقها بنفسه ليصبح Admin.
  • Data Exfiltration (تسريب البيانات): نسخ محتويات قواعد البيانات (RDS) أو الـ S3 Buckets وتحميلها لسيرفر المهاجم.
  • Impact (التأثير): حذف قواعد البيانات وطلب فدية (Ransomware)، أو تشغيل آلاف الخوادم لتعدين العملات (Cryptojacking) مما يكلف الضحية ملايين الدولارات في الفاتورة.

أدوات الفحص السحابي (Cloud Security Tools)

ScoutSuite Pacu (AWS Exploitation Framework) CloudFox Trufflehog (Secrets)

أداة Pacu هي بمثابة Metasploit لكن لبيئات AWS، تستخدم لاختبار الاختراق وتصعيد الصلاحيات. أما ScoutSuite فهي ممتازة لعمل Audit (مراجعة أمنية شاملة) لاستخراج تقرير بالـ Misconfigurations في AWS, Azure, و GCP.

كيفية الحماية (Remediation)

  • تفعيل IMDSv2 (في AWS): يمنع معظم هجمات الـ SSRF التي تسرق المفاتيح عن طريق اشتراط إرسال طلب PUT مسبق للحصول على توكن.
  • تطبيق مبدأ Least Privilege: لا تستخدم * في سياسات IAM أبداً. حدد بالضبط الموارد والإجراءات (Actions) التي يحتاجها التطبيق.
  • إدارة الأسرار (Secrets Management): لا تضع كلمات المرور أو المفاتيح في الكود المصدري. استخدم خدمات مثل AWS Secrets Manager أو HashiCorp Vault.
  • المراقبة (Monitoring & Logging): تفعيل AWS CloudTrail و GuardDuty لمراقبة أي نشاط غير طبيعي (مثل استخدام مفتاحك من دولة مختلفة أو محاولة قراءة بيانات حساسة بكثافة).
