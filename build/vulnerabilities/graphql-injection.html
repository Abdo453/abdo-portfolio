أمن الـ GraphQL — ما بعد الاستعلام الذاتي

GraphQL Security — Beyond Introspection

HIGH OWASP API07
AR

GraphQL هو محرك استعلام حديث للـ APIs. بالرغم من مرونته، إلا أنه يعاني من ثغرات خطيرة مثل الاستعلام الذاتي المفتوح (Introspection)، وهجمات تخمين الحزم (Batching)، وتجاوز قيود عمق الاستعلامات (Depth Limit) مما يسبب حجب الخدمة (DoS)، وحقن الاستعلامات.

EN

GraphQL is a modern query engine for APIs. Despite its flexibility, it introduces unique vulnerabilities such as exposed Introspection, query batching attacks, depth limit bypasses leading to DoS, and query injection.

أنواع ثغرات GraphQL / Vulnerability Types

Introspection Disclosure

استرجاع مخطط الـ API بالكامل (Schema) لمعرفة كل الاستعلامات المتوفرة.

GraphQL Injection

حقن نصوص برمجية (SQL, NoSQL, OS cmd) داخل معاملات الاستعلام.

Query Batching Attack

إرسال مئات الاستعلامات في طلب HTTP واحد لتخطي قيود الـ Rate Limit.

Depth Limit Bypass

استعلامات دائرية لا نهائية (Circular Queries) تتسبب في انهيار خادم الويب.

أوامر الاختبار / Testing Commands

# Introspection Query to dump Schema
POST /graphql HTTP/1.1
Content-Type: application/json

{"query": "{__schema{types{name fields{name args{name type{name}}}}}}"}

# GraphQL Injection (SQLi) inside query
POST /graphql HTTP/1.1
Content-Type: application/json

{"query": "query { users(id: \"1' OR '1'='1\") { username email } }"}
