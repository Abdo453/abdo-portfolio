IDOR — الوصول المباشر غير الآمن للكائنات

IDOR — Insecure Direct Object Reference

HIGH OWASP A01
AR

IDOR يحدث عندما يستطيع المستخدم الوصول لبيانات أو موارد تخص مستخدمين آخرين بتغيير معرّف (ID, UUID, filename) في الطلب — بدون فحص صلاحيات كافٍ من السيرفر.

EN

IDOR happens when a user accesses another user's data by modifying an object identifier (ID, UUID, filename) in the request — without adequate server-side authorization checks.

أنواع IDOR / IDOR Types

URL Parameter IDOR

تغيير user_id=123 → user_id=124

API / JSON Body

تعديل id أو accountId في POST/PUT body

UUID Manipulation

تخمين أو leak UUIDs من responses أخرى

Mass Assignment

إرسال fields زيادة: role: admin, isPremium: true

Function-Level BAC

الوصول لـ admin endpoints كـ regular user

Path Traversal

../../etc/passwd في file download paths

أوامر الاختبار / Testing Commands

# URL IDOR
GET /api/users/1001/profile HTTP/1.1
GET /api/users/1002/profile HTTP/1.1

# JSON body swap
POST /api/orders HTTP/1.1
{"order_id": "ORD-9999", "user_id": "VICTIM_UUID"}

# Path traversal
GET /download?file=../../../etc/passwd HTTP/1.1
