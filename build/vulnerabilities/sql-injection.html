حقن SQL — من الصفر للاحتراف

SQL Injection — From Zero to Hero

CRITICAL OWASP A03
AR

SQL Injection ثغرة تحدث عندما يُدمَج مدخل المستخدم مباشرة في استعلام SQL بدون تعقيم (sanitization). المهاجم يستطيع قراءة، تعديل، أو حذف بيانات قاعدة البيانات — وأحياناً الوصول لـ RCE عبر xp_cmdshell أو file write.

EN

SQL Injection occurs when user input is concatenated into SQL queries without proper sanitization. Attackers can read, modify, or delete database records — and sometimes achieve RCE via xp_cmdshell or file write primitives.

أنواع SQLi / SQLi Types

In-band (Classic)

النتيجة تظهر في نفس response — الأسهل للاستغلال.

Results appear in the same HTTP response.

Union-based

استخدام UNION SELECT لسحب بيانات من جداول أخرى.

Use UNION SELECT to extract data from other tables.

Boolean-based Blind

الفرق بين true/false في المحتوى — استخراج حرف حرف.

Infer data from true/false response differences.

Time-based Blind

SLEEP(5) أو WAITFOR DELAY — لا فرق في المحتوى.

SLEEP() / WAITFOR DELAY — no visible content change.

Error-based

استغلال رسائل الخطأ لاستخراج البيانات.

Extract data from verbose SQL error messages.

NoSQL Injection

MongoDB, Redis — operators مثل $gt, $ne.

MongoDB/Redis — abuse operators like $gt, $ne.
