📚 النظرية — كيف تعمل هجمات عيوب التشفير

يعتمد التشفير الآمن على الرياضيات والبروتوكولات المعقدة، ولكن عيوب التكوين (Misconfigurations) أو طريقة معالجة الأخطاء من قبل التطبيقات تتيح للمهاجمين كشف البيانات دون امتلاك مفتاح التشفير.

🍪 هجوم أوراكل الحشو (Padding Oracle Attack)

تستخدم خوارزميات التشفير الكتلية (Block Ciphers) مثل AES بوضع تشفير (CBC) نظام حشو (Padding) مثل PKCS#7 لجعل النص متوافقاً مع حجم الكتل.

أثناء فك التشفير، إذا كان خادم الويب يرجع خطأً محدداً للمتصفح عند وجود حشو غير صحيح (Bad Padding Error)، بينما يرجع خطأً مختلفاً للبيانات غير الصالحة، يمكن للمهاجم استغلال هذا الرد (Oracle) كمرشد لتخمين وفك تشفير النص المشفر بالكامل بايت تلو بايت بإرسال طلبات معدلة تدريجياً.

🔗 هجوم تمديد طول الهاش (Hash Length Extension Attack)

يحدث في خوارزميات الهاش التي تعتمد على بنية Merkle-Damgård (مثل MD5, SHA-1, SHA-256).

إذا كان السيرفر يعتمد على صيغة تحقق بسيطة Hash(secret + message) للتحقق من هوية البيانات، يستطيع المهاجم (حتى دون معرفة طول أو قيمة الـ secret) توليد رسالة جديدة وتوقيع هاش صالح لها عن طريق بدء الحالة الداخلية للهاش من الهاش القديم المعلوم وإلحاق بيانات جديدة.

🔬 تشريح خطوات الهجوم والـ Playbook

1️⃣
Padding Oracle Exploitation (PadBuster)

تحديد المعامل المصاب واستخدام أداة PadBuster لإرسال حزم معدلة وحساب البايتات وفك تشفير الجلسة.

# Exploit padding oracle on a cookie value: padbuster http://target.com/index.php "ENCRYPTED_COOKIE_VALUE" 16 \ -cookies "SessionID=ENCRYPTED_COOKIE_VALUE" \ -error "Invalid padding"
2️⃣
Hash Length Extension (HashPump)

توليد هاش صالح لرسالة معدلة دون معرفة الكلمة السرية الأساسية:

# Using hashpump to generate new signature: # -s (signature), -d (original data), -a (data to append), -k (estimated key length) hashpump -s "571586a11b0200c01" -d "user=abdo" -a "&role=admin" -k 16

🛡️ الحماية والدفاع — تأمين التشفير والتوقيع

✅ حماية التشفير الكتلي (Prevent Padding Oracle)

  • Use Authenticated Encryption (AEAD): استخدام أوضاع تشفير مدمجة بمصادقة تلقائية مثل AES-GCM بدلاً من استخدام AES-CBC البسيط.
  • HMAC then Encrypt: تطبيق توقيع رقمي (HMAC) على النص المشفر بأكمله للتحقق من سلامة البيانات قبل البدء بفك التشفير، مما يمنع وصول الحزم المتلاعب بها إلى خوارزمية فك الحشو.

🔒 تأمين هاش التحقق (Prevent Length Extension)

  • Use HMAC: استخدام معيار HMAC الرسمي (مثل hash_hmac في PHP أو hmac في بايثون) بدلاً من الدمج البسيط للنصوص.
  • SHA-3 / Blake2: استخدام خوارزميات هاش حديثة ومقاومة لهجمات الامتداد كـ SHA-3.
