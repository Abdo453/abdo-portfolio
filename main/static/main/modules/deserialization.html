📚 النظرية — ما هي Serialization و Deserialization؟

الـ Serialization (التسلسل): هي عملية تحويل كائن برمجي معقد في الذاكرة (Object) إلى تنسيق مسطح من البيانات (مثل بايتات ثنائية أو نص JSON/XML) ليتمكن التطبيق من حفظه في ملف أو إرساله عبر الشبكة.
الـ Deserialization (فك التسلسل): هي العملية العكسية — إعادة بناء الكائن البرمجي الكامل في الذاكرة من خلال قراءة البيانات المسطحة الممررة.

🚨 ثغرة فك التسلسل غير الآمن (Insecure Deserialization)

تحدث الثغرة عندما يقوم التطبيق بفك تسلسل بيانات مرسلة من جهة غير موثوقة (المستخدم) دون التحقق من صحتها. يستغل المهاجم هذه الثغرة عبر حقن كائن من فئة معروفة (Class) يمتلك وظائف خاصة تُستدعى تلقائياً عند فك التسلسل (تُعرف بـ Magic Methods).

🔗 سلاسل الأدوات (Gadget Chains)

لا يملك الكائن المحقون أثراً ضاراً بمفرده في الغالب، ولكن عبر استغلال الـ Magic Methods، يتم تمرير التحكم برمجياً من كائن لآخر كأحجار الدومينو (Gadget Chain) حتى يصل التنفيذ في النهاية إلى دالة قادرة على تشغيل الأوامر (مثل Runtime.getRuntime().exec() في جافا).

🔬 تشريح الهجوم والأساليب البرمجية

🐘 فك التسلسل في PHP (Magic Methods)

تستخدم لغة PHP دوال سحرية تُنفذ تلقائياً عند مراحل معينة من عمر الكائن:
__wakeup() عند فك التسلسل.
__destruct() عند تدمير الكائن وحذفه من الذاكرة.
__toString() عند محاولة معاملة الكائن كنص.

مثال لكود ضعيف بالـ PHP:

// ❌ BAD: Deserializing raw user input
$user = unserialize($_COOKIE['user_session']);

🐍 فك التسلسل في بايثون (Pickle Module)

في بايثون، تُستخدم مكتبة pickle للتسلسل. استدعاء دالة __reduce__() داخل الكائن أثناء فك التسلسل يتيح تحديد كود كامل ليتم تنفيذه فوراً. 

# Python exploit payload generating using pickle
import pickle
import os

class Exploit(object):
    def __reduce__(self):
        return (os.system, ('whoami',))

# Serialize payload
print(pickle.dumps(Exploit()))

🛡️ الحماية والدفاع — تأمين فك تسلسل البيانات

✅ أساليب برمجية آمنة

  • Avoid raw serialization: تجنب استخدام فك التسلسل الافتراضي للغات البرمجية على بيانات قادمة من المستخدمين كلياً.
  • Use Safe Formats: استبدل التنسيقات الثنائية بتنسيقات بيانات مسطحة بسيطة وآمنة لا تملك منطقاً تنفيذياً بداخلها مثل JSON أو YAML.

🔒 قيود فحص الأنواع (Look-Ahead Deserialization)

  • Class Whitelisting: إذا اضطررت لاستخدام فك التسلسل، قم بتطبيق التحقق المسبق (Look-ahead) لقصر الفك على قائمة بيضاء صارمة من الفئات المسموحة فقط وتدمير أي كائن غريب قبل بنائه في الذاكرة.
