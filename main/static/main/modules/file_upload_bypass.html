📤 File Upload Bypass — تخطي فلاتر رفع الملفات

منهجية رفع ملفات الـ Web Shell وتخطي فلاتر الامتداد ونوع الملف وحجمه للحصول على RCE

🛠️ Web Shells · Burp Suite · EXIF modification · Magic Bytes 📋 OWASP: Unrestricted File Upload | Remote Code Execution CVSS Impact: High → Critical (RCE)

📚 النظرية — كيف تحدث ثغرات رفع الملفات غير المقيدة

تحدث ثغرة **Unrestricted File Upload** عندما يسمح خادم الويب للمستخدمين برفع ملفات عشوائية إلى نظام الملفات الداخلي دون فلترة كافية لنوع الملف وامتداده وموقعه، ويسمح بتنفيذ هذه الملفات لاحقاً في المتصفح.

🔍 أنواع التحقق من الملفات (Filters Validation)

  • Client-Side Validation: فحص الامتداد بواسطة Javascript في المتصفح قبل إرسال الطلب (أسهل نوع للتخطي عبر Burp Suite).
  • Extension Blacklisting: حظر امتدادات محددة (مثل .php, .asp, .jsp). يمكن تخطيه باستخدام امتدادات بديلة أو تلاعب بالحروف.
  • MIME Type Validation: فحص ترويسة Content-Type في طلب الـ HTTP (مثال: image/png).
  • Magic Bytes (File Signature): قراءة أول بضع بايتات من الملف للتأكد من بنيته الحقيقية (مثل بايتات GIF89a).

🔥 الأثر الفعلي والـ Web Shell

الهدف النهائي للمهاجم هو رفع ملف **Web Shell** (كود برمجى بلغة السيرفر مثل PHP/ASP) يقبل الأوامر من خلال المتصفح وينفذها على نظام الخادم للوصول الكامل.

🔬 تشريح طرق التخطي (Bypass Techniques) خطوة بخطوة

1️⃣
MIME Type & Client-Side Bypass

إيقاف كود الـ JS في المتصفح أو التقاط الطلب في Burp Suite وتغيير ترويسة الـ Content-Type لتبدو كصورة مقبولة:

# Original request header captured: Content-Disposition: form-data; name="file"; filename="shell.php" Content-Type: application/x-php # Modify to look like image: Content-Type: image/png
2️⃣
Blacklist Bypass & Alternative Extensions

استخدام امتدادات نادرة أو متوافقة يقبلها نظام الحماية بينما يترجمها السيرفر كملف قابل للتنفيذ:

  • PHP Alternatives: .php3, .php4, .php5, .phtml, .phar
  • ASP Alternatives: .aspx, .cer, .asa
  • Double Extensions: shell.png.php أو shell.png.pHp5
  • Null Byte Injection (Older servers): shell.php%00.png
3️⃣
Magic Bytes Bypass & EXIF Injection

إدراج الـ Magic Bytes للصور في أول سطر للملف لخدع فلتر الفحص البرمجى، أو حقن كود الـ PHP بالكامل داخل خانة التعليق لملف الصورة الفعلي (EXIF Metadata) ورفعها بصيغة صورة:

# Magic Bytes for GIF in the first line of PHP file: GIF89a; <?php system($_GET['cmd']); ?> # Inject using exiftool: exiftool -Comment="<?php system(\$_GET['cmd']); ?>" image.png mv image.png shell.php.png

🛡️ الحماية والدفاع — تأمين رفع الملفات بالكامل

✅ حماية الكود البرمجي ونظام التخزين

  • Randomize File Names: تغيير اسم الملف المرفوع تلقائياً إلى قيمة عشوائية Hash (مثال: UUID) لقطع الطريق على المهاجم لمعرفة مسار الملف بعد الرفع.
  • Storage Isolation: تخزين الملفات المرفوعة في سيرفر معزول تماماً أو خدمات تخزين سحابية (S3 / Storage buckets) بدلاً من استضافتها على خادم التطبيق الرئيسي.

🔒 تحصين الخوادم ومنع التنفيذ

  • Non-Executable Permissions: إزالة صلاحيات التنفيذ (Execution Permissions) تماماً عن مجلد الرفع.
  • Web Server Configuration: منع السيرفر من معالجة ملفات الرفع برمجياً. مثال بملف .htaccess لمجلد الرفع:
    • # Disable script execution
<Files ~ "\.(php|php3|php4|php5|phtml|pl|py|jsp|asp|sh|cgi)$">
  ForceType text/plain
</Files>
