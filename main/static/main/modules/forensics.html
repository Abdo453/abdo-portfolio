📚 النظرية — فروع التحقيق الجنائي الرقمي والـ Artifacts الحيوية

يهدف التحقيق الجنائي الرقمي إلى جمع وحفظ وتحليل الأدلة الرقمية لتقديمها إلى الجهات القانونية أو فهم تفاصيل الاختراق (Root Cause Analysis).

🧠 أدلة الذاكرة العشوائية (Memory Forensics)

تعتبر الذاكرة العشوائية (RAM) كنزاً للمحقق لأنها تحتفظ بالبيانات المتطايرة (Volatile Data) التي تفقد عند إطفاء الجهاز. تشمل:

  • العمليات النشطة لحظة الاختراق (Active Processes).
  • مفاتيح التشفير وكلمات المرور الواضحة في الذاكرة.
  • اتصالات الشبكة النشطة وعناوين الـ IP المتصلة.

🖥️ بقايا نظام التشغيل (Windows Artifacts)

مواقع ثابتة في نظام ويندوز تسجل نشاط المستخدم والبرامج تلقائياً:

  • Prefetch Files (.pf): تسجل تاريخ وأول وقت لتشغيل أي برنامج وعدد مرات تشغيله.
  • Shimcache / Amcache: تتبع البرامج التي تم تشغيلها لضمان التوافقية (تساعد في تتبع تشغيل البرمجيات الخبيثة).
  • Windows Registry: يحتوي على إعدادات النظام وقوائم الأجهزة المتصلة بـ USB والبرامج التي تعمل تلقائياً (Run keys).

🔬 تشريح خطوات التحقيق — تحليل الذاكرة بـ Volatility

1️⃣
Memory Image Acquisition

أخذ لقطة كاملة من الذاكرة العشوائية للجهاز المصاب باستخدام أدوات مثل FTK Imager دون إعادة تشغيل الجهاز.

2️⃣
Listing Processes & Network connections

استخدام Volatility لتحليل الذاكرة وكشف العمليات والاتصالات النشطة:

# Identify profile and OS of the memory image (Volatility 2) volatility -f memory.raw imageinfo # List active processes (checking for hidden or suspicious processes) volatility -f memory.raw --profile=Win7SP1x64 pslist volatility -f memory.raw --profile=Win7SP1x64 pstree # Scan for network connections volatility -f memory.raw --profile=Win7SP1x64 netscan
3️⃣
Dumping Suspicious Process & Code Injection Check

استخراج ملف العملية المشبوهة لفحصه، ومسح الذاكرة بحثاً عن حقن كود خبيث (Process Injection):

# Scan for injected code/DLLs in memory volatility -f memory.raw --profile=Win7SP1x64 malfind # Dump a suspicious process by PID (e.g., PID 1337) volatility -f memory.raw --profile=Win7SP1x64 procdump -D ./dumped_files -p 1337
