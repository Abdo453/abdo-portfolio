📚 النظرية — كيف يعمل بروتوكول OAuth 2.0 والعيوب الكامنة

بروتوكول OAuth 2.0 هو معيار تفويض مفتوح يسمح للتطبيقات الخارجية بالوصول المحدود لموارد المستخدم دون مشاركة كلمة مرور حسابه.

🔄 تدفق مصادقة كود التفويض (Authorization Code Flow)

  • 1. Request: يوجه التطبيق (Client) المستخدم إلى سيرفر الـ OAuth (Provider) عبر رابط يحتوي على redirect_uri و scope.
  • 2. Consent: يوافق المستخدم على إعطاء الصلاحيات للتطبيق.
  • 3. Redirect: يعود الخادم بتوجيه المستخدم للتطبيق ومعه code مؤقت.
  • 4. Token Exchange: يرسل التطبيق الـ code والـ client_secret للسيرفر للحصول على Access Token النهائي.

⚠️ المشكلة الأمنية الأساسية

تعتمد سلامة التدفق كلياً على **دقة التحقق من عنوان التوجيه (Redirect URI)**. إذا تم تزوير هذا العنوان أو استغلال ثغرة إعادة توجيه مفتوحة فيه، يمكن للمهاجم اختطاف الـ `code` أو الـ `Access Token` للضحية فوراً.

🔬 تشريح الهجوم — ثغرات OAuth الشائعة خطوة بخطوة

1️⃣
Redirect URI Bypass (اختطاف كود التفويض)

إذا كان السيرفر يقبل مطابقة جزئية للرابط، يمكن للمهاجم تمرير رابط خادمه للحصول على كود التفويض:

# Original Request: https://provider.com/oauth/authorize?client_id=123&redirect_uri=https://victim.com/oauth # Attacker Bypass Attempt (Open Redirect on target): https://provider.com/oauth/authorize?client_id=123&redirect_uri=https://victim.com/oauth?redirect=https://attacker.com
2️⃣
OAuth CSRF (غياب معامل State)

إذا لم يرسل التطبيق معامل state عشوائي للتحقق منه عند عودة الطلب، يمكن للمهاجم إرسال رابط التفويض الخاص به للضحية ليربط حسابه الاجتماعي بحساب المهاجم بشكل غير مكتشف.

🛡️ الحماية والدفاع — تحصين تطبيق OAuth

✅ إعدادات الخادم والـ API

  • Exact Redirect URI Matching: فرض مطابقة حرفية وصارمة لعنوان التوجيه (Strict Exact Match) بدلاً من قبول النطاقات الفرعية أو المطابقة الجزئية.
  • Mandatory State Parameter: فرض استخدام معامل state عشوائي وفريد لكل مستخدم وجلسة للوقاية من هجمات CSRF.

🔒 إدارة رموز الجلسات

  • Short Authorization Code Lifespan: جعل مدة صلاحية كود التفويض قصيرة جداً (أقل من دقيقة) واستخدامه لمرة واحدة فقط.
  • PKCE (RFC 7636): استخدام إطار PKCE لحماية التطبيقات من اختطاف كود التفويض على الأجهزة الذكية والمتصفحات.
