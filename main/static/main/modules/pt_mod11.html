11. أمن جهة العميل (Client Side Security)

CORS, WebSockets, DOM XSS & CSP

الجانب النظري (Theory)

تعتمد تطبيقات الويب الحديثة بشكل كبير على متصفح العميل (Client-Side) في معالجة وعرض البيانات، وهذا يفتح آفاقاً جديدة من الثغرات. أمن جهة العميل يركز على استغلال الثغرات التي تنفذ في المتصفح مثل DOM-based XSS و التكوينات الخاطئة لـ CORS (Cross-Origin Resource Sharing) و WebSockets و CSP (Content Security Policy).

  • DOM XSS: تحدث عندما يقوم سكريبت العميل (JavaScript) بأخذ بيانات غير موثوقة (Source) ويمررها إلى دالة خطرة (Sink) مما يؤدي إلى تنفيذ كود برمجي خبيث داخل المتصفح.
  • CORS: آلية تسمح للخوادم بتحديد من يحق له طلب الموارد. الإعدادات الخاطئة (مثل السماح بأي Origin وإرسال Credentials) تتيح للمهاجم سرقة بيانات حساسة من مستخدمين آخرين.
  • WebSockets: بروتوكول يوفر اتصالاً ثنائي الاتجاه بين العميل والخادم. قد يكون عرضة لهجمات مثل CSWSH (Cross-Site WebSocket Hijacking) إذا لم يتم التحقق من Origin.
  • CSP: سياسة أمنية تهدف إلى منع XSS عبر تقييد مصادر تحميل السكريبتات. تخطي (Bypass) الـ CSP يعتبر جزءاً أساسياً في هجمات جهة العميل المتقدمة.

الاختبار اليدوي (Manual Testing)

لاكتشاف ثغرات جهة العميل، يجب التفاعل بعمق مع سلوك المتصفح:

  • لاكتشاف DOM XSS: استخدم أدوات المطور في المتصفح (DevTools). قم بتغيير الـ URL (مثل إدراج #payload) وراقب كيف يتعامل JavaScript معها باستخدام Breakpoints عند دوال مثل eval() أو innerHTML.
  • لاكتشاف CORS: عبر أداة Burp Suite، قم بتعديل ترويسة Origin: https://evil.com في الطلبات، ولاحظ إذا كان الخادم يرد بـ Access-Control-Allow-Origin: https://evil.com مع Access-Control-Allow-Credentials: true.
  • لاكتشاف WebSockets CSWSH: قم باعتراض طلب ترقية الاتصال (Upgrade Request) وغير قيمة Origin. إذا تم قبول الاتصال وتم تلقي رسائل حساسة بدون التحقق من الجلسة بقوة، فهناك ثغرة.
  • تخطي CSP: قم بقراءة الـ CSP Headers. ابحث عن السماحات المفرطة مثل unsafe-inline أو النطاقات القابلة للاستغلال (مثل استضافة JSONP) لتخطي السياسة الأمنية.

الأتمتة (Automation)

أتمتة الفحص تساعد في اكتشاف الثغرات في التطبيقات الضخمة:

  • DOM Invader: إضافة مدمجة في Burp Suite تكتشف بشكل تلقائي نقاط DOM XSS عبر حقن قيم عشوائية (Canaries) ومراقبة الـ Sinks.
  • CORScanner: أداة تعتمد على Python لاكتشاف الإعدادات الخاطئة في CORS والتحقق من إمكانية الاستغلال.
  • CSP Evaluator: أداة من Google لتحليل سياسات CSP وتحديد الثغرات فيها.

الأدوات والأوامر (Tools & Commands)

CORScanner

bash 
python3 corscanner.py -i targets.txt -t 50

CSP Analysis (curl)

bash 
curl -I -s https://target.com | grep "Content-Security-Policy"

Semgrep (Source Code Review for DOM XSS)

bash 
semgrep --config "p/javascript" path/to/source_code/

التقنية (Technique) الوصف (Description) مثال الاستغلال (Payload / Config)

DOM XSS Sink الدالة التي تنفذ الكود الخطير eval(), innerHTML, setTimeout()
