An interactive checklist to audit and harden Linux server environments. Your progress is saved automatically via localStorage.

Completed: 0/5 items

Use SSH Keys instead of raw passwords to protect against brute-force attacks. Edit /etc/ssh/sshd_config and set PasswordAuthentication no.

Audit SUID binaries and remove any unnecessary execution bits for ordinary users. Use chmod u-s [file].

Ensure /etc/shadow containing user hashes is readable only by root. Run chmod 600 /etc/shadow.

Enable UFW or iptables to block all traffic except required active services (e.g., ports 80, 443, 22). Use ufw default deny incoming.

Deactivate and stop non-essential network services like old FTP/Telnet ports. Use systemctl disable --now inetd.
