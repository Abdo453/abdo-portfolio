1. What is Privilege Escalation? / ما هي ترقية الصلاحيات؟

AR

ترقية الصلاحيات هي عملية استغلال ثغرة أمنية أو تكوين خاطئ (Misconfiguration) في نظام التشغيل للوصول إلى مستوى صلاحيات أعلى من المخصص للحساب الحالي (غالباً الانتقال من مستخدم عادي مثل www-data إلى مسؤول النظام root).

EN

Privilege escalation is the process of exploiting bugs or misconfigurations within an operating system to gain elevated access to resources that are normally protected from standard accounts (usually moving from a low-priv privilege like www-data to root).

2. SUID/SGID Exploits / استغلال صلاحية SUID

AR

عند تعيين بت SUID على ملف تنفيذي، يتم تشغيل هذا الملف بصلاحيات مالك الملف (غالباً root) بدلاً من المستخدم الذي قام بتشغيله. إذا كان هذا الملف يسمح بتنفيذ أوامر أو قراءة ملفات بطرق غير مباشرة، يمكن استخدامه لترقية الصلاحيات.

EN

When the SUID bit is set on an executable, it runs with the privileges of the file owner (usually root) rather than the user invoking it. If the binary supports command execution or interactive shells, it can be exploited to spawn a root shell.

Find all SUID binaries / البحث عن ملفات SUID 
find / -perm -4000 -type f 2>/dev/null

3. Sudo Privileges & GTFOBins / صلاحيات Sudo و GTFOBins

AR

يسمح أمر sudo للمستخدمين بتشغيل تطبيقات بصلاحيات root. يفحص المخترقون الصلاحيات المسموح بتشغيلها دون كلمة مرور (NOPASSWD) ثم يستعينون بموقع GTFOBins لمعرفة كيفية استغلال هذه البرمجيات (مثل awk, find, vim, bash) للحصول على shell.

EN

The sudo utility lets users execute commands with root permissions. Security auditors check allowed commands without password prompt via sudo -l, and then reference resources like GTFOBins to exploit applications (e.g. awk, find, vim) to drop into a root terminal.

List Sudo Privileges / عرض الصلاحيات المتاحة 
sudo -l
Example Awk Sudo Exploit / استغلال أداة Awk 
sudo awk 'BEGIN {system("/bin/sh")}'

4. Writable Cron Jobs / المهام المجدولة القابلة للكتابة

AR

الـ Cron Jobs هي سكربتات يتم تشغيلها دورياً (مثلاً كل دقيقة) بصلاحيات معينة. إذا كان السكربت الذي يقوم الـ Cron بتشغيله مملوكاً لـ root ولكنه قابل للكتابة (Writable) لمستخدم عادي، يمكننا تعديله لإضافة أوامر مثل منح صلاحية SUID للـ Bash أو إنشاء اتصال عكسي (Reverse Shell).

EN

Cron Jobs are automated tasks executed periodically (e.g., every minute). If a cron script runs as root but is writable by low-privilege users, attackers can inject malicious payloads (like adding SUID to bash or establishing a reverse shell) to achieve root status.

Inject SUID backdoor into writable script / حقن باب خلفي في سكربت Cron 
echo "chmod u+s /bin/bash" >> /path/to/writable_script.sh
