API Security — OWASP API Top 10

AR

API Security تركز على حماية واجهات برمجة التطبيقات (REST, GraphQL, gRPC) من الاستخدام الخاطئ. تعد الـ APIs هي العمود الفقري للتطبيقات الحديثة وتطبيقات الجوال، وتصنف ثغراتها وفق OWASP API Top 10 التي تشمل تخطي الصلاحيات، وغياب Rate Limit، والتسريب المفرط للبيانات.

EN

API Security focuses on protecting backend APIs (REST, GraphQL, gRPC) from abuse. As APIs are the backbone of modern web and mobile apps, they are cataloged under the OWASP API Top 10, covering broken object authorization, rate limiting failures, and excessive data exposure.

أشهر ثغرات الـ APIs / Top Vulnerabilities

BOLA (Broken Object Level Auth)

الوصول لبيانات كائن آخر بتغيير المعرّف (مثل IDOR).

BFLA (Broken Function Level Auth)

الوصول لوظائف إدارية (Admin APIs) بدون صلاحيات مناسبة.

Unrestricted Resource Consumption

غياب الـ Rate Limiting مما يسمح بإغراق الـ API بملايين الطلبات وتعطيلها.

Broken Object Property Level Auth

تعديل خصائص داخل الـ JSON Body لم يكن مسموحاً بتغييرها (Mass Assignment).

أوامر الاختبار / API Testing Commands

# Fuzzing API endpoints with Kiterunner
kiterunner scan https://api.target.com/v1 -w /opt/wordlists/routes.kb

# Discovering hidden parameters with Arjun
arjun -u https://api.target.com/v1/users -m GET

# Intercepting and testing with curl
curl -X DELETE -H "Authorization: Bearer USER_TOKEN" https://api.target.com/v1/posts/2001
