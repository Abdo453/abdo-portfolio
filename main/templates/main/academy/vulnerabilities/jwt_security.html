JWT Security — Attacks & Forgery

HIGH OWASP A02
AR

JWT (JSON Web Token) هو معيار مستخدم لتمثيل الادعاءات بين طرفين بشكل آمن كـ JSON Objects. تكمن المشاكل الأمنية في كيفية إعداد الخادم للتحقق من التوقيع (Signature)، مما يسمح بتزوير الرموز عبر هجمات Algorithm Confusion، وتخمين المفاتيح الضعيفة، وحقن رؤوس jku/jwk.

EN

JWT (JSON Web Token) is an open standard used to share claims securely between parties. Vulnerabilities arise from improper signature verification on the server side, allowing token forgery via algorithm confusion, weak key brute-forcing, and jku/jwk header injection.

أشهر ثغرات وهجمات JWT / Vulnerability Types

Algorithm Confusion (none alg)

تعديل خوارزمية التوقيع في الـ Header لـ none وحذف التوقيع بالكامل لخداع السيرفر لقبول التوكن المعدل.

Weak Signing Key Brute Force

تخمين المفتاح السري (Secret Key) المستخدم لتوقيع الرمز خارجياً عبر القواميس.

jku Header Injection

حقن رأس jku ليشير لنطاق يمتلكه المهاجم يحتوي على مفاتيح عمومية خبيثة يقبلها الخادم لتأكيد التوقيع.

kid (Key ID) Manipulation

تعديل معرّف المفتاح jku ليوجه لمسار متوقع أو ملف فارغ على السيرفر لتخطي فحص التوقيع.
