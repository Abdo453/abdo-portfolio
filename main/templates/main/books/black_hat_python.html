1. شبكات بايثون وبرمجة الـ Sockets (Network Programming)

أكواد برمجية كاملة وتفاعلية بالبايثون لإنشاء اتصال TCP Socket بالخادم الهدف وسحب الترويسات:

import socket

target_host = "10.10.10.25"
target_port = 9999

client = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
client.connect((target_host, target_port))
client.send(b"GET / HTTP/1.1\r\nHost: 10.10.10.25\r\n\r\n")
response = client.recv(4096)
print(response.decode())

2. اعتراض وتحليل حزم الشبكة (Raw Socket Sniffing)

كتابة أدوات قراءة الحزم الخام مباشرة من كرت الشبكة، وفك تشفير ترويسات الـ IP والـ ICMP باستخدام مكتبة struct بالبايثون.

3. بناء فاحص ومخمن ويب (Web Directory Fuzzer)

بناء أداة فحص وتخمين مسارات ويب متعددة الخيوط (Multi-threaded Directory Fuzzer) مستعينة بمكتبة requests وقاموس كلمات خارجي.

4. تطوير حصان طروادة (Trojan Command & Control)

شرح لكيفية بناء برنامج خبيث يتصل بالـ C&C server الخاص بك (مثلاً عبر مستودع GitHub مخفي) لتنزيل الأوامر البرمجية وتحديثات الاستغلال وتنفيذها على جهاز الضحية.

5. حقن العمليات بـ Ctypes والمسجلات

التعامل المباشر مع واجهات نظام ويندوز (Windows API) واستدعاء دوال الذاكرة مثل VirtualAllocEx و WriteProcessMemory لحقن الـ DLLs والـ shellcode بداخل العمليات الآمنة.

6. بنك الأسئلة والاختبارات التفاعلية الـ 10

الإجابة النموذجية: الـ `socket.SOCK_STREAM` يعبر عن بروتوكول TCP الموثوق والمهيأ لإنشاء اتصال مصافحة كامل. بينما `socket.SOCK_DGRAM` يعبر عن بروتوكول UDP غير الموثوق والسريع لإرسال حزم البيانات المستقلة دون الحاجة لإنشاء اتصال أولي مسبق.
الإجابة النموذجية: لأن حزم الشبكة تأتي كبيانات ثنائية خام (Raw bytes). مكتبة `struct` تتيح لنا تفكيك هذه البايتات وتحويلها لمتغيرات بايثون مفهومة (مثل أرقام وحروف) بناءً على صيغ معيارية ومحددة لبصمات الترويسات بالشبكة.
الإجابة النموذجية: يسمح بإنشاء وتشغيل مهام متعددة بالوقت نفسه (مثل إرسال 100 طلب للموقع بالثانية عبر خيوط معالجة منفصلة) بدلاً من الانتظار لانتهاء كل طلب بالتتابع، مما يقلل الوقت المستغرق للفحص الأمني بشكل كبير.
الإجابة النموذجية: مكتبة `ctypes` توفر إمكانية استدعاء دوال ومكتبات لغة C المكتوبة وجاهزة بالنظام (مثل DLLs الرسمية كـ kernel32.dll) والتعامل المباشر مع الذاكرة واستدعاء الوظائف الحساسة للويندوز مباشرة من كود بايثون.
الإجابة النموذجية: يقوم السكريبت المثبت بجهاز الضحية بالاتصال تلقائياً بمستودع GitHub خاص بالمهاجم باستخدام الـ API، ويسحب ملفات التكوين والأكواد المطلوب تشغيلها، ثم ينفذها ديناميكياً باستخدام دالة `exec` ويرفع النتائج للمستودع مرة أخرى، مما يصعب كشف الاتصال لأن حركة المرور تبدو كزيارة عادية لموقع GitHub الموثوق.
الإجابة النموذجية: دالة `exec()` تقوم بتشغيل وتفسير نصوص بايثون الممرة إليها ديناميكياً كأكواد برمجية صالحة. خطورتها تكمن في أنه إذا نجح مهاجم في حقن أي نص بداخل الدالة، فسيتم تشغيله بالكامل بصلاحيات السكريبت الحالي وقد يؤدي لتدمير النظام.
الإجابة النموذجية: تتحقق من مواصفات العتاد باستدعاء واجهات النظام (مثل حجم الرام الضعيف أو قلة عدد المعالجات)، أو مراقبة حركة الماوس ونقرات الكيبورد للتأكد من وجود تفاعل بشري حقيقي، وفي حال رصد خمول كامل تتوقف عن العمل فوراً لتجنب كشف سلوكها.
الإجابة النموذجية: تقوم بحجز مساحة ذاكرة جديدة وصالحة للقراءة والكتابة والتشغيل (PAGE_EXECUTE_READWRITE) بداخل عملية مستهدفة أخرى بعيدة، لتسمح لنا لاحقاً بكتابة وحقن الـ Shellcode بداخل هذه المساحة المحجوزة.
الإجابة النموذجية: يستعين بـ Hooking للوحة المفاتيح باستخدام دالة الـ Windows API وهي `SetWindowsHookEx` ليعترض كافة نقرات المفاتيح قبل إرسالها للتطبيق المستهدف، ويقوم بحفظها وتسجيلها بملف سري تمهيداً لتسريبها للمهاجم.
الإجابة النموذجية: تتيح `Scapy` صياغة الحزم يدوياً طبقة تلو طبقة (مثل `IP()/TCP()/"payload"`). هذا يمنح المهاجم قدرة كاملة على التلاعب بحقول الترويسات (مثل تغيير عنوان المرسل Spoofing أو تعديل الأعلام TCP flags) للقيام بعمليات فحص واختبار دقيقة جداً.
