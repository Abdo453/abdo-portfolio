💻 Phase 0: Environment Setup

تجهيز بيئة العمل ونظم التشغيل مع فحص مقارنة الأدوات البرمجية

🖥️ Operating Systems Selection

للقيام بفحص أمني متكامل، تحتاج إلى نظام تشغيل مستقر يوفر دعمًا لجميع مكتبات بايثون ومستودعات Go. التوزيعات الشائعة:

  • ✔ Kali Linux: المعيار الذهبي لمجتمع الهكر الأخلاقي، يضم أكبر عدد من الأدوات المثبتة مسبقًا.
  • ✔ Parrot OS Security: نظام خفيف وفعال، ممتاز للأجهزة ذات الموارد المتوسطة ويوفر أدوات حماية الخصوصية مسبقًا.
  • ✔ Ubuntu Custom build: يفضله صائدو الثغرات ذوو الخبرة لبناء بيئة سريعة ونظيفة تثبت فيها الأدوات يدوياً لضمان استقرار الخادم.

⚙️ Essential Tools Setup Command

تثبيت الأساسيات البرمجية لنظام Debian/Ubuntu:

sudo apt update && sudo apt install -y git python3 python3-pip golang curl snapd

🎯 Phase 1: Rules of Scope

فهم قواعد العمل ونطاق الفحص وتجنب المشاكل القانونية

📋 Rules of Engagement (قواعد العمل)

قبل إرسال أي حزم بيانات للهدف، يجب مراجعة وثيقة الـ Scope الخاص بالبرنامج أو الفحص الأمني وفهمه بالكامل لتفادي المشاكل القانونية.

✅ الأشياء المسموحة (In-Scope):

  • ✔ النطاقات المحددة صراحة (e.g. *.target.com).
  • ✔ واجهات برمجة التطبيقات (APIs) التابعة للهدف.
  • ✔ تطبيقات الهاتف المحمول (Android/iOS) المتاحة للفحص.

❌ الأشياء الممنوعة (Out-of-Scope):

  • ❌ هجمات حجب الخدمة (DDoS/DoS) التي تضر بالخدمة.
  • ❌ هجمات الهندسة الاجتماعية ضد موظفي الشركة (Phishing).
  • ❌ استهداف خوادم شركات أخرى مستضافة (e.g. AWS/Cloudflare) بشكل مباشر.

🌍 Phase 2: Subdomain Enumeration

جمع واستكشاف النطاقات الفرعية لتحديد مساحة الهجوم بالكامل

🔍 Subdomain Tools Comparison

الأداة السرعة أفضل استخدام الآلية
Subfinder سريع جداً (ثوانٍ) البحث السريع المعتمد على الـ APIs الخارجية Passive OSINT API querying
Amass متوسط (عميق) بناء خرائط أهداف كاملة وعمل تخمين ذكي Active + Passive DNS mapping
Assetfinder فوري أداة خفيفة بدون تكوين خارجي Scrapes crt.sh & certspotter

🛠️ Commands & Automation Pipelines

1. Subfinder basic execution:

subfinder -d target.com -all -silent -o subfinder_subs.txt

2. DNS validation using Httpx:

بعد جمع النطاقات، يجب فحص الحية منها ومعرفة حالتها البرمجية:

httpx -l subfinder_subs.txt -silent -status-code -tech-detect -title -o alive.txt

📡 Phase 3: Port Scanning & Services

فحص الخوادم واستخراج المنافذ والخدمات المفتوحة

🛡️ Service Scanning Command (Nmap)

فحص المنافذ والتعرف على الخدمات وإصداراتها على السيرفر المستهدف بشكل دقيق:

nmap -sC -sV -Pn -T4 -p- target.com -oN full_nmap_scan.txt
"دائماً استخدم خيار `-Pn` إذا كان السيرفر خلف جدار حماية يمنع الـ Ping، و `-sV` لتحديد الإصدارات الدقيقة."

📂 Phase 4: Directory Discovery (Fuzzing)

البحث عن المجلدات الحساسة والملفات والمسارات المخفية

⚙️ Tool Specifications: FFUF (Fuzz Faster U Fool)

What is FFUF? أداة سريعة جداً مكتوبة بلغة Go لتخمين مسارات الويب وعناوين URL والـ headers.

🎯 Why use FFUF?

  • سرعة خيالية تصل لـ آلاف الطلبات بالثانية.
  • تصفية ذكية للمحتوى بناء على الحجم أو السطور أو الكلمات.
  • تدعم Fuzzing الـ POST body والـ Headers بسهولة.

❌ Common Mistakes

  • عدم فلترة كود الاستجابة 404 أو الأحجام المتكررة مما ينتج عنه نتائج مزيفة ضخمة.
  • استخدام معدل طلبات (Threads) مرتفع جداً يؤدي لحجب عنوان IP الخاص بك أو تدمير خادم العميل.

🛠️ Commands & Wordlists

التخمين مع إخفاء الردود ذات الكود 404 وبأحجام محددة لتجنب التشويش:

ffuf -u https://target.com/FUZZ -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -fc 404 -fs 4242

أفضل القوائم (Best Wordlists): نوصي باستخدام SecLists/Discovery/Web-Content/raft-large-directories.txt أو OneListForAll.

🕷️ Phase 5: Web Crawling & Spidering

استخراج الروابط البرمجية والمسارات والـ endpoints من كود HTML

🧬 Katana (ProjectDiscovery) vs Hakrawler

Katana: يدعم محرك headless كروميوم، مما يجعله ممتازاً في قراءة الـ JavaScript الحديث (React, Angular, Vue) واستخراج الروابط منها بشكل تفاعلي.

Hakrawler: أداة سريعة للغاية مبنية على Regex سريع، ممتازة للأهداف البسيطة التي لا تعتمد على تحميل المحتوى بالـ client-side JS.

🛠️ Katana Advanced Command

katana -u https://target.com -jc -kf -d 3 -o katana_endpoints.txt

🕰️ Phase 6: Historical URLs & OSINT

البحث في أرشيف الويب التاريخي لاستعادة روابط حساسة قديمة

📜 Fetching Historical Endpoints (GAU)

سحب جميع الروابط المسجلة للهدف على Wayback Machine و CommonCrawl وغيرها:

gau target.com --threads 10 --subs | grep -E "\.(zip|bak|sql|config|env|conf)"

يبحث هذا الأمر في التاريخ عن ملفات النسخ الاحتياطية أو التكوين المنسية التي قد لا تظهر في الفحص الفعلي حاليًا.

🔑 Secrets extraction with SecretFinder

🎯 Arjun Parameter Fuzzer

🧬 Phase 9: API Hacking

تحليل واجهات برمجة التطبيقات (APIs) وفحص ثغرات تخطي التوثيق والـ endpoints

🛠️ Kiterunner: API Discovery Tool

تخمين مسارات الـ API بالاعتماد على قواعد بيانات الكلمات البرمجية المصممة للـ REST/GraphQL APIs:

kr scan https://target.com/api -w /usr/share/wordlists/kiterunner/routes-large.kite

🔑 Phase 10: Authentication & JWT Testing

فحص آليات التوثيق وتخطي التحقق الثنائي (MFA) وثغرات الـ JSON Web Tokens

🛠️ JWT Signature Bypass Cheat Sheet

أهم الهجمات على الـ JWT:

  • ✔ None Algorithm: تعديل قيمة الـ Header إلى "alg": "none" وإزالة التوقيع.
  • ✔ Key Confusion (HS256 vs RS256): استغلال استخدام نفس المفتاح العام لـ RS256 لتوقيع التوكن بـ HS256 متماثل.
  • ✔ Weak Secret Cracking: تخمين التوقيع محلياً باستخدام Hashcat أو jwt_tool.

🌐 Cross-Site Scripting (XSS)

Description: حقن أكواد جافا سكربت خبيثة في متصفح المستخدم عبر مدخلات لا يتم فحصها بشكل صحيح.

🚀 Common Payloads:

<script>alert(document.cookie)</script> <img src=x onerror=alert(1)> javascript:alert(1)

🛡️ Mitigation:

تطبيق الـ HTML entity encoding الصارم، وتفعيل الـ Content Security Policy (CSP).

💉 SQL Injection

Description: إرسال أوامر SQL خبيثة لقاعدة البيانات عبر مدخلات الويب للوصول لمعلومات سرية أو تعديلها.

🚀 Common Payloads:

' OR 1=1 -- ' UNION SELECT username, password FROM users -- admin' --

🛡️ Mitigation:

استخدام الـ Parameterized Queries (Prepared Statements) ومنع الاستعلام المباشر عبر تكوين سلاسل النصوص.

📡 Server Side Request Forgery (SSRF)

Description: إجبار سيرفر الويب على إرسال طلبات لصالح المهاجم إلى خدمات داخلية غير مصرح بالوصول إليها من الخارج.

🚀 Common Payloads:

http://127.0.0.1:80/admin http://169.254.169.254/latest/meta-data/ (AWS Metadata)

🛡️ Mitigation:

تفعيل الـ Whitelisting للنطاقات المسموح بالاتصال بها، وحجب الشبكات المحلية الداخليّة (RFC 1918).

🔑 Insecure Direct Object Reference (IDOR)

Description: إمكانية وصول المستخدم لبيانات مستخدمين آخرين بمجرد تغيير رقم الـ ID أو الاسم في الطلب لغياب التحقق من الصلاحيات.

🚀 Scenario:

طلب الرابط /api/v1/invoice?id=1001 وتعديله إلى ?id=1002 لرؤية فاتورة مستخدم آخر.

🛡️ Mitigation:

تنفيذ نظام تحقق صارم من الصلاحيات على مستوى الكود (Object-level Access Control) في كل طلب.

💻 Remote Code Execution (RCE)

Description: أخطر أنواع الثغرات، تتيح للمهاجم تنفيذ أوامر نظام التشغيل مباشرة على السيرفر المصاب.

🚀 Common Payloads:

; whoami | cat /etc/passwd `id`

🛡️ Mitigation:

تجنب استخدام الدوال البرمجية التي تنفذ أوامر النظام مثل (system, exec, eval) وتعقيم مدخلات المستخدم بالكامل.

⚙️ Phase 16: Automated Tool Chains

بناء سلاسل الأدوات وخطوط الأنابيب البرمجية لأتمتة الاستكشاف

🔗 Automation Pipelines

المخترقون المحترفون لا يشغلون الأدوات بشكل منفصل، بل يربطون مخرجات كل أداة بمدخلات الأداة التالية.

1. JS Recon Pipeline:

الزحف -> تجميع ملفات JS -> كشف الأسرار والثغرات التلقائية:

katana -u https://target.com -jc | grep -E "\.js$" | xargs -I %% python3 SecretFinder.py -i %% -o cli

2. Subdomains vulnerability scanner chain:

جمع النطاقات الحية -> فحص الثغرات الأمنية المعروفة (CVEs) تلقائياً:

subfinder -d target.com -silent | httpx -silent | nuclei -t cves/ -severity critical,high
