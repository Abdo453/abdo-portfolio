📚 النظرية — تشريح رمز JWT وعلاقة التوقيع بالأمان

الـ JSON Web Token (JWT) هو معيار مفتوح (RFC 7519) لنقل البيانات بين خادم وعميل بشكل مضغوط وآمن. يتكون الرمز من ثلاثة أجزاء مفصولة بنقاط (.) ومقروءة بصيغة Base64URL:

1. Header (الترويسة)

يحتوي على نوع الرمز وخوارزمية التوقيع.

{
  "alg": "HS256",
  "typ": "JWT"
}
2. Payload (البيانات)

يحتوي على البيانات الفعلية (Claims) مثل معرف المستخدم وصلاحياته.

{
  "user_id": 1337,
  "role": "user"
}
3. Signature (التوقيع)

ينتج عن تشفير الجزئين السابقين باستخدام مفتاح سري للتأكد من عدم التلاعب.

HMACSHA256(
  base64Url(header) + "." +
  base64Url(payload),
  secret_key
)

🔬 تشريح الهجوم — مسار تجاوز المصادقة (Auth Bypass)

1️⃣
Algorithm Confusion Attack (الخلط بين الخوارزميات)

إذا كان السيرفر يستخدم خوارزمية مفتاح غير متماثل (RS256 - مفتاح عام للتأكيد ومفتاح خاص للتوقيع)، يقوم المهاجم بتغيير خوارزمية التوكن إلى متماثل (HS256) واستخدام المفتاح العام (المتاح للجميع) كسرّ متماثل للتوقيع. السيرفر الضعيف سيستخدم المفتاح العام كسرّ للتحقق، مما يتيح تزوير التوكن بالكامل.

2️⃣
None Algorithm Attack (تخطى التوقيع)

تعديل ترويسة التوكن لتغيير الخوارزمية إلى none أو None، وحذف جزء التوقيع تماماً مع الاحتفاظ بالنقطة الأخيرة. إذا كان السيرفر يعتمد على الترويسة لتحديد نوع الفحص دون التحقق من بيئة التشغيل، فسيقبل التوكن دون التحقق من التوقيع.

3️⃣
Header Parameter Injection (JWK/JKU/KID)

حقن معاملات الترويسة للتحكم في المفتاح المستخدم للتحقق:

  • JWK (JSON Web Key): تضمين مفتاح المهاجم العام في الترويسة مباشرة.
  • JKU (JSON Web Key Set URL): الإشارة لرابط خارجي خاص بالمهاجم يحتوي على المفتاح العام.
  • KID (Key ID): استغلال اسم الملف للوصول لملف ثابت مثل /dev/null بحيث يتم التحقق دائماً ضد قيمة فارغة أو ثابتة.

⚡ ترسانة الهجمات والـ Payloads

None Algorithm Signature Bypass

تغيير alg إلى none وحذف التوقيع:

# Original token: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyX2lkIjoxMzM3LCJyb2xlIjoidXNlciJ9.signature # Modified token (None attack): eyJhbGciOiJub25lIiwidHlwIjoiSldUIn0.eyJ1c2VyX2lkIjoxMzM3LCJyb2xlIjoiYWRtaW4ifQ.

KID Directory Traversal

استغلال معامل kid للإشارة لملف ثابت:

# Target Header: { "alg": "HS256", "typ": "JWT", "kid": "../../../../../dev/null" } # Sign the token against an empty string/null secret

🛡️ الحماية والدفاع — تأمين استخدام JWT

✅ تأمين خوارزمية التحقق

  • Enforce Algorithm: يجب فرض خوارزمية التحقق برمجياً في السيرفر بشكل ثابت (مثال: استخدام jwt.verify(token, secret, { algorithms: ['HS256'] })) بدلاً من الوثوق بالخوارزمية المحددة في الترويسة.
  • Block "none" Algorithm: رفض أي توكن يستخدم خوارزمية none بشكل صريح ونهائي.

🔒 تأمين المفاتيح والمقاييس

  • Strong Secrets: استخدام كلمات مرور معقدة وطويلة جداً للمفاتيح المتماثلة لتجنب الكسر أوفلاين.
  • Validate URLs: تصفية والتحقق من صحة معامل jku بحيث لا يقبل السيرفر جلب ملفات المفاتيح إلا من نطاق معتمد (Strict Whitelist).
