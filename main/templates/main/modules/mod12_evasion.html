📚 النظرية — مبادئ تخطي أنظمة الدفاع عن الشبكات

تعتمد أنظمة الدفاع مثل **IDS (Intrusion Detection Systems)** و **Firewalls** على فحص حركة البيانات بحثاً عن أنماط مشبوهة (Signatures) أو سلوك غير معتاد (Anomalies). يهدف التخطي إلى تعديل شكل أو طريقة إرسال الحزم بحيث تبدو طبيعية ومصرحة لنظام الحماية، بينما يتمكن خادم الهدف من إعادة تجميعها وفهمها بشكل ضار.

📡 تقنيات تخطي الـ IDS/IPS

  • Packet Fragmentation: تقسيم حزمة الهجوم إلى حزم صغيرة جداً. لا يستطيع الـ IDS مطابقة الـ Signature لعدم قدرته على إعادة التجميع في الوقت الفعلي، بينما يقوم نظام تشغيل الهدف بتجميعها تلقائياً.
  • Obfuscation: ترميز البيانات (مثال بـ Base64 أو Hex) لإخفاء الكلمات المفتاحية الخطرة من فلاتر الفحص.
  • Session Splitting: تقسيم طلب الاتصال عبر مسارات متعددة لخداع أنظمة الفحص.

🔥 تقنيات تخطي جدران الحماية (Firewall Evasion)

  • Port Tunneling: تشفير حركة البيانات ونقلها عبر منافذ مصرح بها مسبقاً (مثل نقل حركة الـ SSH داخل منفذ الـ HTTP/HTTPS 443).
  • IP Spoofing / Decoy Scanning: إرسال طلبات فحص ممزوجة بطلبات وهمية من عناوين IP متعددة لإخفاء الهوية الحقيقية للمهاجم.

🔬 تشريح مسار التخطي — فحص الشبكة مع إخفاء الهوية

1️⃣
Decoy Scanning (الفحص بالتمويه)

تشغيل الفحص عبر إدراج عناوين IP وهمية (Decoys) لتبدو حزم المهاجم ضائعة وسط آلاف الطلبات الأخرى في سجلات جدار الحماية.

# Nmap scan with 4 decoys and random IP spoofing nmap -D RND:4 192.168.1.50
2️⃣
Fragmentation Attacks (تجزئة الحزم)

إجبار Nmap على تقسيم حزم الفحص إلى أجزاء بحجم 8-bit لتفادي مطابقة الـ Signatures الكلاسيكية للـ IDS.

# Fragment packets during port scan nmap -f 192.168.1.50
3️⃣
DNS / ICMP Tunneling (النفق الرقمي)

تغليف بيانات الاتصال المحظورة وإرسالها داخل طلبات استعلام DNS (TXT records) أو حزم Ping غير المحظورة لتجاوز جدران الحماية بالكامل.

🛡️ الحماية والدفاع — تأمين الشبكات ضد التخطي

✅ تأمين أنظمة كشف التسلل (IDS hardening)

  • IP Reassembly: تهيئة أنظمة الـ IDS لإعادة تجميع الحزم المجزأة (Packet Reassembly) وفحصها بالكامل قبل السماح بمرورها للهدف.
  • Behavioral Analysis: استخدام الذكاء الاصطناعي وتحليل السلوك لكشف أنماط Decoy Scans التي تفشل في محاكاتها الفلاتر التقليدية.

🔒 قواعد جدران الحماية الصارمة

  • Deep Packet Inspection (DPI): فحص محتوى الحزمة الفعلي وليس فقط رأس المنفذ لمنع هجمات الـ Tunneling.
  • Block Unused Protocols: إيقاف وحظر الاتصالات غير المستخدمة كلياً مثل تعطيل حزم ICMP الصادرة والواردة غير الضرورية.
