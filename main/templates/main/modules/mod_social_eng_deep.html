📚 مبادئ Cialdini — الأسس النفسية للهندسة الاجتماعية

حدد عالم النفس Robert Cialdini في كتابه "Influence: The Psychology of Persuasion" ستة مبادئ رئيسية للإقناع — يستغلها منفذو الهندسة الاجتماعية لبناء سيناريوهاتهم:

1️⃣ Authority (السلطة)

الناس يطيعون من يبدو في موقع سلطة. المهاجم ينتحل صفة: CEO، IT Support، محقق حكومي، مدقق ضرائب.

"أنا من قسم أمن المعلومات، نحتاج كلمة مرورك فوراً للتحقق من حسابك المخترق"

2️⃣ Urgency (الإلحاح)

الضغط الزمني يمنع التفكير النقدي. "حسابك سيُغلق خلال 24 ساعة!" أو "عرض ينتهي اليوم!"

"تحديث أمني عاجل — انقر الآن وإلا سيُعطَّل حسابك المصرفي"

3️⃣ Scarcity (الندرة)

الفرص النادرة تدفع للتصرف السريع. "فرصة عمل حصرية لـ 3 أشخاص فقط" أو "وصول محدود للنظام".

"آخر 2 مقاعد في البرنامج — سجّل الآن باستخدام بيانات حسابك"

4️⃣ Social Proof (الإثبات الاجتماعي)

الناس يتبعون ما يفعله الآخرون. "1000 موظف وقّعوا الوثيقة بالفعل".

"فريقك بأكمله حدّث بياناته — أنت الوحيد المتبقي"

5️⃣ Liking (التعاطف)

نساعد من نحبه أو نثق به. المهاجم يبني علاقة، يتشارك اهتمامات، ينتحل صفة صديق مشترك.

"أنا أحمد زميلك السابق في [شركة X]، أحتاج مساعدة صغيرة"

6️⃣ Reciprocity (المعاملة بالمثل)

إعطاء شيء لاستقبال شيء. "أرسلت لك هدية رقمية / قسيمة خصم، كل ما نحتاجه هو..."

"لقد أرسلنا لك مكافأة $100 — فعّلها بتسجيل الدخول هنا"

🔬 تشريح حملة Phishing — من البداية للنهاية

1️⃣
Target Reconnaissance (استطلاع الهدف)

جمع المعلومات: LinkedIn (الوظائف، الهيكل التنظيمي)، Hunter.io (الإيميلات)، WHOIS، منصات التواصل الاجتماعي، تقارير الشركة السنوية. الهدف: بناء pretexting موثوق.

# Email discovery hunter.io → search company domain theHarvester -d company.com -b google,linkedin # LinkedIn OSINT # Search: site:linkedin.com/in "company name" "IT Manager" # Find email format # Try: firstname.lastname@ / first.last@ / f.lastname@
2️⃣
Pretext Building (بناء الذريعة)

تصميم القصة المقنعة: هل هو IT helpdesk؟ مدقق مالي؟ مورد؟ شريك؟ يجب أن يتطابق مع احتياج حقيقي للضحية ويستخدم أسماء حقيقية (CEO، زميل، مدير).

3️⃣
Infrastructure Setup (البنية التحتية)

نطاق مشابه (Typosquatting): paypa1.com بدل paypal.com، أو company-helpdesk.com. إعداد SSL (Let's Encrypt — الـ HTTPS لا يعني الأمان!)، GoPhish server، صفحة Clone.

# Setup GoPhish phishing server # Download from: https://github.com/gophish/gophish/releases ./gophish # Start server on :3333 (admin) and :80 (phishing) # Access admin panel: https://localhost:3333 # Default credentials: admin / gophish # In GoPhish dashboard: # 1. Sending Profiles → configure SMTP # 2. Email Templates → craft phishing email # 3. Landing Pages → clone target page # 4. Campaigns → set targets and launch
4️⃣
Credential Harvesting (حصاد البيانات)

الضحية تدخل بياناتها في الصفحة المزيفة. GoPhish يسجل كل نقرة وإدخال. evilginx2 يعمل كـ Reverse Proxy ويلتقط حتى الـ Session Cookies — يتجاوز 2FA!

# GoPhish tracks: # - Email opened (pixel tracking) # - Link clicked # - Credentials submitted # - Reports dashboard with timeline # evilginx2 — intercepts cookies even with 2FA: # Creates reverse proxy between victim and real site # Captures session tokens in real-time evilginx # Start evilginx2 # phishlets list → see available phishlets # phishlets enable microsoft → enable Microsoft phishlet # lures create microsoft → create lure URL

⚡ أنواع هجمات Phishing — دليل شامل

النوعالتعريفالقناةمستوى التخصيصمثال
Phishing هجوم جماعي لأعداد كبيرة البريد الإلكتروني منخفض — عام "حسابك PayPal معلق"
Spear Phishing هجوم مستهدف لشخص/منظمة محددة البريد الإلكتروني عالي — معلومات شخصية إيميل باسم المدير للمحاسب
Whaling Spear Phishing يستهدف كبار المسؤولين البريد الإلكتروني عالي جداً — CEOs, CFOs طلب تحويل مالي عاجل من "CEO"
Vishing Voice Phishing عبر الهاتف الهاتف/VoIP متوسط — انتحال هوية "أنا من البنك، حسابك تعرض للاختراق"
SMiShing SMS Phishing عبر الرسائل SMS منخفض-متوسط "طردك في الجمارك، ادفع $3.99"
BEC Business Email Compromise البريد الإلكتروني عالي جداً — تحليل عميق تحويل ملايين لحساب المهاجم
Pharming تزوير DNS لتوجيه لموقع مزيف DNS Poisoning لا يحتاج تفاعل زيارة bank.com تذهب لموقع مزيف
Baiting إغراء بمحتوى/USB ملغوم مادي/رقمي منخفض USB drive "مجاني" في موقف السيارات

🏆 دراسات حالة حقيقية

💸 Case 1: BEC هجوم — خسارة $47 مليون (Ubiquiti Networks 2015)

ما حدث: انتحل المهاجمون صفة مسؤولين تنفيذيين في شركة Ubiquiti Networks وأرسلوا إيميلات للقسم المالي تطلب تحويل أموال لحساب خارجي.

  • الإيميلات بدت من نطاق شبيه: ubiquiti.co بدلاً من ubiquiti.com
  • التحويلات تمت على مدى عدة أسابيع لحسابات في هونج كونج وأوروبا
  • لم يُكتشف الأمر إلا بعد $47 مليون!
الدرس: التحقق الإضافي (خارج الإيميل) لأي طلب تحويل مالي ضروري، حتى لو بدا من CEO.
🦠 Case 2: COVID-19 Phishing Campaign 2020

ما حدث: خلال الوباء، انتشرت حملات Phishing ضخمة تنتحل صفة: منظمة الصحة العالمية (WHO)، وزارة الصحة، برامج دعم حكومي.

  • نسبة النجاح: أعلى من الطبيعي بسبب الـ Urgency والـ Authority المزدوجة
  • إيميلات تحمل "تعليمات الحجر الصحي" أو "نتيجة تحليل كوفيد"
  • Payloads: Emotet، Trickbot، Ransomware
الدرس: الأزمات العالمية = فرصة ذهبية للـ SE. يجب رفع وعي الموظفين بشكل استباقي في الأوقات الحرجة.

🛡️ الحماية — Defense Against Social Engineering

📧 Email Authentication (SPF/DKIM/DMARC)

# SPF Record (in DNS TXT): # Defines which servers can send email for your domain v=spf1 include:_spf.google.com ip4:203.0.113.0/24 ~all # DKIM — cryptographic signature on each email # Setup via email provider, verify: nslookup -type=TXT selector._domainkey.company.com # DMARC Record — policy for SPF/DKIM failures v=DMARC1; p=reject; rua=mailto:reports@company.com; pct=100 # Check email security with: # https://mxtoolbox.com/SuperTool.aspx

🎓 Security Awareness Training

  • تدريب دوري على التعرف على Phishing
  • محاكاة حملات Phishing داخلية (GoPhish)
  • سياسة التحقق من الهوية عبر قناة ثانية
  • لا تفتح مرفقات غير متوقعة
  • تحقق من URL قبل الإدخال (hover before click)
  • سياسة إبلاغ واضحة عن الرسائل المشبوهة

🔒 تقنيات الحماية الإضافية

  • MFA/2FA: يقلل تأثير Credential Theft
  • Hardware Keys (FIDO2): يتجاوز evilginx2-style attacks
  • Anti-Phishing DNS: Cloudflare Gateway، Cisco Umbrella
  • Email Gateway: فحص المرفقات والروابط
  • Browser Isolation: تشغيل المواقع المشبوهة في sandbox

📋 Social Engineering Assessment Checklist

  • ☑ SPF, DKIM, DMARC configured & enforced
  • ☑ Email gateway anti-spoofing enabled
  • ☑ MFA enabled for all users
  • ☑ Annual phishing simulation conducted
  • ☑ Incident response for phishing defined
  • ☑ Callback verification for wire transfers
  • ☑ Executive impersonation policies

🧪 تحدي — اختبر معلوماتك في SE

❓ أي مبدأ Cialdini يستغله هذا السيناريو: "حسابك سيُغلق خلال 24 ساعة!"
✅ الإجابة: Urgency (الإلحاح)

الضغط الزمني يمنع الضحية من التفكير النقدي والتحقق من صحة الطلب.

❓ ما الفرق بين Phishing و Spear Phishing؟
✅ الإجابة:

Phishing = هجوم جماعي عشوائي لأعداد كبيرة. Spear Phishing = هجوم مستهدف لشخص/مؤسسة محددة مع معلومات شخصية لزيادة المصداقية.

❓ لماذا evilginx2 يستطيع تجاوز 2FA بينما GoPhish لا يستطيع؟
✅ الإجابة:

GoPhish يلتقط فقط ما يُدخله المستخدم في الصفحة المزيفة (credentials). evilginx2 يعمل كـ Reverse Proxy حقيقي — يمرر الطلبات للموقع الأصلي ويلتقط الـ Session Cookies بعد اكتمال المصادقة بما فيها الـ 2FA token.
