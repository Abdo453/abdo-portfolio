07. Web Testing (OWASP Top 10)

اختبار اختراق تطبيقات الويب واكتشاف ثغرات OWASP Top 10 مثل SQLi و XSS

[Theory] الإطار النظري: اختبار تطبيقات الويب

تُعد تطبيقات الويب الواجهة الأمامية الأساسية للعديد من الأنظمة الحيوية، مما يجعلها هدفاً رئيسياً للمهاجمين. يُركز اختبار اختراق تطبيقات الويب (Web Application Penetration Testing) على تقييم أمان التطبيقات والمواقع الإلكترونية من خلال محاكاة هجمات حقيقية لاكتشاف الثغرات وتأمينها قبل استغلالها.

يستند المنهج القياسي في اختبار الويب إلى قائمة OWASP Top 10، وهي وثيقة توعوية قياسية تُمثل أبرز المخاطر الأمنية لتطبيقات الويب. من أهم هذه الثغرات:

  • Injection (مثل SQLi): حيث يتم دمج بيانات غير موثوقة في استعلامات قاعدة البيانات أو الأوامر النظامية، مما يؤدي إلى تنفيذ أوامر غير مصرح بها.
  • Cross-Site Scripting (XSS): إدراج سكربتات ضارة في صفحات الويب التي يعرضها المستخدمون الآخرون، مما يسمح بسرقة الجلسات (Sessions) أو تنفيذ إجراءات نيابة عن المستخدم.
  • Broken Access Control: فشل النظام في فرض قيود صارمة على ما يُسمح للمستخدمين الموثقين بالقيام به.
  • Security Misconfiguration: الإعدادات الافتراضية غير الآمنة، وترك خدمات غير ضرورية تعمل، أو عرض رسائل خطأ تفصيلية.

تتطلب هذه العملية فهماً عميقاً لبروتوكول HTTP(S)، وإدارة الجلسات (Session Management)، وآليات التحقق من المدخلات (Input Validation).

[Manual Testing] الفحص اليدوي

لا غنى عن الفحص اليدوي في اختبار تطبيقات الويب، حيث لا تستطيع الأدوات الآلية فهم السياق (Context) (المنطق الإداري للتطبيق) بالشكل الصحيح.

  • اعتراض الطلبات (Traffic Interception): استخدم بروكسي مثل Burp Suite أو OWASP ZAP لاعتراض وتعديل طلبات HTTP واستجاباتها بحثاً عن سلوكيات غير متوقعة.
  • اختبار ثغرات SQLi: إدخال رموز خاصة مثل ' أو " أو -- أو WAITFOR DELAY '0:0:5' في حقول الإدخال، مسارات URL، وترويسات HTTP لمراقبة استجابات قواعد البيانات (أخطاء، أو تأخيرات زمنية).
  • اختبار ثغرات XSS: محاولة حقن حمولات (Payloads) برمجية مثل <script>alert(1)</script> أو "><img src=x onerror=prompt(document.cookie)> ومراقبة أين وكيف ينعكس هذا الإدخال في كود HTML الخاص بالصفحة بدون تشفير سليم (Encoding).
  • اختبار التحكم في الوصول (IDOR): التلاعب بالمعرفات الرقمية أو الحرفية في مسارات URL أو المتغيرات (مثال: user_id=123 إلى user_id=124) للوصول إلى بيانات مستخدمين آخرين بصلاحيات أفقية أو عمودية.

[Automation] أدوات الأتمتة والسكربتات

تُسهم الأتمتة في تسريع عملية الاكتشاف، خاصة للثغرات الشائعة والسطحية.

  • Burp Suite Active Scanner: أداة قوية مدمجة في النسخة الاحترافية من Burp تقوم بفحص آلي شامل لنقاط الإدخال.
  • SQLmap: الأداة الأقوى عالمياً في أتمتة اكتشاف واستغلال ثغرات حقن قواعد البيانات (SQL Injection) بمختلف أنواعها.
  • Nuclei: أداة فحص تعتمد على قوالب (Templates) حديثة وسريعة لاكتشاف الثغرات المعروفة والتكوينات الخاطئة.
  • FFuF / Gobuster: لأتمتة البحث عن الملفات والمجلدات المخفية (Directory Bruteforcing) واكتشاف مسارات غير مصرح بها.

[Tools] الأوامر والأدوات التطبيقية

فحص ثغرات SQL Injection باستخدام SQLmap

sqlmap -u "http://target.com/page.php?id=1" --dbs --random-agent --batch

فحص ثغرات XSS باستخدام XSStrike

python xsstrike.py -u "http://target.com/search.php?q=test" --crawl

البحث عن المسارات باستخدام FFuF

ffuf -w wordlist.txt -u http://target.com/FUZZ -mc 200,301,302,403

الفحص السريع باستخدام Nuclei

nuclei -u http://target.com -t cves/ -t vulnerabilities/

[Lab] بيئات التدريب العملي

  • PortSwigger Web Security Academy: أفضل منصة مجانية لتعلم ثغرات الويب من مبدعي Burp Suite.
  • HackTheBox (HTB): مسارات مثل Bug Bounty Hunter، وأجهزة مخصصة لتطبيقات الويب.
  • TryHackMe (THM): غرف OWASP Top 10 و Web Fundamentals.
  • OWASP Juice Shop: تطبيق ويب حديث ومليء بالثغرات مصمم للتدريب والأبحاث.

[Cheat Sheet] المرجع السريع لثغرات الويب

نوع الثغرة (Vulnerability) الفحص اليدوي (Manual Test) التأثير (Impact)
SQL Injection إضافة ' أو " أو محاولة ORDER BY 1-- تسريب قواعد البيانات، تجاوز تسجيل الدخول، تنفيذ أوامر (RCE).
Reflected XSS إدخال <script>alert(document.cookie)</script> في حقول البحث سرقة الجلسات (Cookies)، إعادة التوجيه لصفحات خبيثة، اختطاف حسابات.
Stored XSS حفظ الكود الخبيث في التعليقات أو الملف الشخصي إصابة كل مستخدم يزور الصفحة التي تحتوي على التعليق أو الملف.
IDOR تعديل الأرقام المعرفة للموارد في الروابط أو الطلبات المخفية الوصول غير المصرح به لبيانات المستخدمين الآخرين.
LFI (Local File Inclusion) تغيير المعامل إلى مسار مثل ../../../../etc/passwd قراءة الملفات الحساسة للنظام الذي يستضيف التطبيق.
