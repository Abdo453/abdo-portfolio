Level 1: المتغيرات والشروط

أول خطوة في الهاكينج هي تخزين البيانات والتحقق منها. في هذا المستوى سنتعلم كيف نُعرّف متغيرات (Variables) وكيف نتخذ قرارات باستخدام if.

المطلوب في المحرر (Editor):

يوجد كود على اليسار يعرّف متغيراً يسمى target_ip و port.

🎯 Challenge 1: Port Validation

أضف جملة if برمجية تتحقق مما إذا كان المنفذ port أكبر من 0 وأصغر من أو يساوي 65535.

- إذا كان صحيحاً: اطبع Valid Port

- إذا كان خاطئاً: اطبع Invalid Port

ثم اضغط Run Code بالأعلى.

Level 2: التكرار والدوال

الهاكر لا يقوم بفحص الأهداف يدوياً أبداً! نستخدم الحلقات (Loops) لتمرير الأهداف على دوال الفحص (Functions) أوتوماتيكياً.

🎯 Challenge 2: Network Sweeper

المطلوب كتابة حلقة for تمر على الأرقام من 1 إلى 5 باستخدام الدالة range(1, 6).

في كل دورة للحلقة، اطبع النص التالي بالضبط: Ping 192.168.1.X (حيث X هو الرقم الحالي في الحلقة).

Level 3: استخراج الكنوز بالـ Regex

نتائج أدواتنا تكون ضخمة ومعقدة. استخراج الآيبيهات أو الإيميلات يتطلب سحراً يسمى التعبيرات النمطية (Regex).

🎯 Challenge 3: IP Extractor

في المحرر، قمنا بتجهيز متغير log_data يحتوي على نص طويل.

استخدم الدالة re.findall مع نمط الـ IP لاستخراج الأيبيهات من النص وطباعتها (سواء كقائمة أو فرادى).

Level 4: الشبكات (HTTP)

التحدث مع السيرفرات وقراءة الـ Headers والـ Status Codes هي أساس الـ Web Hacking.

🎯 Challenge 4: API Status Checker

باستخدام مكتبة urllib.request المدمجة، قم بإرسال طلب (Request) إلى الرابط: https://httpbin.org/status/200

قم بطباعة حالة الاستجابة (Status Code). يجب أن تطبع 200.

Level 5: الأتمتة والسرعة

استخدام خيوط المعالجة المتزامنة (Multithreading) يجعل أدواتك تعمل بلمح البصر، مثل أداة Dirb أو Subfinder.

🎯 Challenge 5: Threaded Brute-Force

اكتب دالة crack(pwd) تطبع Cracking {pwd}...

ثم استخدم ThreadPoolExecutor من مكتبة concurrent.futures لعمل map لقائمة كلمات مرور ['admin', '12345', 'root'] على تلك الدالة.

Level 6: مشروع التخرج الاحترافي

أنت الآن جاهز لدمج المتغيرات، الدوال، الـ Threads، والشبكات لإنتاج أدوات فعلية مثل Port Scanner.
