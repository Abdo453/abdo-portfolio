اختراق الـ Active Directory والوصول للتحكم الكامل

Active Directory Hacking: From Guest to Domain Admin

CRITICAL Bounty: Internal Audit

Overview / نظرة عامة

AR

تعتمد أغلب الشركات الكبرى على Active Directory (AD) لإدارة هويات وصلاحيات الموظفين والأجهزة. يوضح هذا التقرير سلسلة اختراق بدأت من مستخدم عادي بالشركة وانتهت بالتحكم بالـ Domain Controller بالكامل.

EN

Enterprise networks rely on Active Directory (AD) to handle authentication. This writeup describes a full attack chain starting from low-privileged workstation access to gaining Domain Admin rights.

Attack Path / سلسلة الهجوم

  1. اكتشاف حساب عادي بكلمة مرور ضعيفة عبر هجمات التخمين الموجهة (Password Spraying).
  2. تشغيل أداة BloodHound لرسم خريطة شبكة AD ومعرفة مسارات ترقية الصلاحيات.
  3. استغلال ثغرة Kerberoasting لاستخراج وتشفير تذاكر الخدمة (TGS) وكسرها للحصول على كلمة مرور حساب الخدمة.
  4. استغلال صلاحيات حساب الخدمة المرتفعة على مجلدات النسخ الاحتياطي لاستخراج تفاصيل Domain Controller.
  5. حقن تذكرة Kerberos ذهبية (Golden Ticket) للوصول للتحكم الكامل.

Exploit Commands / أوامر الفحص والاستخراج

Kerberoasting with GetUserSPNs / استخراج تذاكر Kerberoast 
GetUserSPNs.py -request -dc-ip 10.10.10.50 domain.local/user:password
TGS Ticket Cracking with Hashcat / كسر التذاكر المستخرجة 
hashcat -m 13100 kerberoast_hashes.txt wordlist.txt

Remediation / التخفيف والحماية

AR

احرص على استخدام كلمات مرور طويلة ومعقدة لحسابات الخدمة (Service Accounts)، وفعل ميزات الأمان مثل Kerberos AES Encryption، وراقب التغييرات غير المألوفة في مجموعات المسؤولين.

EN

Enforce long, complex passwords on service accounts (GMSAs), restrict access to sensitive AD objects, and enable Kerberos AES encryption standard.
