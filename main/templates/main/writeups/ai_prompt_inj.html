AI Security: Prompt Injection and Prompt Leaking in LLMs

MEDIUM Bounty: $800

Overview / نظرة عامة

AR

مع تزايد استخدام نماذج اللغة الضخمة (LLMs) في الأنظمة، زادت هجمات حقن التعليمات (Prompt Injection). يوضح هذا التقرير كيفية تجاوز حواجز الحماية وإرغام روبوت الدعم على تسريب التعليمات الداخلية وأكواد النظام الحساسة.

EN

Integrating Large Language Models (LLMs) in applications introduces Prompt Injection vectors. This writeup discusses how safety alignments were bypassed, forcing support bots to reveal system instructions.

Exploitation Methodology / آلية الاستغلال

  1. تحديد تطبيق الدردشة الذكي وقراءة الاستجابات المبدئية لفهم نطاق الخدمة وقواعدها.
  2. استخدام هجمات تجاوز السياق (Context Switching) لتغيير دور البوت البرمجي.
  3. تمرير مدخلات حقن تطلب من البوت التصرف كمبرمج النظام وطباعة التعليمات السرية السابقة بالكامل.
  4. نجاح سحب ملف التهيئة ومفاتيح ربط الـ API المخفية في النظام الداخلي.

Remediation / التخفيف والحماية

AR

استخدم جدار حماية للمدخلات (Input Guardrails) لتنقية فحص المدخلات قبل إرسالها للـ LLM، وعالج المخرجات لتصفية الكلمات السرية ومفاتيح الـ APIs بشكل تلقائي.

EN

Deploy input and output guardrail models (like Llama Guard) to filter toxic/malicious prompts, and sanitize context windows to prevent data leakage.
