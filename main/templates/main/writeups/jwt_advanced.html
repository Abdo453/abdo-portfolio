هجمات JWT المتقدمة وتزوير التواقيع

Advanced JWT Attacks: Algorithm Confusion and Key Cracking

HIGH Bounty: $2,000

Overview / نظرة عامة

AR

تعتمد العديد من تطبيقات الويب الحديثة على JSON Web Tokens (JWT) للمصادقة وتخزين الجلسات. يوضح هذا التقرير كيفية استغلال التكوينات الخاطئة في التحقق من التوقيع لتجاوز المصادقة بالكامل والدخول كمسؤول.

EN

Modern applications heavily rely on JSON Web Tokens (JWT) for session management. This writeup illustrates how verification implementation flaws (like algorithm confusion or weak keys) were exploited to bypass access control.

Exploitation vectors / نوافذ الاستغلال

  1. تخطي التحقق من التوقيع عن طريق تعيين خوارزمية التشفير إلى "none" في رأس التوكن.
  2. استغلال خلط الخوارزمية (Algorithm Confusion) بتزوير توقيع RS256 باستخدام خوارزمية متماثلة HS256 بمفتاح عام مكشوف للجميع.
  3. تخمين وكسر كلمات السر الضعيفة للـ HMAC باستخدام هجوم القوة العمياء القائم على القواميس.

Cracking & Forging Commands / أوامر الكسر والتزوير

JWT Hash Cracking using John the Ripper / كسر مفتاح JWT 
john --format=HMAC-256 --wordlist=passwords.txt jwt_hash.txt
JWT Tool Encoding / تشفير توكن معدل 
jwt_tool.py [JWT_TOKEN] -X a -r admin

Remediation / التخفيف والحماية

AR

لا تثق أبداً بـ alg header المار من المتصفح دون فلترة، واحرص على استخدام مكتبات مصادقة قياسية محدثة ومفاتيح تشفير معقدة وغير قابلة للتخمين.

EN

Never trust the alg header from user input without validation, enforce cryptographic checks on the backend, and employ strong secret keys for signing tokens.
