من فحص النواة التلقائي إلى ثغرة P1

From Nuclei Automation to P1 Critical Vulnerability Chain

CRITICAL Bounty: $3,000

Overview / نظرة عامة

AR

يوضح هذا التقرير كيفية الانتقال من مسح بسيط باستخدام أداة Nuclei لاكتشاف ملفات مكشوفة، إلى بناء سلسلة استغلال كاملة تؤدي لتشغيل الأوامر عن بعد (RCE) على خادم حساس والحصول على مكافأة بقيمة 3,000 دولار.

EN

This writeup details how an initial template match in Nuclei scanner pointing to exposed config files was leveraged into a full Remote Code Execution (RCE) chain on a sensitive portal, earning a $3,000 bounty.

Exploitation Steps / خطوات الاستغلال

  1. تشغيل فحص Nuclei على النطاق المستهدف واكتشاف ملف تكوين Git مكشوف /.git/config.
  2. تحميل مستودع Git بالكامل باستخدام أداة git-dumper لاستعادة الكود البرمجي للموقع.
  3. مراجعة الكود البرمجي واكتشاف مفاتيح AWS السحابية وكلمة مرور قاعدة البيانات المكتوبة بشكل صلب (Hardcoded).
  4. استغلال لوحة تحكم إدارية داخلية مكشوفة عن طريق تسجيل الدخول بالبيانات المستخرجة.
  5. رفع ملف webshell عبر ميزة ترقية وتحديث ملفات التكوين لتشغيل أوامر النظام كـ root.

Testing Commands / الأوامر المستخدمة

Nuclei Scan Command / أمر فحص النواة 
nuclei -u https://target.com -t exposures/configs/git-config.yaml
Git Dumper Tool / استخراج مستودع جيت 
git-dumper https://target.com/.git/ output_dir/

Remediation / التخفيف والحماية

AR

احرص على تقييد الوصول للمجلدات الحساسة مثل .git في خادم الويب (Nginx/Apache)، ولا تقم أبداً بكتابة كلمات المرور بشكل صلب داخل الكود.

EN

Always restrict public access to sensitive folders like .git in Nginx/Apache configuration, and never hardcode API keys or credentials in source code files.
