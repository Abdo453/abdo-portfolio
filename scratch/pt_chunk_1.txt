1. Active Reconnaissance & Subdomain Enumeration
1.1 Overview
Active subdomain enumeration involves direct interaction with the target’s DNS infrastructure to discover subdomains that may not be publicly indexed. This is critical for expanding the attack surface beyond what passive reconnaissance reveals.
1.2 DNS Brute Forcing
Tools & Commands
Gobuster DNS Mode:
gobuster dns -d target.com -w wordlist.txt -t 50 --delay 100ms -o gobuster_results.txt
FFuF for Subdomain Fuzzing:
ffuf -w subdomains.txt -u https://FUZZ.target.com -mc 200,301,302 -t 100 -rate 50 -o fuf_results.json
Puredns (MassDNS Wrapper):
puredns -r resolvers.txt bruteforce wordlist.txt target.com
MassDNS (High-speed resolver):
massdns -r lists/resolvers.txt -t A domains.txt -o results.txt
ShuffleDNS (ProjectDiscovery):
# Subdomain resolvingshuffledns -d example.com -list subdomains.txt -r resolvers.txt -mode resolve# Subdomain bruteforcingshuffledns -d hackerone.com -w wordlist.txt -r resolvers.txt -mode bruteforce
1.3 Permutation & Alteration Techniques
Altdns:
python altdns.py -i input_domains.txt -o ./output/path -w altdns/words.txt
Gotator (Advanced permutation):
gotator -sub domains.txt -perm permutations.txt -depth 1 -numbers 10 -mindup -adv -md
ShuffleDNS for permutations:
shuffledns -l all.txt -r resolvers.txt -o permuted.txt

1.8 Best Practices & Anti-Detection
Rate limiting: Add delays between requests (--delay 100ms)
Rotate DNS resolvers: Use multiple resolver lists
Use VPS: Run active enumeration from cloud instances
Respect scope: Always check bug bounty scope before brute forcing
Wildcard detection: Filter wildcard responses using puredns or shuffledns
