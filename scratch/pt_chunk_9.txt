9.1 PTES (Penetration Testing Execution Standard)
Phase 1: Pre-engagement Interactions
Define project scope and objectives
Establish Rules of Engagement (RoE)
Set testing windows and emergency contacts
Get written authorization
Define deliverables and timeline

Passive reconnaissance
Active reconnaissance
DNS enumeration
Certificate transparency logs
Search engine dorks
Social media analysis
Data breach repositories (HaveIBeenPwned)
Phase 3: Threat Modeling
Analyze business context
Identify high-value assets
Map probable attack paths
Prioritize targets based on risk
Create threat actor profiles
Phase 4: Vulnerability Analysis
Automated scanning (Nessus, OpenVAS)
Manual verification of findings
False positive elimination
Vulnerability chaining analysis
CVSS scoring
Phase 5: Exploitation
Controlled exploitation attempts
Proof-of-concept development
Impact demonstration
Avoid production disruption

Phase 6: Post-Exploitation
Privilege escalation attempts
Lateral movement testing
Data access demonstration
Persistence testing (with permission)
Pivoting to internal networks
Impact assessment
Phase 7: Reporting
Executive summary
Technical findings
Risk ratings and CVSS scores
Remediation roadmap
Retesting requirements

9.4 Testing Types
Black Box Testing
No prior knowledge of target
Simulates external attacker
Time-consuming but realistic
May miss internal vulnerabilities
Grey Box Testing
Limited knowledge (credentials, basic architecture)
Most common approach
Balances realism with efficiency
Tests access control and lateral movement
White Box Testing
Full knowledge (source code, documentation)
Most thorough analysis
Identifies root causes
Does not simulate real attacker
Red Team vs Penetration Test
Penetration Test: Time-boxed, scoped, cooperative
Red Team: Extended duration, broader scope, stealthy, tests detection/response
